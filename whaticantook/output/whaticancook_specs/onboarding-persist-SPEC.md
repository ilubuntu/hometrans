# 引导完成状态持久化 规格文档

## 需求概述
用户完成或跳过引导后，应用应在本地持久化记录"引导已完成"。此后每次启动应用都读取该标记：已完成则跳过引导直接进入首页；未完成（如首次安装或清除数据后）才进入引导。引导标记在正常运行期间不会被应用自身重置，只有清除应用本地数据才能恢复引导流程。

## 原子场景

### 场景1: 完成引导时写入标记
- **触发条件**: 用户在引导页点击"Start cooking"（末页）或"Skip"（任意页）。
- **动作序列**: 用户触发引导完成动作。
- **预期行为**: 应用将"引导已完成"标记写入本地持久化存储，并在内存中同步更新为 true。
- **逻辑步骤**:
  1. 引导页调用完成逻辑。
  2. 将布尔标记 true 持久化到本地设置存储（键值对，键名为引导完成标记）。
  3. 更新内存中的引导完成状态流为 true，使所有订阅方收到通知。
  4. 导航到首页并从导航栈移除引导页。

### 场景2: 再次启动跳过引导
- **触发条件**: 引导已完成，应用被关闭后重新启动。
- **动作序列**: 用户再次启动应用。
- **预期行为**: 直接进入首页（Discover），不展示引导页。
- **逻辑步骤**:
  1. 应用启动时读取本地持久化的引导完成标记。
  2. 应用启动闪屏保持显示，直到引导与主题等启动状态加载完成（标记读取就绪）。
  3. 标记为 true → 将首页设为启动目标页，跳过引导页。
  4. 标记为 false → 将引导页设为启动目标页（见 onboarding-flow 规格）。

### 场景3: 清除应用数据后恢复引导
- **触发条件**: 用户清除应用全部本地数据（如系统设置中清除应用数据/存储）。
- **动作序列**: 用户清除数据后重新启动应用。
- **预期行为**: 引导完成标记被一并清除（视为未完成），再次进入引导页。
- **逻辑步骤**:
  1. 清除数据使本地设置存储恢复默认（引导完成标记默认为 false / 未设置）。
  2. 再次启动时读取到未完成，进入引导页。

## 边界场景
- 应用内无"重新查看引导"入口：正常使用下无法再次触发引导。
- 引导标记与主题等其他设置共存于同一本地设置存储，但互不影响（清除数据会同时清除二者）。
- 标记读取失败/未设置时按"未完成"处理（默认进入引导），保证首次安装体验正确。

## 偏差说明
- 持久化技术为 Android SharedPreferences（键值存储，文件名 `wcc_settings`，键 `onboarding_complete`）。规格按"本地持久化布尔标记"描述，迁移到 HarmonyOS 时应使用等价的本地偏好存储（如用户首选项 Preferences），不绑定具体框架。
- 代码中默认标记值为 false（未完成），与"首次安装进入引导"一致；未发现运行时重置入口，与验收重点"清除应用数据后才重新出现引导"一致。
