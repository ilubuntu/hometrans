# 设置页面基础信息规格文档

## 需求概述
设置页面以分组卡片的形式展示应用的配置信息和元数据。页面分为「外观」「数据」「关于」三组内容：外观组包含主题切换选项；数据组包含清空食材库操作；关于组显示应用名称、版本号、构建技术和离线说明。页面顶部提供标题和返回按钮。

## 原子场景

### 场景1: 设置页面整体布局
- **触发条件**: 用户从首页点击设置入口进入设置页
- **动作序列**:
  1. 用户点击首页右上角设置图标
- **预期行为**:
  - 页面顶部显示导航栏，含返回按钮和标题「Settings」
  - 页面内容分为三组：APPEARANCE、DATA、ABOUT
  - 每组以大写标签开头，内容放在圆角卡片中
  - 整体可垂直滚动
  - 底部导航栏隐藏（设置页非顶级 Tab）
- **逻辑步骤**:
  1. 路由到设置页
  2. 渲染顶部导航栏（返回按钮 + 标题）
  3. 依次渲染三组分区标签和内容卡片

### 场景2: 外观（Appearance）分组
- **触发条件**: 设置页已加载
- **动作序列**:
  1. 用户查看外观分组
- **预期行为**:
  - 显示大写分区标签「APPEARANCE」
  - 卡片内显示「Theme」标题
  - 下方显示三个等宽主题选项标签（Match system / Light / Dark）
  - 当前选中的主题标签高亮
- **逻辑步骤**:
  1. 读取当前主题模式
  2. 渲染分区标签
  3. 渲染主题卡片，含标题和三个选项标签
  4. 当前主题对应的标签高亮

### 场景3: 数据（Data）分组
- **触发条件**: 设置页已加载
- **动作序列**:
  1. 用户查看数据分组
- **预期行为**:
  - 显示大写分区标签「DATA」
  - 卡片内显示一行操作项
  - 左侧为红色删除图标
  - 中间显示标题「Clear pantry」和说明「Remove all ingredients you've added」
  - 整行可点击
- **逻辑步骤**:
  1. 渲染分区标签
  2. 渲染清空食材库操作行
  3. 含图标、标题和说明文案

### 场景4: 关于（About）分组
- **触发条件**: 设置页已加载
- **动作序列**:
  1. 用户查看关于分组
- **预期行为**:
  - 显示大写分区标签「ABOUT」
  - 卡片内显示三行键值对：
    - App — WhatCanICook
    - Version — 1.0.0
    - Built with — Jetpack Compose
  - 下方显示离线说明文案：「Recipe data is bundled on-device, so the whole app works fully offline. Typeface: Plus Jakarta Sans (SIL Open Font License).」
- **逻辑步骤**:
  1. 渲染分区标签
  2. 渲染三行键值对（标签左对齐，值右对齐）
  3. 渲染离线说明文案

### 场景5: 返回设置页
- **触发条件**: 用户在设置页
- **动作序列**:
  1. 用户点击顶部返回按钮
- **预期行为**:
  - 页面返回到进入设置前的页面（首页 Discover）
  - 首页底部导航高亮 Discover
- **逻辑步骤**:
  1. 接收返回按钮点击
  2. 执行返回栈弹出操作
  3. 回到来源页面（首页）

## 边界场景
- **设置页滚动**: 内容超出屏幕时可垂直滚动，底部有安全间距
- **主题选项固定为三个**: Match system、Light、Dark
- **关于信息为静态文本**: 不随用户操作变化（版本号等固定值）

## 偏差说明
- 设置页不显示底部导航栏（非顶级 Tab）
- 三组内容按固定顺序排列：Appearance -> Data -> About
- 关于信息为硬编码的应用元数据，不从网络获取
