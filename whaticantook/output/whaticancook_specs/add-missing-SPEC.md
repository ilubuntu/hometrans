# 可一键补齐缺少食材SPEC

当某菜谱存在缺少的必需食材时，菜谱详情页提供"一键补齐缺少食材"按钮，点击后把全部缺少的必需食材一次性加入食材库，详情页状态随之从"缺少"切换为"准备就绪"。

## 场景一

### 场景概述

某菜谱存在缺少的必需食材时，详情页展示"一键补齐"按钮，点击后补齐全部缺少食材并转为可做状态

### 场景逻辑步骤

- 1. 用户打开一个存在缺少必需食材的菜谱详情页（例如只具备部分必需食材的 Chicken Fried Rice）
- 2. 详情页状态区在"缺少 N 个食材"标题与缺少食材列表下方，展示一个全宽"一键补齐缺少食材（Add missing to pantry）"按钮
- 3. 用户点击该按钮
- 4. 系统把当前缺少的全部必需食材一次性加入食材库：
  - 4.1 每个缺少食材在加入前对其名称做归一化处理（大小写、复数、同义词）
  - 4.2 归一化后非空的食材写入食材库；名称为空则跳过
  - 4.3 补齐加入的食材统一归入"其他"分组
- 5. 食材库更新后，系统重新比对，该菜谱缺少数变为 0
- 6. 详情页状态区从"缺少 N 个食材"切换为"准备就绪（You're all set）"可做状态，缺少食材列表与"一键补齐"按钮随之消失

## 场景二

### 场景概述

补齐按钮仅在详情页且仅在该菜谱存在缺少必需食材时出现

### 场景逻辑步骤

- 1. 当某菜谱存在至少一个缺少的必需食材时，详情页状态区展示"一键补齐缺少食材"按钮
- 2. 当某菜谱全部必需食材已满足（可做状态）时，详情页展示"准备就绪"提示框，"一键补齐"按钮不出现
- 3. 首页、搜索页、收藏页等菜谱卡片不提供"一键补齐"入口，仅菜谱详情页提供

## 场景三

### 场景概述

补齐范围只覆盖"缺少的必需食材"，已满足项与可选食材不受影响

### 场景逻辑步骤

- 1. 某菜谱食材清单含 7 个必需食材与 1 个可选食材，当前缺少 4 个必需食材
- 2. 用户点击"一键补齐缺少食材"按钮
- 3. 系统仅把这 4 个缺少的必需食材加入食材库；已满足的必需食材不重复添加
- 4. 可选食材不进入补齐范围（即使该可选食材当前未在食材库，也不被补齐按钮加入）
- 5. 补齐后必需食材全部满足，详情页转为"准备就绪"可做状态

## 场景四

### 场景概述

"一键补齐缺少食材"的作用范围与不作用范围

### 场景逻辑步骤

- 1. **仅作用于** 以下入口：菜谱详情页状态区的"一键补齐缺少食材"按钮（且仅在该菜谱存在缺少必需食材时出现）
- 2. **不作用于** 以下入口：首页、搜索页、收藏页的菜谱卡片均无补齐入口；详情页做菜步骤区、食材清单区无补齐入口；可做态详情页无补齐按钮
- 3. **不作用于**：补齐不改变可选食材的拥有情况，也不移除或修改食材库中已有的任何食材

> [偏差] 补齐加入的食材在食材库中统一归入"其他"分组，而非按食材原始类别（如 Rice 归"谷物"）分类；功能结果（补齐后转可做）与需求一致。
