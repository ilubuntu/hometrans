# 主题切换规格文档

## 需求概述
设置页面的外观分组中，用户可以在「跟随系统」「浅色」「深色」三种主题模式间切换。选择后应用界面立即切换到对应主题，无需重启。三种模式分别对应：跟随设备系统设置、强制浅色、强制深色。

## 原子场景

### 场景1: 查看当前主题
- **触发条件**: 用户进入设置页
- **动作序列**:
  1. 用户查看外观分组
- **预期行为**:
  - 显示三个主题选项：Match system、Light、Dark
  - 当前生效的主题选项高亮选中
  - 默认主题为 Match system（跟随系统）
- **逻辑步骤**:
  1. 读取存储的主题模式
  2. 渲染三个选项标签
  3. 当前主题对应的标签高亮

### 场景2: 切换到深色主题
- **触发条件**: 当前为浅色或跟随系统（系统为浅色）
- **动作序列**:
  1. 用户点击「Dark」选项标签
- **预期行为**:
  - 「Dark」标签变为选中高亮
  - 其他标签取消高亮
  - 整个应用界面立即切换到深色主题（深色背景、浅色文字）
  - 无需重启或刷新
- **逻辑步骤**:
  1. 接收主题选择事件
  2. 更新主题模式为深色
  3. 持久化主题设置到本地存储
  4. 主题状态通过响应式流传播
  5. 应用根组件感知变更，切换深色配色方案
  6. 所有界面元素重新渲染为深色主题

### 场景3: 切换到浅色主题
- **触发条件**: 当前为深色或跟随系统（系统为深色）
- **动作序列**:
  1. 用户点击「Light」选项标签
- **预期行为**:
  - 「Light」标签高亮
  - 界面切换到浅色主题（浅色背景、深色文字）
- **逻辑步骤**:
  1. 更新主题模式为浅色
  2. 持久化设置
  3. 响应式传播 -> 根组件切换浅色配色
  4. 界面重新渲染

### 场景4: 切换到跟随系统
- **触发条件**: 当前为强制浅色或深色
- **动作序列**:
  1. 用户点击「Match system」选项标签
- **预期行为**:
  - 「Match system」标签高亮
  - 界面主题跟随设备系统的暗色模式设置
  - 系统为深色 -> 应用深色；系统为浅色 -> 应用浅色
- **逻辑步骤**:
  1. 更新主题模式为跟随系统
  2. 持久化设置
  3. 读取设备系统暗色模式状态
  4. 根据系统状态决定使用浅色或深色配色

### 场景5: 跟随系统模式下系统主题变化
- **触发条件**: 主题设为 Match system
- **动作序列**:
  1. 用户在设备设置中切换系统暗色/浅色模式
- **预期行为**:
  - 应用主题自动跟随系统变化
  - 系统切深色 -> 应用变深色
  - 系统切浅色 -> 应用变浅色
- **逻辑步骤**:
  1. 主题模式为 SYSTEM
  2. 监听系统暗色模式变更
  3. 系统变更时重新计算实际主题
  4. 切换配色方案

### 场景6: 主题切换全局生效
- **触发条件**: 用户在设置页切换主题
- **动作序列**:
  1. 切换主题后返回首页
  2. 进入搜索页、食材库页、详情页
- **预期行为**:
  - 所有页面均应用新主题
  - 包括卡片、按钮、文字、背景等所有 UI 元素
- **逻辑步骤**:
  1. 主题在应用最外层包裹
  2. 配色方案变更影响所有子组件
  3. 所有页面自动应用新主题

## 边界场景
- **首次安装**: 默认 Match system
- **主题切换动画**: 配色变更即时生效，无过渡动画延迟
- **三种选项互斥**: 同时只有一个选项被选中

## 偏差说明
- 主题切换通过响应式状态流实现，即时生效无需重启
- Match system 模式实时跟随系统设置变化
- 主题在最外层包裹整个应用，确保全局一致性
