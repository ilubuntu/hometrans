# 主题切换SPEC

用户在"设置"页面的外观区块从"Light / Dark / Match system"三个主题选项中选择其一，决定整个应用的配色方案（浅色 / 深色 / 跟随系统）。

## 场景一

### 场景概述

用户在"设置"页面外观区块选择一个主题选项

### 场景逻辑步骤

- 1. 用户在"首页"右上角点击设置图标进入"设置"页面
- 2. 外观区块展示主题项"Theme"及其下三个选项，文案依次为"Light" / "Dark" / "Match system"
- 3. 三个选项中与当前生效主题一致的那个处于选中高亮状态
- 4. 用户点击其中一个选项
- 5. 被点击的选项立即变为选中高亮状态，其余两个变为未选中状态
- 6. 新选中的主题立即生效（具体配色变化见场景二至场景四）
- 7. 切换过程不关闭"设置"页面，也不重置该页面或任何其他页面已有的数据与浏览位置

## 场景二

### 场景概述

当生效主题为"Light"时，全应用呈现浅色配色

### 场景逻辑步骤

- 1. 当前生效主题为"Light"（含用户刚刚选择"Light"，或启动时已是"Light"）
- 2. 应用所有页面（设置页、首页、搜索页、食材库页、收藏页、详情页、底部导航）统一呈现浅色配色方案（浅色背景、深色文字、相应的主色/强调色）
- 3. 顶部状态栏与底部导航栏图标外观相应调整为浅色底深色图标风格
- 4. 此后无论系统处于深色还是浅色模式，应用始终维持浅色配色，不受系统变化影响

## 场景三

### 场景概述

当生效主题为"Dark"时，全应用呈现深色配色

### 场景逻辑步骤

- 1. 当前生效主题为"Dark"（含用户刚刚选择"Dark"，或启动时已是"Dark"）
- 2. 应用所有页面统一呈现深色配色方案（深色背景、浅色文字、相应的主色/强调色）
- 3. 顶部状态栏与底部导航栏图标外观相应调整为深色底浅色图标风格
- 4. 此后无论系统处于深色还是浅色模式，应用始终维持深色配色，不受系统变化影响

## 场景四

### 场景概述

当生效主题为"Match system"时，配色跟随系统深浅色模式

### 场景逻辑步骤

- 1. 当前生效主题为"Match system"（含用户刚刚选择，或从未设置过时的默认值）
- 2. 实时读取系统当前的深浅色模式：
  - 2.1 系统为浅色模式时，应用呈现浅色配色（同场景二效果）
  - 2.2 系统为深色模式时，应用呈现深色配色（同场景三效果）
- 3. 应用处于前台时，若系统在深色与浅色之间切换，应用配色立即跟随刷新，无需重新打开页面

## 场景五

### 场景概述

切换主题时保持当前页面与数据状态

### 场景逻辑步骤

- 1. 用户已在某个页面（如"设置"页、"首页"等）浏览或输入了内容（如搜索词、所选分类、列表滚动位置）
- 2. 用户选择任一主题选项，全应用配色立即刷新
- 3. 切换仅刷新配色，不发生页面跳转或重建：用户当前所在页面不变，已选分类、已输入文本、列表位置等已有状态均不丢失

## 场景六

### 场景概述

"主题切换"的作用范围与不作用范围

### 场景逻辑步骤

- 1. **仅作用于** 以下界面: 应用全部页面的配色方案；"设置"页面外观区块三个主题选项的选中高亮状态
- 2. **仅可通过** 以下入口修改: "设置"页面外观区块的主题选项
- 3. **不作用于** 以下内容: 页面内的具体文案、菜谱数据、食材库内容、收藏列表、用户已输入的搜索词或已选分类（这些数据在切换主题时保持不变）

> [偏差] 需求未明确"Match system"跟随系统切换的实时性与默认值；代码默认主题为"Match system"，且前台时跟随系统切换即时刷新（见场景四）。需求未提及状态栏/导航栏图标外观随主题调整；代码会同步调整（见场景二、场景三步骤 3）。均为补充说明，不与需求冲突。
> [交叉引用] 主题选择的重启持久保存见 `theme-persist-SPEC.md`；主题选项所在的"设置"页面区块结构见 `settings-overview-SPEC.md`。
