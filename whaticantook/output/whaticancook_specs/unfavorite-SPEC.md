# 取消收藏规格文档

## 需求概述
用户可以在收藏页或详情页取消收藏菜谱。取消收藏后，该菜谱应立即从收藏列表中移除，所有页面的心形状态同步更新为未收藏。当取消最后一个收藏导致列表为空时，收藏页应自动恢复显示空状态。

## 原子场景

### 场景1: 在收藏页取消收藏
- **触发条件**: 收藏列表中有至少 1 个菜谱
- **动作序列**:
  1. 用户点击某菜谱卡片的心形按钮
- **预期行为**:
  - 心形从实心变为空心
  - 该菜谱从收藏列表中移除（带动画效果）
  - 收藏数量文案减少
  - 其他页面（首页/搜索/详情）的心形同步变为空心
- **逻辑步骤**:
  1. 接收 toggle 收藏事件
  2. 当前菜谱 isFavorite=true -> 传 false 给收藏接口
  3. 从本地存储删除该菜谱的收藏记录
  4. 收藏数据变更通过响应式流传播
  5. 收藏页列表移除该项，数量更新
  6. 其他页面菜谱的 isFavorite 更新为 false

### 场景2: 在详情页取消收藏
- **触发条件**: 用户在某已收藏菜谱的详情页
- **动作序列**:
  1. 用户点击详情页顶部的心形按钮
- **预期行为**:
  - 心形从实心变为空心（带缩放淡出动画）
  - 该菜谱从收藏列表移除
  - 返回收藏页时该菜谱不再出现
- **逻辑步骤**:
  1. 接收详情页 toggle 事件
  2. 执行取消收藏（删除记录）
  3. 详情页通过响应式流更新 isFavorite=false
  4. 收藏列表数据流更新，移除该项

### 场景3: 取消唯一收藏恢复空态
- **触发条件**: 收藏列表中仅有 1 个菜谱
- **动作序列**:
  1. 用户取消该菜谱的收藏
- **预期行为**:
  - 列表中该项移除
  - 列表为空后自动切换到空状态
  - 显示空态标题「No saved recipes yet」
  - 显示操作按钮「Browse recipes」
- **逻辑步骤**:
  1. 取消收藏删除最后一条记录
  2. 收藏数据流发射空列表
  3. items.isEmpty() 且 loading=false -> 满足空态条件
  4. 渲染空状态组件（含操作按钮）

### 场景4: 取消收藏后再次收藏
- **触发条件**: 菜谱已被取消收藏（isFavorite=false）
- **动作序列**:
  1. 用户再次点击心形按钮
- **预期行为**:
  - 心形从空心变为实心
  - 菜谱重新加入收藏列表
  - 收藏时间更新为当前时间（排到列表顶部）
- **逻辑步骤**:
  1. toggle 操作检测 isFavorite=false -> 传 true
  2. 写入收藏记录（含新时间戳）
  3. 响应式更新所有页面

### 场景5: 批量取消收藏
- **触发条件**: 收藏列表有多个菜谱
- **动作序列**:
  1. 用户依次取消多个菜谱的收藏
- **预期行为**:
  - 每次取消后列表立即更新
  - 数量文案逐次递减
  - 全部取消后显示空态
- **逻辑步骤**:
  1. 每次 toggle 操作独立处理
  2. 响应式流逐次更新
  3. 列表清空后切换空态

## 边界场景
- **快速连续点击同一心形**: toggle 交替执行，最终状态取决于最后操作
- **取消收藏时菜谱卡片动画**: 列表项支持移除动画
- **取消后菜谱数据仍可访问**: 取消收藏仅移除收藏记录，菜谱本身仍存在于菜谱库

## 偏差说明
- 取消收藏为 toggle 操作的逆向（当前已收藏时点击即取消）
- 数据通过响应式流自动传播，无需手动刷新
- 取消收藏后菜谱仍可在首页/搜索页正常浏览，仅心形状态变化
