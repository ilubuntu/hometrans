# 收藏顺序SPEC

收藏列表按收藏时间从新到旧排列，即最近收藏的菜谱排在最前；先收藏的菜谱排在后面。收藏顺序由每次收藏时记录的时间戳决定，并在收藏页列表中据此呈现。

## 场景一

### 场景概述

依次收藏多道菜谱，后收藏的排在更前

### 场景逻辑步骤

- 1. 用户先收藏菜谱 A
- 2. 用户随后收藏菜谱 B
- 3. 系统为每次收藏记录当时的时间戳：A 的时间戳早于 B 的时间戳
- 4. 用户进入收藏页
- 5. 收藏列表按收藏时间从新到旧排列：后收藏的菜谱 B 排在菜谱 A 之前

## 场景二

### 场景概述

新增收藏即时出现在列表顶部

### 场景逻辑步骤

- 1. 用户收藏页列表中已有若干菜谱，按收藏时间倒序排列
- 2. 用户在首页、搜索页、收藏页卡片或详情页任一处新收藏一道菜谱
- 3. 系统以当前时间作为该菜谱收藏记录的时间戳
- 4. 该菜谱即时出现在收藏列表的最顶部（因其时间戳最新）
- 5. 原有菜谱相对顺序保持不变，整体下移一位

## 场景三

### 场景概述

取消收藏不影响其余菜谱的相对顺序

### 场景逻辑步骤

- 1. 用户收藏页列表中有多道菜谱，按收藏时间倒序排列
- 2. 用户取消其中某道菜谱的收藏（在任一入口）
- 3. 该菜谱从收藏列表移除
- 4. 其余菜谱仍按各自原收藏时间倒序排列，相对顺序不变

## 场景四

### 场景概述

重新收藏（先取消再加入）的菜谱移至列表顶部

### 场景逻辑步骤

- 1. 某菜谱 A 此前已收藏，位于收藏列表某一位置
- 2. 用户取消菜谱 A 的收藏，A 从列表移除
- 3. 用户随后再次收藏菜谱 A
- 4. 系统以"再次收藏"的当前时间作为菜谱 A 的新时间戳
- 5. 菜谱 A 移至收藏列表的最顶部，而非回到原位置

## 场景五

### 场景概述

"收藏顺序"的作用范围与不作用范围

### 场景逻辑步骤

- 1. **仅作用于** 收藏页（Saved）列表的排列顺序
- 2. **顺序由收藏时间戳决定**：最近收藏优先，与菜谱的标题、分类、制作时间、匹配度等无关
- 3. **不作用于** 首页菜谱列表——首页列表按食材匹配度等独立排序，不受收藏顺序影响
- 4. **不作用于** 搜索页列表——搜索页列表按所选排序项（Best match / Quickest / Fewest missing）排列，不受收藏顺序影响

> [偏差] 需求描述未提及"重新收藏移至顶部"的情形，实际重新收藏会刷新时间戳使该菜谱移至列表顶部（覆盖式写入新时间戳），为合理的实现细节。
