# 搜索页分类筛选SPEC

搜索页分类筛选条提供"All"项与各菜谱分类项（Breakfast、Lunch、Dinner、Dessert、Snack、Drinks），用户选择某一分类后，结果列表即时过滤为该分类菜谱；分类筛选与搜索词、可做筛选、排序可同时生效、相互叠加。

## 场景一

### 场景概述

用户选择一个具体分类，结果列表只展示该分类菜谱

### 场景逻辑步骤

- 1. 用户位于搜索页分类筛选条，看到横排的"All"项与各分类项（每项前带分类图标）
- 2. 用户点击某一具体分类项（如"Breakfast"）
- 3. 该分类项进入选中态（背景高亮、文字反色、去除边框），原选中项退出选中态
- 4. 结果列表即时过滤，只展示属于该分类（早餐）的菜谱

## 场景二

### 场景概述

用户选择"All"，取消分类过滤，结果列表恢复全部菜谱

### 场景逻辑步骤

- 1. 用户位于搜索页，当前已选中某一具体分类（如"Breakfast"），列表只显示早餐菜谱
- 2. 用户点击"All"项
- 3. "All"项进入选中态，原分类项退出选中态
- 4. 结果列表取消分类过滤，恢复展示全部菜谱

## 场景三

### 场景概述

分类筛选与搜索词共同生效，二者叠加

### 场景逻辑步骤

- 1. 用户在搜索页输入搜索词（如"banana"）
- 2. 用户再点击某一分类项（如"Breakfast"）
- 3. 系统先按搜索词过滤出匹配菜谱，再按所选分类进一步过滤
- 4. 结果列表只展示既匹配搜索词、又属于所选分类的菜谱（即早餐分类中匹配 banana 的菜谱）
- 5. 若用户随后清空搜索词，分类过滤仍然保留；若用户随后改选"All"，搜索词过滤仍然保留

## 场景四

### 场景概述

分类项为单选互斥，任意时刻最多一项选中，默认为"All"

### 场景逻辑步骤

- 1. 用户位于搜索页分类筛选条
- 2. "All"与各具体分类项构成一个单选组
- 3. 任意时刻最多只有一项处于选中态（"All"或某一个具体分类）
- 4. 应用首次进入搜索页时，"All"项默认处于选中态（即默认不应用分类过滤）

## 场景五

### 场景概述

"搜索页分类筛选"的作用范围与不作用范围

### 场景逻辑步骤

- 1. **仅作用于** 搜索页结果列表的分类过滤
- 2. **不与首页共用状态**：首页虽也有分类筛选，但二者各自独立，在搜索页选定的分类不影响首页，反之亦然
- 3. **不与"可做"筛选、排序项互相影响**：分类筛选、可做筛选、排序三者各自独立，可任意组合同时生效
- 4. **分类项集合固定** 为应用内置的菜谱分类，用户不能新增、编辑或删除分类项
