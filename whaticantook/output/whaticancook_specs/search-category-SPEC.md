# Search 分类筛选 规格文档

## 需求概述
搜索页提供与首页一致的分类型菜谱筛选。用户在分类筛选条中选择某个菜谱分类（如早餐、午餐、晚餐、甜点、零食、饮品），结果列表只展示属于该分类的菜谱；选择"全部"则展示所有分类的菜谱。分类筛选可与文本搜索、可做筛选、排序方式组合使用，筛选条以可横向滚动的标签组形式呈现，当前选中分类高亮。

## 原子场景

### 场景1: 分类筛选条展示
- **触发条件**: 搜索页渲染分类筛选区。
- **动作序列**: 用户查看横向筛选条。
- **预期行为**: 筛选条展示"全部"标签与各菜谱分类标签（含分类图标），与首页分类一致；当前选中分类高亮，默认选中"全部"。
- **逻辑步骤**:
  1. 取菜谱分类集合（与首页一致）。
  2. 渲染"全部"标签 + 各分类标签。
  3. 当前选中（默认"全部"）高亮。

### 场景2: 选择分类过滤结果
- **触发条件**: 用户点击某分类标签（非"全部"）。
- **动作序列**: 用户点击如"早餐"。
- **预期行为**: 结果列表只展示属于该分类的菜谱，其余分类菜谱被过滤；选中标签高亮。
- **逻辑步骤**:
  1. 记录选中的分类。
  2. 对结果按"菜谱分类 == 选中分类"过滤。
  3. 刷新结果列表，高亮选中标签。

### 场景3: 选择"全部"展示所有分类
- **触发条件**: 当前选中某具体分类。
- **动作序列**: 用户点击"全部"标签。
- **预期行为**: 取消分类过滤，结果展示所有分类的菜谱；"全部"标签高亮。
- **逻辑步骤**:
  1. 清除选中分类（设为"全部"）。
  2. 跳过分类过滤。
  3. 展示所有分类菜谱。

### 场景4: 分类筛选与其它条件组合
- **触发条件**: 已输入搜索词或开启可做筛选或选择排序。
- **动作序列**: 用户在此基础上选择某分类。
- **预期行为**: 分类筛选与文本搜索、可做筛选、排序叠加生效——结果为同时满足所有条件的菜谱，并按所选排序呈现。
- **逻辑步骤**:
  1. 应用文本过滤（如有）。
  2. 应用分类过滤。
  3. 应用可做筛选（如开启）。
  4. 应用排序。
  5. 输出组合结果。

### 场景5: 切换分类
- **触发条件**: 已选中某分类。
- **动作序列**: 用户点击另一个分类标签。
- **预期行为**: 选中切换为新分类，结果即时更新为新分类的菜谱；新标签高亮，原标签取消高亮。
- **逻辑步骤**:
  1. 更新选中分类为新值。
  2. 重新过滤与刷新。
  3. 高亮状态切换。

### 场景6: 验证示例（Breakfast 筛选）
- **触发条件**: 搜索页默认筛选态。
- **动作序列**: 选择"早餐"分类。
- **预期行为**: 结果可见香蕉燕麦煎饼（早餐分类）；不应看到鸡肉炒饭（晚餐分类）。
- **逻辑步骤**:
  1. 香蕉燕麦煎饼分类为早餐 → 命中。
  2. 鸡肉炒饭分类为晚餐 → 被过滤。

## 边界场景
- 某分类无菜谱：选择该分类后结果为空，显示空态提示。
- 分类筛选为精确分类匹配（菜谱分类等于选中分类），非子串/模糊。
- 分类集合与首页一致，固定为若干个菜谱分类 + "全部"。
- 分类筛选可与文本、可做、排序任意组合。

## 偏差说明
- 无。规格按需求描述：与首页分类一致，选择分类后只展示对应分类。迁移 HarmonyOS 时应实现等价的分类筛选条（与首页共用同一套分类定义）与精确分类过滤。
- 本规格与"首页分类筛选（category-filter-SPEC.md）"行为一致、共用分类定义；搜索页差异在于可与文本/可做/排序叠加。
