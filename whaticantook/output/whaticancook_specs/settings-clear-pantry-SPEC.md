# Settings 清空食材库SPEC

用户在"设置"页面数据区块点击"Clear pantry"项，一次性清空全部已添加食材，并令所有依赖食材库的页面同步刷新。

## 场景一

### 场景概述

用户在"设置"页面点击"Clear pantry"清空全部食材

### 场景逻辑步骤

- 1. 用户在"首页"右上角点击设置图标进入"设置"页面
- 2. 用户在数据区块点击"Clear pantry"项
- 3. 应用立即清空食材库中的全部已添加食材，无需二次确认
- 4. 清空完成后无额外提示弹窗或浮动提示

## 场景二

### 场景概述

清空后首页食材数量归零并恢复空食材提示

### 场景逻辑步骤

- 1. 用户此前已添加若干食材，首页显示食材数量与"可做菜谱"区
- 2. 用户进入"设置"页面点击"Clear pantry"
- 3. 返回首页后：
  - 3.1 食材库入口卡片上的食材数量变为 0
  - 3.2 "可做菜谱"区不再展示任何菜谱
  - 3.3 首页恢复展示引导用户添加食材的空态提示，并提供前往食材库的入口

## 场景三

### 场景概述

清空后菜谱详情页缺失状态重新计算

### 场景逻辑步骤

- 1. 用户此前已添加若干食材，某菜谱详情页显示部分或全部必需食材已满足
- 2. 用户进入"设置"页面点击"Clear pantry"
- 3. 再次进入同一菜谱详情页：
  - 3.1 食材清单中此前标记为"已满足"的必需食材，全部重新标记为"缺失"
  - 3.2 缺失数量重新计算为该菜谱的全部必需食材数量
  - 3.3 缺失食材列表重新列出全部必需食材
  - 3.4 可做状态由"已备齐"变为"缺失 N 项"

## 场景四

### 场景概述

清空后搜索页与收藏页的匹配状态重新计算

### 场景逻辑步骤

- 1. 用户此前已添加若干食材，搜索页菜谱卡片显示匹配计数，收藏页菜谱卡片显示匹配计数
- 2. 用户进入"设置"页面点击"Clear pantry"
- 3. 再次进入搜索页与收藏页：
  - 3.1 各菜谱卡片的食材匹配计数刷新为 0
  - 3.2 搜索页若已开启"仅看可做"筛选，则筛选结果为空（空食材库下无任何可做菜谱）
  - 3.3 "最佳匹配"等排序按刷新后的匹配度重新排列

## 场景五

### 场景概述

"Settings 清空食材库"的作用范围与不作用范围

### 场景逻辑步骤

- 1. **仅作用于** 以下内容: 食材库中的全部已添加食材；以及依赖食材库的所有页面状态——首页食材数量与空态提示、菜谱详情页匹配/缺失状态、搜索页匹配计数与可做筛选、收藏页匹配计数、食材库页已添加列表
- 2. **仅可通过** 以下入口触发: "设置"页面数据区块的"Clear pantry"项
- 3. **不作用于** 以下内容: 收藏的菜谱关系（收藏列表本身不受影响）、菜谱种子数据、主题与引导设置 —— 这些数据有各自独立存储，清空食材库不会改动它们

> [偏差] 需求未明确清空是否需要二次确认；代码为点击即清空，无确认弹窗、无成功提示（见场景一步骤 3、4）。需求未提及搜索页与收藏页的匹配刷新；代码会同步重新计算这两处（见场景四）。均为补充说明，不与需求冲突。
> [交叉引用] 同样的"清空全部食材"语义在"食材库"页面的"Clear all"按钮也存在，见 `pantry-clear-SPEC.md`；两者入口不同但清空行为与影响范围一致。
