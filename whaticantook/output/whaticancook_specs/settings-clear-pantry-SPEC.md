# 设置页清空食材库规格文档

## 需求概述
设置页面的数据分组提供「清空食材库」操作。用户点击后，应用移除所有已添加的食材，并自动更新首页、搜索页和详情页的菜谱匹配状态。清空操作直接执行，影响全局匹配计算。

## 原子场景

### 场景1: 清空食材库
- **触发条件**: 用户已添加若干食材，进入设置页
- **动作序列**:
  1. 用户找到数据分组的「Clear pantry」操作行
  2. 用户点击该行
- **预期行为**:
  - 所有已添加食材被移除
  - 操作立即执行（无确认对话框）
  - 返回首页时食材库卡片显示空态文案「Add what you have at home」
- **逻辑步骤**:
  1. 接收清空操作点击事件
  2. 调用食材库清空接口
  3. 删除食材库中所有记录
  4. 数据变更通过响应式流传播到所有页面
  5. 各页面重新计算匹配状态

### 场景2: 清空后首页匹配状态更新
- **触发条件**: 食材库已清空
- **动作序列**:
  1. 用户返回首页
- **预期行为**:
  - 食材库卡片显示「Add what you have at home」
  - 「Ready to cook」横向列表消失（无可做菜谱）
  - 显示空食材库引导提示
  - 所有菜谱匹配计数变为 0/N
- **逻辑步骤**:
  1. 食材库数据流发射空列表
  2. 首页重新计算：pantryNames 为空
  3. 所有菜谱 matchAgainst(空列表) -> haveCount=0, missing=全部必需食材
  4. cookNow 列表为空（pantryNames.isNotEmpty() 为 false）
  5. 显示空食材库引导

### 场景3: 清空后搜索页排序更新
- **触发条件**: 食材库已清空
- **动作序列**:
  1. 用户进入搜索页
  2. 查看菜谱排序
- **预期行为**:
  - 所有菜谱匹配计数变为 0/N
  - 排序结果根据当前排序模式重新计算
  - 开启 Cookable 筛选时结果为空
- **逻辑步骤**:
  1. 食材库数据流更新
  2. 搜索页重新计算每个菜谱的 CookMatch
  3. 所有 ratio=0，missing=全部必需食材
  4. 排序/筛选基于新匹配状态

### 场景4: 清空后详情页状态更新
- **触发条件**: 食材库已清空，之前某菜谱为「已齐备」状态
- **动作序列**:
  1. 用户进入该菜谱详情页
- **预期行为**:
  - 之前显示「You're all set!」的菜谱变为「You're missing N ingredients」
  - 食材清单中所有必需食材变为缺失状态
  - 显示「Add missing to pantry」操作按钮
- **逻辑步骤**:
  1. 食材库数据流更新
  2. 详情页重新计算匹配
  3. isCookable 从 true 变为 false
  4. CookStatusSection 从「已齐备」切换到「缺少食材」

### 场景5: 食材库已为空时清空
- **触发条件**: 食材库已无任何食材
- **动作序列**:
  1. 用户点击 Clear pantry
- **预期行为**:
  - 操作正常执行（删除 0 条记录）
  - 不报错，不崩溃
  - 状态无变化（已为空态）
- **逻辑步骤**:
  1. 清空操作执行 DELETE ALL
  2. 无记录可删，操作幂等
  3. 数据流发射空列表（与之前一致）

## 边界场景
- **无确认对话框**: 清空操作直接执行，无二次确认
- **清空后快速添加**: 清空后用户可立即重新添加食材，匹配状态实时更新
- **清空不影响收藏**: 清空食材库仅移除食材，不影响收藏记录

## 偏差说明
- 清空操作为全表删除，不可撤销（但可重新添加食材）
- 数据变更通过响应式流自动传播，无需手动刷新各页面
- 清空食材库与设置页其他操作（主题切换）独立，互不影响
