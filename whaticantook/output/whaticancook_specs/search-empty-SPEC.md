# 搜索页空结果SPEC

当搜索结果为空（无任何菜谱匹配当前搜索词、分类、可做等筛选条件）且页面已完成加载时，搜索页结果区展示居中的空态提示，包含图标、标题与提示文案，不出现崩溃或长时间空白。

## 场景一

### 场景概述

搜索词无任何匹配菜谱时，结果区展示空态

### 场景逻辑步骤

- 1. 用户位于搜索页，页面已完成加载
- 2. 用户在搜索框输入一个无任何菜谱匹配的词（如"zzz-not-exist"）
- 3. 系统将输入与所有菜谱的标题、分类标签文案、菜谱标签、食材名称做包含匹配
- 4. 没有任何菜谱命中，结果列表为空
- 5. 结果区展示居中空态：圆形背景内的搜索图标、"No recipes found"标题，以及"尝试更换食材关键词或清除筛选条件"形式的提示文案
- 6. 页面不崩溃，不出现长时间空白

## 场景二

### 场景概述

分类筛选或可做筛选导致无结果时，同样展示空态

### 场景逻辑步骤

- 1. 用户位于搜索页，页面已完成加载
- 2. 用户选择某一分类，或开启"可做"筛选，或二者叠加，或再叠加搜索词
- 3. 当这些筛选条件的组合使结果集合为空时（如某分类下无菜谱，或当前食材库下无可做菜谱）
- 4. 结果列表为空
- 5. 结果区展示与场景一相同的居中空态（图标、标题、提示文案）

## 场景三

### 场景概述

放宽筛选条件使结果非空后，空态消失、列表恢复

### 场景逻辑步骤

- 1. 当前结果区处于空态（结果列表为空）
- 2. 用户放宽筛选条件（如清空搜索词、改选"All"、关闭"可做"、或切换到有菜谱的分类）
- 3. 结果集合重新变为非空
- 4. 空态立即消失，结果区恢复为菜谱卡片列表

## 场景四

### 场景概述

页面加载过程中不显示空态，避免误显

### 场景逻辑步骤

- 1. 应用首次进入搜索页、菜谱数据尚在初始化时
- 2. 此时结果列表为空，但页面处于加载中状态
- 3. 系统不显示空态（空态仅在加载完成后才可能出现）
- 4. 加载完成后，若结果仍为空才显示空态；若结果非空则直接显示列表

## 场景五

### 场景概述

"搜索页空结果"的作用范围与不作用范围

### 场景逻辑步骤

- 1. **仅作用于** 搜索页结果区的空态展示
- 2. **不提供操作按钮**：搜索页空态仅展示图标、标题与提示文案，不含可点击的操作入口（与收藏页空态不同，后者含"浏览菜谱"入口）
- 3. **不作用于** 收藏页空态——收藏页有自己的空态文案与按钮
- 4. **空态由结果集是否为空自动判定**，用户无需手动触发

> [偏差] 需求描述称"显示无结果提示或空列表状态"，实际显示居中空态（图标+标题"No recipes found"+提示文案），优于纯空列表，与需求一致。搜索页空态未提供操作按钮，需求未要求按钮，无冲突。
