# 收藏页空态SPEC

当用户没有任何收藏菜谱时，收藏页（Saved）展示居中空态，包含图标、空态标题、提示文案与一个"浏览菜谱"入口按钮；点击该按钮跳转到首页，便于用户去浏览菜谱并收藏。

## 场景一

### 场景概述

无任何收藏菜谱时，收藏页展示空态与浏览入口

### 场景逻辑步骤

- 1. 用户取消全部收藏（或从未收藏过任何菜谱），收藏集合为空
- 2. 用户点击底部导航的"Saved"入口，进入收藏页
- 3. 页面已完成加载，检测到收藏集合为空
- 4. 收藏页展示居中空态：
  - 4.1 圆形背景内的书签图标
  - 4.2 空态标题"No saved recipes yet"
  - 4.3 提示文案，引导用户点击菜谱心形按钮将其收藏到此处
  - 4.4 下方"浏览菜谱"操作按钮

## 场景二

### 场景概述

用户点击空态"浏览菜谱"按钮，跳转到首页

### 场景逻辑步骤

- 1. 用户位于收藏页空态
- 2. 用户点击"浏览菜谱"按钮
- 3. 应用跳转到首页（Discover）
- 4. 用户可在首页浏览菜谱列表，并点击菜谱卡片的心形按钮将其收藏

## 场景三

### 场景概述

新增收藏后空态消失，列表恢复

### 场景逻辑步骤

- 1. 当前收藏页处于空态（收藏集合为空）
- 2. 用户在首页、搜索页或详情页将某菜谱加入收藏
- 3. 收藏集合由空变为非空
- 4. 用户回到收藏页时，空态立即消失，收藏页恢复为收藏菜谱列表展示

## 场景四

### 场景概述

页面加载过程中不显示空态，避免误显

### 场景逻辑步骤

- 1. 应用首次进入收藏页、数据尚在加载时
- 2. 此时收藏列表为空，但页面处于加载中状态
- 3. 系统不显示空态（空态仅在加载完成后才可能出现）
- 4. 加载完成后，若收藏集合仍为空才显示空态；若非空则直接显示列表

## 场景五

### 场景概述

"收藏页空态"的作用范围与不作用范围

### 场景逻辑步骤

- 1. **仅作用于** 收藏页（Saved）的空态展示
- 2. **含浏览入口**：空态提供"浏览菜谱"按钮跳转首页，便于用户去收藏
- 3. **不作用于** 搜索页空态——搜索页有自己的空态文案，且搜索页空态不含浏览入口按钮
- 4. **空态由收藏集合是否为空自动判定**，用户无需手动触发

> [偏差] 需求描述称空态标题"No saved recipes"，实际标题为"No saved recipes yet"，语义一致；"浏览菜谱"入口跳转目标为首页（Discover），与需求"浏览入口"一致。
