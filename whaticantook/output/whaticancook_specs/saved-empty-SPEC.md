# 收藏页空态规格文档

## 需求概述
当用户尚未收藏任何菜谱时，收藏页面应显示友好的空状态提示，告知用户「还没有收藏的菜谱」，说明收藏功能的作用，并提供一个「浏览菜谱」操作按钮引导用户去发现和收藏菜谱。

## 原子场景

### 场景1: 无收藏时显示空态
- **触发条件**: 用户首次使用应用或已取消所有收藏，收藏数据列表为空，页面数据已加载完成
- **动作序列**:
  1. 用户切换到收藏页（底部导航 Saved）
- **预期行为**:
  - 页面显示居中的空状态视图
  - 显示圆形背景图标
  - 显示标题「No saved recipes yet」
  - 显示说明文案「Tap the heart on any recipe to keep it here for later.」
  - 显示主色调操作按钮「Browse recipes」
- **逻辑步骤**:
  1. 页面加载收藏数据
  2. 数据加载完成（loading=false）
  3. 收藏列表为空（items.isEmpty()）
  4. 满足条件 !loading && items.isEmpty()
  5. 渲染空状态组件（图标 + 标题 + 说明 + 操作按钮）

### 场景2: 点击 Browse recipes 跳转首页
- **触发条件**: 收藏页显示空态
- **动作序列**:
  1. 用户点击「Browse recipes」按钮
- **预期行为**:
  - 页面切换到首页（Discover）
  - 底部导航高亮 Discover
- **逻辑步骤**:
  1. 接收操作按钮点击事件
  2. 执行页面跳转到首页 Tab
  3. 跳转时保留各 Tab 状态
  4. 首页 Tab 被激活并高亮

### 场景3: 加载中不显示空态
- **触发条件**: 页面正在加载收藏数据
- **动作序列**:
  1. 用户刚切换到收藏页，数据尚未加载
- **预期行为**:
  - 不显示空态（loading=true）
  - 加载完成后根据数据决定显示空态或列表
- **逻辑步骤**:
  1. 初始状态 loading=true
  2. 空态条件 !loading && items.isEmpty() 中 loading 为 true，不触发空态
  3. 数据加载完成后 loading=false，再判断列表是否为空

### 场景4: 从空态到有收藏
- **触发条件**: 收藏页显示空态
- **动作序列**:
  1. 用户通过 Browse recipes 跳转到首页
  2. 在首页收藏某菜谱
  3. 返回收藏页
- **预期行为**:
  - 空态消失
  - 显示收藏列表
- **逻辑步骤**:
  1. 收藏操作更新收藏数据源
  2. 收藏页通过响应式数据流感知变更
  3. items 列表非空 -> 不再满足空态条件 -> 渲染列表

### 场景5: 取消最后一个收藏恢复空态
- **触发条件**: 收藏页有 1 个收藏菜谱
- **动作序列**:
  1. 用户在收藏页或详情页取消该菜谱收藏
- **预期行为**:
  - 列表中该项移除
  - 列表变空后自动切换到空态
  - 空态标题「No saved recipes yet」
- **逻辑步骤**:
  1. 取消收藏删除记录
  2. 收藏数据流发射空列表
  3. items.isEmpty() 为 true -> 渲染空态

## 边界场景
- **首次安装**: 无任何收藏记录，直接显示空态
- **数据加载失败**: loading 保持 true 直到数据加载完成；加载完成后若无收藏则显示空态
- **空态文案固定**: 不随上下文变化

## 偏差说明
- 收藏页空态有操作按钮（Browse recipes），与搜索页空态（无按钮）不同
- Browse recipes 跳转到首页 Tab，而非搜索页
- 空态为完整页面替换，非部分覆盖
