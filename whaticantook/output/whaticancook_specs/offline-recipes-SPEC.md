# 离线菜谱数据初始化SPEC

应用随安装包内置一份菜谱目录，无需登录、联网或调用远程接口即可在首页与搜索页展示菜谱；首次进入时把内置菜谱装载入本地数据库，此后直接读库。

## 场景一

### 场景概述

应用在无网络、无账号状态下进入首页，仍能展示内置菜谱

### 场景逻辑步骤

- 1. 应用处于断网或未登录任意状态（应用本身不依赖网络与账号）
- 2. 用户进入首页
- 3. 首页检查本地菜谱数据是否已装载：
  - 3.1 若尚未装载（本地菜谱数据为空），则从应用内置的菜谱目录解析全部菜谱，写入本地数据库
  - 3.2 若已装载，则跳过解析写入
- 4. 首页从本地数据库读取菜谱，展示菜谱列表，其中包含"5-Minute Mug Cake""Banana Oat Pancakes""Chicken Fried Rice"等内置菜谱

## 场景二

### 场景概述

应用在无网络、无账号状态下进入搜索页，仍能展示与检索内置菜谱

### 场景逻辑步骤

- 1. 应用处于断网或未登录任意状态
- 2. 用户进入搜索页
- 3. 搜索页从本地数据库读取已装载的菜谱数据（首页通常已先行装载）
- 4. 搜索页展示全部内置菜谱，并支持按菜谱名或食材关键词检索

## 场景三

### 场景概述

内置菜谱仅装载一次，后续进入不再重复解析写入

### 场景逻辑步骤

- 1. 用户此前已进入过首页，内置菜谱已装载入本地数据库
- 2. 用户关闭应用后再次打开，进入首页或搜索页
- 3. 应用检测到本地菜谱数据非空，跳过解析写入步骤
- 4. 首页与搜索页直接从本地数据库读取菜谱并展示

## 场景四

### 场景概述

内置菜谱数据的作用范围与不作用范围

### 场景逻辑步骤

- 1. **仅来源于** 应用随安装包分发的内置菜谱目录（一份固定的菜谱集合），不来自任何远程接口或用户上传
- 2. **不依赖** 以下任何条件即可展示：网络连接、用户登录、账号注册、远程数据拉取——应用不具备联网与账号能力
- 3. **装载入口仅有** 首页一处（首页进入时触发装载）；搜索页、详情页、收藏页均只读取已落库的菜谱数据，不触发装载
- 4. **作用于** 所有展示菜谱的页面：首页列表、搜索页列表、详情页单个菜谱、收藏页列表（均读取同一本地菜谱数据）

> [偏差] 需求称"首次进入...即可展示"，实际首页首次装载时有一段短暂的加载过渡后才展示菜谱（加载/错误态详见相关加载状态需求），菜谱本身确为离线内置、与"无需联网"一致。需求列举的三例菜谱为内置目录的示例，内置目录实际包含更多菜谱。
