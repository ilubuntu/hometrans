# Search 文本搜索 规格文档

## 需求概述
搜索页提供按关键词查找菜谱的能力。用户在搜索框输入文本后，应用以"子串包含、大小写不敏感"的方式，在菜谱的标题、分类名、标签、食材名四个维度中匹配，命中任一维度的菜谱即纳入结果。查询会自动忽略首尾空格，输入为空（或纯空格）时不过滤、展示全部菜谱。文本搜索可与分类筛选、可做筛选、排序组合使用。

## 原子场景

### 场景1: 输入搜索词实时过滤
- **触发条件**: 搜索页结果列表可见。
- **动作序列**: 用户在搜索框输入关键词。
- **预期行为**: 结果列表实时按关键词过滤，仅保留标题、分类名、标签或食材名中包含该关键词（大小写不敏感）的菜谱。
- **逻辑步骤**:
  1. 取搜索框文本，去除首尾空格并转小写。
  2. 非空时，对每个菜谱检查标题、分类名、各标签、各食材名是否包含该词。
  3. 命中任一维度即保留。
  4. 刷新结果列表。

### 场景2: 按标题命中
- **触发条件**: 关键词出现在某菜谱标题中。
- **动作序列**: 用户输入该关键词。
- **预期行为**: 标题包含该词的菜谱出现在结果中。
- **逻辑步骤**:
  1. 关键词与菜谱标题做子串匹配。
  2. 命中 → 纳入结果。
  - 示例：输入 "rice" 命中标题含 "rice" 的菜谱（如鸡肉炒饭）。

### 场景3: 按食材命中
- **触发条件**: 关键词出现在某菜谱的食材名中。
- **动作序列**: 用户输入该食材关键词。
- **预期行为**: 含该食材的菜谱出现在结果中。
- **逻辑步骤**:
  1. 关键词与菜谱每个食材名做子串匹配。
  2. 任一食材命中 → 纳入结果。
  - 示例：输入 "yogurt" 命中含酸奶食材的菜谱（如蜂蜜酸奶杯、芒果酸奶饮）。

### 场景4: 按分类或标签命中
- **触发条件**: 关键词对应某分类名或标签。
- **动作序列**: 用户输入分类或标签关键词。
- **预期行为**: 分类名或标签包含该词的菜谱出现在结果中。
- **逻辑步骤**:
  1. 关键词与菜谱分类名做子串匹配。
  2. 关键词与菜谱每个标签做子串匹配。
  3. 任一命中 → 纳入结果。

### 场景5: 空查询展示全部
- **触发条件**: 搜索框为空或仅含空格。
- **动作序列**: 用户清空搜索框。
- **预期行为**: 不按文本过滤，结果展示全部菜谱（再受分类/可做/排序影响）。
- **逻辑步骤**:
  1. 去空格后查询为空。
  2. 跳过文本过滤。
  3. 展示全部菜谱。

### 场景6: 大小写与空格容错
- **触发条件**: 用户输入含大小写差异或首尾空格的关键词。
- **动作序列**: 用户输入如 "  Rice "。
- **预期行为**: 匹配忽略大小写与首尾空格，与输入 "rice" 结果一致。
- **逻辑步骤**:
  1. 去首尾空格并转小写。
  2. 与各维度的小写形式做子串匹配。

### 场景7: 验证示例（rice / yogurt）
- **触发条件**: 搜索页默认筛选态。
- **动作序列**: 分别输入 "rice"、"yogurt"。
- **预期行为**: "rice" 命中鸡肉炒饭；"yogurt" 命中蜂蜜酸奶杯与芒果酸奶饮。
- **逻辑步骤**:
  1. "rice" → 标题命中鸡肉炒饭。
  2. "yogurt" → 食材命中两道含酸奶的菜谱。

## 边界场景
- 无匹配：结果为空，显示空态提示（见"Search 空结果"规格）。
- 子串匹配非前缀匹配：关键词出现在中间或末尾也能命中。
- 匹配域仅标题/分类名/标签/食材名，不含简介与步骤文本。
- 文本搜索与分类筛选、可做筛选、排序可叠加（按顺序组合生效）。

## 偏差说明
- 无。规格按需求描述：按标题、分类、标签、食材匹配过滤。迁移 HarmonyOS 时应实现等价的子串包含匹配（大小写不敏感、去首尾空格、四维度任一命中）。
- 文本匹配不使用模糊/语音/分词，仅简单子串包含，避免误匹配；匹配域固定为四项，不含简介与步骤。
