# 搜索页文本搜索SPEC

用户在搜索页输入框中输入文本后，结果列表即时按该文本过滤；输入内容在去除首尾空白并转为小写后，与菜谱的标题、分类标签文案、菜谱标签和食材名称进行包含匹配，任一字段命中即展示该菜谱。

## 场景一

### 场景概述

用户输入文本后，结果列表即时过滤出匹配的菜谱

### 场景逻辑步骤

- 1. 用户位于搜索页，输入框当前为空，结果列表展示全部菜谱
- 2. 用户在输入框中输入文本（如"rice"）
- 3. 系统将输入文本去除首尾空白并转为小写，得到匹配词
- 4. 系统对每一道菜谱，依次检查以下字段是否"包含"该匹配词（不区分大小写）：
  - 4.1 菜谱标题（如输入"rice"可命中"Chicken Fried Rice"）
  - 4.2 菜谱的分类标签文案（如输入"breakfast"可命中早餐分类菜谱）
  - 4.3 菜谱的标签（如输入"protein"可命中带该标签的菜谱）
  - 4.4 菜谱的食材名称（如输入"yogurt"可命中含酸奶的菜谱）
- 5. 只要上述任一字段包含匹配词，该菜谱即保留在结果列表中
- 6. 结果列表即时刷新为过滤后的菜谱，过滤在用户每次输入变化时即时生效

## 场景二

### 场景概述

输入框为空（含仅输入空白）时，结果列表展示全部菜谱，不应用文本过滤

### 场景逻辑步骤

- 1. 用户位于搜索页
- 2. 当输入框为空，或仅输入空白字符时
- 3. 系统判定匹配词为空，不应用文本过滤
- 4. 结果列表展示当前全部菜谱（仍可受分类、可做、排序影响）
- 5. 用户输入有效字符后，文本过滤才生效

## 场景三

### 场景概述

用户点击清除按钮，立即清空输入并恢复全部结果

### 场景逻辑步骤

- 1. 用户在输入框中已输入文本，结果列表处于过滤状态，输入框右侧显示清除按钮
- 2. 用户点击清除按钮
- 3. 输入框立即清空
- 4. 文本过滤取消，结果列表恢复为无文本过滤的全部菜谱

## 场景四

### 场景概述

文本搜索不区分大小写，且对输入做空白规整

### 场景逻辑步骤

- 1. 用户在输入框输入文本（如前后带空格、或大小写混合，如"  Rice "）
- 2. 系统先去除输入的首尾空白，再转为小写，得到匹配词"rice"
- 3. 系统以该匹配词与各菜谱字段做包含匹配（字段同样按小写比较）
- 4. "  Rice "与"rice"等效，均可命中标题/食材等含"rice"的菜谱

## 场景五

### 场景概述

"搜索页文本搜索"的作用范围与不作用范围

### 场景逻辑步骤

- 1. **仅作用于** 搜索页的结果列表过滤
- 2. **不作用于** 首页菜谱列表——首页列表不经搜索文本过滤
- 3. **不作用于** 收藏页列表——收藏页仅展示全部收藏，无文本搜索
- 4. 文本过滤可与分类筛选、可做筛选、排序同时叠加生效，四者相互独立组合

> [偏差] 需求描述称"输入菜谱名或食材关键词后即时过滤"，实际匹配范围更广：除菜谱标题与食材名称外，还匹配分类标签文案与菜谱标签，任一命中即保留，行为是需求的超集。
