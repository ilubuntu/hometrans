# 底部导航规格文档

## 需求概述
应用底部导航栏包含四个主入口：Discover（发现）、Search（搜索）、Pantry（食材库）、Saved（收藏）。用户可以通过底部导航在主功能页面间切换，切换时保留各页面的基本状态。菜谱详情页和设置页不显示底部导航栏。

## 原子场景

### 场景1: 底部导航栏显示与四个入口
- **触发条件**: 用户在四个主 Tab 页面之一（Discover/Search/Pantry/Saved）
- **动作序列**:
  1. 用户查看底部导航栏
- **预期行为**:
  - 底部显示浮动药丸形导航栏
  - 包含四个入口：Discover、Search、Pantry、Saved
  - 每个入口有对应图标
  - 当前所在页面的入口高亮显示（含背景色和文字标签展开）
- **逻辑步骤**:
  1. 检测当前路由是否属于四个顶级 Tab
  2. 当前路由在 Tab 列表中 -> 显示底部导航栏
  3. 当前路由对应的 Tab 项设为选中状态
  4. 选中项高亮背景 + 展开文字标签

### 场景2: 切换到不同 Tab
- **触发条件**: 用户在 Discover 页面
- **动作序列**:
  1. 用户点击底部导航栏的「Search」入口
- **预期行为**:
  - 页面切换到搜索页
  - Search 入口高亮，Discover 取消高亮
  - 底部导航栏保持显示
- **逻辑步骤**:
  1. 接收 Tab 点击事件
  2. 导航到目标 Tab 路由
  3. 弹出返回栈到首页，保存各 Tab 状态
  4. 恢复目标 Tab 之前的状态（如有）
  5. 更新高亮项

### 场景3: Tab 状态保留
- **触发条件**: 用户在搜索页输入了搜索词并选择了分类
- **动作序列**:
  1. 用户切换到 Pantry 页面
  2. 用户再切回 Search 页面
- **预期行为**:
  - 搜索页恢复之前的搜索词和分类选择
  - 搜索结果保持一致
- **逻辑步骤**:
  1. 切出 Search 时保存 Tab 状态
  2. 切入其他 Tab 时不销毁 Search 的状态
  3. 切回 Search 时恢复保存的状态
  4. 页面渲染恢复到离开时的样子

### 场景4: 进入详情页隐藏底部导航
- **触发条件**: 用户在任意 Tab 页面
- **动作序列**:
  1. 用户点击某菜谱卡片进入详情页
- **预期行为**:
  - 底部导航栏消失（带滑出动画）
  - 详情页占满全屏
  - 所有 Tab 入口不再可见
- **逻辑步骤**:
  1. 导航到详情页路由
  2. 检测当前路由不在 Tab 列表中
  3. 底部导航栏可见性设为 false
  4. 导航栏带动画滑出隐藏

### 场景5: 进入设置页隐藏底部导航
- **触发条件**: 用户在 Discover 页面
- **动作序列**:
  1. 用户点击首页右上角设置图标
- **预期行为**:
  - 底部导航栏消失
  - 设置页全屏显示
- **逻辑步骤**:
  1. 导航到设置页路由
  2. 设置路由不在 Tab 列表中
  3. 底部导航栏隐藏

### 场景6: 从详情/设置页返回后恢复底部导航
- **触发条件**: 用户在详情页或设置页
- **动作序列**:
  1. 用户点击返回按钮
- **预期行为**:
  - 返回到来源 Tab 页面
  - 底部导航栏重新出现（带滑入动画）
  - 来源 Tab 高亮
- **逻辑步骤**:
  1. 返回栈弹出
  2. 当前路由回到 Tab 列表中的路由
  3. 底部导航栏可见性设为 true
  4. 对应 Tab 高亮

### 场景7: 选中项的视觉反馈
- **触发条件**: 用户在某个 Tab 页面
- **动作序列**:
  1. 查看底部导航栏选中项
- **预期行为**:
  - 选中 Tab 有背景色高亮（药丸形）
  - 选中 Tab 的文字标签展开显示（图标 + 文字）
  - 未选中 Tab 仅显示图标
  - 颜色和展开有平滑动画过渡
- **逻辑步骤**:
  1. 选中项背景色为主题主色容器色
  2. 选中项文字标签通过展开动画显示
  3. 未选中项背景透明，仅图标
  4. 切换时颜色和展开动画同时过渡

## 边界场景
- **重复点击当前 Tab**: 不创建新页面实例，保持当前状态
- **Tab 切换动画**: 底部导航栏显隐有滑入/滑出动画
- **首次启动**: 默认显示 Discover Tab 并高亮

## 偏差说明
- 底部导航栏为自定义浮动式设计（非系统标准导航栏）
- 仅四个顶级 Tab 显示底部导航，详情页和设置页隐藏
- Tab 切换通过 saveState/restoreState 保留各页面状态
