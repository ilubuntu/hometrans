# 底部导航SPEC

应用主界面提供一条底部导航栏，包含"Discover / Search / Pantry / Saved"四个一级入口，用户可在四个主页面之间切换，当前所在入口处于高亮状态，且各主页面的浏览状态相互独立保留。

## 场景一

### 场景概述

四个一级入口的展示与顺序

### 场景逻辑步骤

- 1. 用户进入任一主页面（首页、搜索页、食材库页、收藏页）
- 2. 页面底部展示一条导航栏，按顺序包含四个入口：
  - 2.1 "Discover"（对应首页）
  - 2.2 "Search"（对应搜索页）
  - 2.3 "Pantry"（对应食材库页）
  - 2.4 "Saved"（对应收藏页）
- 3. 每个入口包含一个图标；导航栏仅在四个主页面显示，进入菜谱详情页、设置页或引导页时自动隐藏

## 场景二

### 场景概述

当前所在入口处于选中高亮状态

### 场景逻辑步骤

- 1. 用户位于某个主页面（例如"首页"）
- 2. 底部导航栏中"Discover"入口处于选中高亮状态：
  - 2.1 该入口显示高亮底色与高亮图标颜色
  - 2.2 该入口的文字"Discover"展开显示
- 3. 其余三个入口处于未选中状态：
  - 3.1 透明底色，图标颜色弱化
  - 3.2 入口文字收起不显示
- 4. 用户切换到另一主页面后，新所在页面对应的入口变为选中高亮，原入口恢复未选中状态，颜色变化与文字展开/收起带过渡效果

## 场景三

### 场景概述

点击入口切换到对应主页面

### 场景逻辑步骤

- 1. 用户点击底部导航栏中某个入口（例如"Search"）
- 2. 应用切换到该入口对应的主页面（搜索页）
- 3. 该入口立即变为选中高亮状态（见场景二）
- 4. 页面标题与内容刷新为该入口对应页面的标题与内容

## 场景四

### 场景概述

各主页面的浏览状态相互独立保留

### 场景逻辑步骤

- 1. 用户在"首页"选择了某个分类筛选，并滚动到列表某位置
- 2. 用户点击"Search"切换到搜索页，输入了搜索词
- 3. 用户点击"Discover"切回首页：首页此前所选分类、列表位置均恢复，未丢失
- 4. 用户再次点击"Search"切回搜索页：此前输入的搜索词恢复，未丢失
- 5. 四个主页面各自的浏览状态彼此独立，互不影响

## 场景五

### 场景概述

"底部导航"的作用范围与不作用范围

### 场景逻辑步骤

- 1. **仅作用于** 以下页面间切换: "首页" / "搜索页" / "食材库页" / "收藏页" 四个主页面
- 2. **仅提供** 以下入口: 底部导航栏的 Discover / Search / Pantry / Saved 四个入口
- 3. **不提供** 进入以下页面的入口: "菜谱详情页"(由菜谱卡片进入) / "设置页"(由首页右上角设置图标进入) / "引导页"(仅首次或重置后) —— 这些页面均无底部导航入口
- 4. **不作用于** 以下页面: "菜谱详情页" / "设置页" / "引导页" —— 进入这些页面时底部导航栏自动隐藏

> [偏差] 需求未明确底部导航栏在二级页面的显隐；代码在菜谱详情页、设置页、引导页自动隐藏导航栏（见场景一步骤 3）。需求未明确各主页面状态的独立保留；代码通过各页面独立保留浏览状态实现（见场景四）。均为补充说明，不与需求冲突。
> [交叉引用] 从菜谱详情页、设置页等二级页面返回上一级的行为见 `back-navigation-SPEC.md`。
