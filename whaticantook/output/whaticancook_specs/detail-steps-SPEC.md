# Recipe Detail步骤进度SPEC

菜谱详情页展示做菜步骤，用户可逐个勾选已完成步骤，并实时看到已完成计数与进度条变化；取消勾选后计数与进度恢复。

## 场景一

### 场景概述

进入菜谱详情页，展示全部做菜步骤与初始进度

### 场景逻辑步骤

- 1. 用户打开某菜谱详情页，滚动到步骤区
- 2. 步骤区标题显示"Steps"，标题右侧显示已完成计数，初始为"0/N"（N 为该菜谱总步骤数）
- 3. 标题下方显示进度条，初始填充为 0
- 4. 进度条下方按顺序逐行展示每个步骤：
  - 4.1 每个未完成步骤左侧圆圈显示该步骤的序号（从 1 开始）
  - 4.2 每个步骤右侧显示对应步骤文案，未完成时为正常样式

## 场景二

### 场景概述

用户勾选一个步骤，已完成计数与进度条即时增加

### 场景逻辑步骤

- 1. 用户点击某个未完成步骤所在行
- 2. 该步骤标记为已完成：
  - 2.1 该行左侧圆圈由序号变为"已完成"勾选标记
  - 2.2 该行步骤文案变为已完成样式（删除线、弱化颜色）
- 3. 步骤区标题右侧已完成计数由"0/N"变为"1/N"
- 4. 进度条按"已完成数/总步骤数"比例即时填充（如勾选 1 个总 5 步，进度条填充约 1/5）

## 场景三

### 场景概述

用户取消已勾选步骤，已完成计数与进度条即时恢复

### 场景逻辑步骤

- 1. 用户再次点击某个已完成的步骤所在行
- 2. 该步骤取消已完成标记：
  - 2.1 该行左侧圆圈由勾选标记恢复为原序号
  - 2.2 该行步骤文案恢复正常样式
- 3. 步骤区标题右侧已完成计数由"1/N"恢复为"0/N"
- 4. 进度条按新的"已完成数/总步骤数"比例即时回退（如恢复为 0，进度条填充为 0）

## 场景四

### 场景概述

用户可连续勾选多个步骤，计数与进度按比例累加

### 场景逻辑步骤

- 1. 用户依次勾选多个步骤
- 2. 每勾选一个，已完成计数加一（如 1/N → 2/N → 3/N），进度条按比例递增
- 3. 当全部步骤均被勾选时，已完成计数为"N/N"，进度条填满
- 4. 中途取消任一已勾选步骤，已完成计数减一，进度条按比例回退

## 场景五

### 场景概述

步骤勾选进度不跨页面保留

### 场景逻辑步骤

- 1. 用户在某菜谱详情页勾选了若干步骤，已完成计数显示为"M/N"
- 2. 用户离开该详情页（返回上一页或切换到其他菜谱详情页）
- 3. 再次进入同一菜谱详情页时，步骤勾选进度归零，已完成计数恢复为"0/N"，进度条为空
- 4. 即步骤勾选仅为当前浏览内的临时进度，不持久保存

## 场景六

### 场景概述

"Recipe Detail 步骤进度"的作用范围与不作用范围

### 场景逻辑步骤

- 1. **仅作用于**：菜谱详情页步骤区（已完成计数、进度条、每个步骤的勾选/取消交互）
- 2. **不作用于**：详情页基础信息区、食材清单区、食材匹配状态区不受步骤勾选影响；首页/搜索/收藏页卡片不展示步骤进度
- 3. **不作用于**：步骤勾选不影响食材匹配、可做状态或收藏状态

> [偏差] 需求未说明步骤进度是否持久化；实际实现中步骤勾选进度仅保留在当前浏览内，离开详情页即丢失，不跨次保留（参见场景五）。
