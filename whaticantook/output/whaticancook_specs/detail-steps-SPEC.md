# Recipe Detail 步骤进度 规格文档

## 需求概述
菜谱详情页底部展示"烹饪步骤"列表，帮助用户按步骤完成料理。每个步骤可点击切换完成状态，顶部显示完成进度（已完成数 / 总步骤数）并配进度条。点击步骤即在已完成/未完成之间切换，进度计数与进度条随之增减，并提供删除线、勾选等视觉反馈，让用户清晰跟踪烹饪进度。

## 原子场景

### 场景1: 步骤列表与初始进度
- **触发条件**: 用户进入菜谱详情页并滚动到步骤区。
- **动作序列**: 用户查看步骤区。
- **预期行为**: 步骤区标题为"步骤"，标题右侧显示完成进度"0/总步骤数"；下方进度条为空；每个步骤以序号开头展示步骤文本，均为未完成状态。
- **逻辑步骤**:
  1. 取菜谱步骤列表。
  2. 标题"步骤" + 右侧"已完成/总数"（初始 0/N）。
  3. 渲染空进度条。
  4. 逐条渲染步骤行：序号指示器 + 步骤文本。

### 场景2: 点击步骤标记为已完成
- **触发条件**: 步骤区存在未完成步骤。
- **动作序列**: 用户点击某未完成步骤行。
- **预期行为**: 该步骤切换为已完成状态——指示器变为勾选，步骤文本添加删除线并淡化；完成进度计数 +1，进度条相应增长。
- **逻辑步骤**:
  1. 点击触发该步骤索引加入完成集合。
  2. 指示器切换为勾选样式，文本加删除线。
  3. 完成进度计数 +1，进度条动画增长。

### 场景3: 再次点击取消完成
- **触发条件**: 某步骤已处于已完成状态。
- **动作序列**: 用户再次点击该步骤行。
- **预期行为**: 该步骤切换回未完成——勾选变回序号，删除线移除；完成进度计数 -1，进度条相应回退。
- **逻辑步骤**:
  1. 点击触发该步骤索引移出完成集合。
  2. 指示器恢复序号样式，文本恢复正常。
  3. 完成进度计数 -1，进度条动画回退。

### 场景4: 全部完成
- **触发条件**: 用户依次点击完成所有步骤。
- **动作序列**: 完成最后一个步骤。
- **预期行为**: 完成进度达到"总步骤数/总步骤数"，进度条填满；所有步骤行均为勾选 + 删除线状态。
- **逻辑步骤**:
  1. 完成集合大小等于步骤总数。
  2. 进度计数显示 N/N，进度条填满。

### 场景5: 进度条与计数动画
- **触发条件**: 步骤完成态发生变化。
- **动作序列**: 完成集合增减。
- **预期行为**: 进度计数文字即时更新，进度条以平滑动画过渡到新的填充比例。
- **逻辑步骤**:
  1. 计算新完成比例 = 完成数 / 总步骤数。
  2. 进度计数文字即时刷新。
  3. 进度条以动画过渡到新比例。

### 场景6: 验证示例（Chicken Fried Rice 0/5 ↔ 1/5）
- **触发条件**: 进入 Chicken Fried Rice 详情页步骤区（共 5 步）。
- **动作序列**: 点击第一步。
- **预期行为**: 进度由"0/5"变为"1/5"，第一步变为勾选 + 删除线；再次点击恢复"0/5"。
- **逻辑步骤**:
  1. 初始完成集合为空 → "0/5"。
  2. 点击第一步 → 索引 0 加入集合 → "1/5"。
  3. 再次点击 → 索引 0 移出集合 → "0/5"。

## 边界场景
- 菜谱无步骤（步骤列表为空）：进度为 0，进度条为空，不渲染步骤行。
- 步骤可按任意顺序点击完成/取消，不强制顺序。
- 完成状态为会话内内存状态：离开详情页或重建后进度重置为 0/N（不持久化）。
- 进度计数与进度条同源（完成集合大小 / 步骤总数）。

## 偏差说明
- 完成状态不持久化：步骤完成态仅在当前详情页会话内有效，离开页面或重建后归零。需求未要求持久化，规格按"会话内可切换"描述；迁移 HarmonyOS 时若需保留进度可额外持久化，但默认行为应与会话内一致。
- 无其它偏差。迁移时应实现等价的步骤列表、点击切换、进度计数与进度条动画。
