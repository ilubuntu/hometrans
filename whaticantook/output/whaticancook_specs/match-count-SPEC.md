# 菜谱匹配计数 规格文档

## 需求概述
应用根据用户食材库中已有的食材与每个菜谱的"必需食材"清单进行比对，计算出"已拥有数量 / 必需数量"。该匹配计数以紧凑的计数形式（如 3/7、0/4）或语义化的状态文案呈现在菜谱卡片、可做菜谱列表以及菜谱详情页，帮助用户直观判断"凭手头食材能做哪些菜、还差多少"。匹配关系会随食材库的增删实时刷新。

## 原子场景

### 场景1: 匹配计数计算
- **触发条件**: 任何需要展示菜谱匹配情况的页面渲染前（首页、详情页、搜索/排序前）。
- **动作序列**: 系统拿当前食材库的全部食材名，与某菜谱的食材清单逐项比对。
- **预期行为**: 对菜谱每个食材判定"是否被食材库中某项满足"；其中"必需"食材被满足的累加为"已拥有数量"，所有"必需"食材的总数为"必需数量"，未满足的必需食材归入"缺少"集合，未满足的可选食材归入"缺少可选"集合。
- **逻辑步骤**:
  1. 取当前食材库全部食材名集合。
  2. 对菜谱每个食材，用食材库集合判定是否被满足（匹配规则见"食材归一化与同义词匹配"规格）。
  3. 必需且被满足 → 计入已拥有；必需且未被满足 → 计入缺少；可选且未被满足 → 计入缺少可选。
  4. 汇总得到"已拥有数量 / 必需数量"。

### 场景2: 计数随三档状态映射为不同文案
- **触发条件**: 得到匹配结果后，需要把计数映射为展示文案。
- **动作序列**: 系统根据缺少数量决定展示档位。
- **预期行为**:
  - 必需食材全部满足（或菜谱无必需食材）→ 展示"可做"语义文案。
  - 缺少 1–2 个必需食材 → 展示"还差 N 个"语义文案。
  - 缺少超过 2 个必需食材 → 展示"已拥有/必需"计数（如 3/7、0/4）。
- **逻辑步骤**:
  1. 必需食材数为 0 或缺少集合为空 → 可做态文案。
  2. 缺少集合大小 ≤ 2 → 接近可做态文案（含缺少数量）。
  3. 否则 → 探索态，显示"已拥有/必需"计数。

### 场景3: 菜谱卡片展示匹配计数
- **触发条件**: 首页菜谱列表或可做菜谱横向列表渲染卡片。
- **动作序列**: 用户查看菜谱卡片。
- **预期行为**: 卡片视觉区角标以胶囊形式展示匹配状态（可做态/接近态/探索态计数三选一），与卡片其它信息共存。
- **逻辑步骤**:
  1. 取该菜谱匹配结果。
  2. 按场景2映射为对应文案与配色。
  3. 渲染为状态胶囊。

### 场景4: 详情页展示匹配计数
- **触发条件**: 用户进入菜谱详情页。
- **动作序列**: 用户查看详情页顶部状态区。
- **预期行为**: 详情页顶部状态区以同一套状态映射展示匹配状态（非全满足时显示计数胶囊，并附"还差 N 个食材"说明）；全满足时显示完成态提示（见"可做状态 Ready"规格）。
- **逻辑步骤**:
  1. 详情页加载时用食材库与该菜谱比对得到匹配结果。
  2. 非全满足 → 渲染计数胶囊 + 缺少数量说明。
  3. 全满足 → 渲染完成态提示。

### 场景5: 食材库变化实时刷新计数
- **触发条件**: 用户在食材库页或设置页增删/清空食材。
- **动作序列**: 食材库内容变化。
- **预期行为**: 首页卡片、可做列表、详情页的匹配计数与状态文案立即同步刷新，无需手动重载。
- **逻辑步骤**:
  1. 食材库变更触发数据流下发新食材集合。
  2. 各页面重新比对计算匹配结果。
  3. 受影响的计数与文案重新渲染。

### 场景6: 验证示例（Chicken Fried Rice 显示 3/7）
- **触发条件**: 食材库含 carrot、green onion、garlic，查看 Chicken Fried Rice。
- **动作序列**: 查看该菜谱卡片或详情页。
- **预期行为**: 该菜谱必需食材 7 个（米饭、鸡肉、鸡蛋、胡萝卜、葱、酱油、蒜），已拥有 3 个 → 探索态，计数显示 "3/7"。
- **逻辑步骤**:
  1. 必需食材总数 7，食材库满足其中 3 个 → 已拥有数量 3。
  2. 缺少 4 个（>2）→ 探索态。
  3. 展示 "3/7"。

## 边界场景
- 菜谱无任何必需食材（仅含可选食材）：必需数为 0，直接判为可做态，比例视为 1。
- 食材库为空：所有菜谱必需数不变，已拥有数量为 0，探索态显示 "0/N"。
- 匹配计数仅是展示的一部分；"可做/接近/探索"三态文案同时决定颜色与图标，迁移时需一并实现等价映射。
- 匹配比例（已拥有/必需）另作排序用途（见"搜索 Best match 排序"规格），与展示计数同源但用途不同。

## 偏差说明
- 无。规格按需求描述：以"已拥有数量/必需数量"形式在卡片和详情页展示（如 3/7、0/4），并随食材库实时刷新。迁移 HarmonyOS 时应实现等价的比对算法与三档状态映射，匹配规则须与"食材归一化与同义词匹配"规格保持一致。
- 本规格与"食材归一化与同义词匹配（ingredient-matching-SPEC.md）"配套：后者定义单食材如何判定为满足，本规格定义汇总后的计数与展示。
