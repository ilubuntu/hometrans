# 搜索页Quickest排序SPEC

搜索页排序项"Quickest"按菜谱的制作时间从短到长对结果列表排序（升序），制作时间最短的菜谱排在最前。

## 场景一

### 场景概述

用户选择"Quickest"，结果列表按制作时间从短到长排序

### 场景逻辑步骤

- 1. 用户位于搜索页排序条
- 2. 用户点击"Quickest"项
- 3. "Quickest"项进入选中态，其余排序项退出选中态
- 4. 系统读取每道菜谱的制作时间（以分钟计）
- 5. 结果列表按制作时间从短到长重新排列，制作时间最短的菜谱排在最前（如"5-Minute Mug Cake"等短时间菜谱排在前面）

## 场景二

### 场景概述

"Quickest"排序与其他筛选叠加，且只对已过滤结果重排

### 场景逻辑步骤

- 1. 用户在搜索页可能已输入搜索词、选定分类、开启"Cookable"
- 2. 系统先按搜索词、分类、可做筛选过滤出结果集合
- 3. 再对过滤后的结果集合按制作时间从短到长重排
- 4. 排序只改变结果集合的顺序，不改变其成员（即不会因排序增减菜谱）

## 场景三

### 场景概述

切换排序项后，列表立即按新排序刷新

### 场景逻辑步骤

- 1. 当前生效排序为"Quickest"，结果列表按制作时间升序排列
- 2. 用户点击另一个排序项（如"Best match"）
- 3. "Quickest"项退出选中态，新点击项进入选中态
- 4. 结果列表立即按新排序项重新排列

## 场景四

### 场景概述

"Quickest"排序的作用范围与不作用范围

### 场景逻辑步骤

- 1. **仅作用于** 搜索页结果列表的排序
- 2. **不作用于** 首页菜谱列表——首页列表有独立的排序，不受搜索页"Quickest"影响
- 3. **可与文本搜索、分类筛选、可做筛选叠加**：排序作用于已过滤结果，四者可任意组合
- 4. **不改变菜谱的制作时间数据**：排序只读取制作时间用于排序，不修改菜谱本身

> [偏差] 需求描述未明确制作时间相同时的次序，实际相同制作时间的菜谱保持其在已过滤结果中的相对顺序（稳定排序），为合理的实现细节。
