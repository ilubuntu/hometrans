# 食材归一化与同义词匹配 规格文档

## 需求概述
食材匹配逻辑用于判断用户食材库中的食材是否满足某菜谱的食材需求。匹配应"宽容"：忽略大小写与标点、统一空格，支持英文单复数、常见同义词/别称（如 scallions 与 green onion、chicken breast 与 chicken、garlic cloves 与 garlic），以及整词包含关系（如 green onion 满足 onion）。这样即使用户输入形式与菜谱食材表述不完全一致，也能正确判定为"已拥有"。

## 原子场景

### 场景1: 食材名称归一化
- **触发条件**: 添加食材或比对食材前。
- **动作序列**: 系统对食材名称执行归一化。
- **预期行为**: 名称转为小写、去除标点、压缩多余空格，再查同义词表映射到规范名（命中则替换，否则保留清理后名称）。
- **逻辑步骤**:
  1. 小写化并裁剪首尾空格。
  2. 将非字母数字字符替换为空格，压缩连续空格。
  3. 在同义词表中查找，命中则取规范名，否则使用清理后名称。

### 场景2: 单复数匹配
- **触发条件**: 食材名存在单复数差异。
- **动作序列**: 比对食材库项与菜谱食材。
- **预期行为**: 复数形式归一化后映射到单数规范名，与单数菜谱食材判定为匹配。
- **逻辑步骤**:
  1. 复数名经同义词表归一为单数规范名。
  2. 与菜谱食材规范名相等 → 匹配。
  - 示例：食材库 "eggs" → "egg"，匹配菜谱食材 "egg"。

### 场景3: 同义词/别称匹配
- **触发条件**: 食材存在多种叫法。
- **动作序列**: 比对食材库项与菜谱食材。
- **预期行为**: 不同叫法归一化后映射到同一规范名，判定为匹配。
- **逻辑步骤**:
  1. 别称经同义词表归一为规范名。
  2. 与菜谱食材规范名相等 → 匹配。
  - 示例："scallions" → "green onion" 匹配菜谱食材 "green onion"；"chicken breast" → "chicken" 匹配 "chicken"；"garlic cloves" → "garlic" 匹配 "garlic"。

### 场景4: 整词包含关系匹配
- **触发条件**: 一方食材名是另一方食材名的整词超集（且不在同义词表）。
- **动作序列**: 比对食材库项与菜谱食材。
- **预期行为**: 若一方词集是另一方词集的子集，则判定为匹配。
- **逻辑步骤**:
  1. 将归一化后的名称按空格拆分为词集。
  2. 比较两个词集，任一方包含另一方的全部词 → 匹配。
  - 示例：食材库 "green onion" 词集 {green,onion} 包含菜谱食材 "onion" 词集 {onion} → 匹配。

### 场景5: 不匹配的情形
- **触发条件**: 两个名称归一化后既不相等也无整词包含关系。
- **动作序列**: 比对食材库项与菜谱食材。
- **预期行为**: 判定为不匹配（即该菜谱食材缺失）。
- **逻辑步骤**:
  1. 归一化后不相等且词集互不包含 → 不匹配。
  - 说明：不做部分子串匹配，避免如 "chi" 匹配 "chicken" 的误判。

### 场景6: 验证示例（手动添加 chicken breast）
- **触发条件**: 食材库为空，手动添加 "chicken breast"。
- **动作序列**: 添加后查看 Chicken Fried Rice 匹配状态。
- **预期行为**: "chicken breast" 归一化为 "chicken"，菜谱 Chicken Fried Rice 的必需食材 Chicken 被判定为已拥有。
- **逻辑步骤**:
  1. "chicken breast" 经同义词表归一为 "chicken"。
  2. 菜谱食材 "chicken" 归一为 "chicken"。
  3. 二者相等 → chicken 判定已拥有，匹配计数 +1。

## 边界场景
- 空名称：归一化后为空则不参与匹配（视为不匹配）。
- 名称含标点：归一化去除标点后参与匹配。
- 同义词表未覆盖的变体：退化为整词包含关系判定；若仍无包含则不匹配。
- 匹配为整词级别，非子串级别，避免误匹配。

## 偏差说明
- 匹配算法为大小写不敏感 + 去标点 + 压缩空格 + 同义词表归一 + 整词双向子集包含。规格按此逻辑描述，迁移 HarmonyOS 时应实现等价的归一化函数与同义词表。
- 同义词表为代码内固定映射（单复数、常见别称如 scallions→green onion、chicken breast→chicken、garlic cloves→garlic 等），迁移时应保留同样的映射集合。
- 规范名与菜谱数据、快速添加目录使用的名称一致，故精确匹配占主导，匹配可靠。
