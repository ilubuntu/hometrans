# 可做状态ReadySPEC

当食材库已覆盖某菜谱的全部必需食材时，菜谱详情页顶部展示"准备就绪"的可做状态，相关卡片状态胶囊同步显示"Ready to cook"。

## 场景一

### 场景概述

当某菜谱的全部必需食材均已存在于食材库时，详情页展示"准备就绪"可做状态

### 场景逻辑步骤

- 1. 用户在食材库中已具备某菜谱的全部必需食材（可分次手动添加，或通过"一键补齐缺少食材"补全）
- 2. 系统将该菜谱每个必需食材与食材库逐一比对，结果为"缺少数为 0"
- 3. 详情页顶部状态区显示"准备就绪"提示框，包含：
  - 3.1 一个庆祝图标
  - 3.2 主文案"准备就绪（You're all set!）"
  - 3.3 副文案"你已具备做这道菜的全部必需食材（You have everything to make this.）"
- 4. 此时详情页不再显示"缺少 N 个食材"列表，也不显示"一键补齐"按钮

## 场景二

### 场景概述

菜谱无任何必需食材时，同样视为可做状态

### 场景逻辑步骤

- 1. 某菜谱食材清单中全部食材均标记为"可选"，即没有必需食材
- 2. 系统计算该菜谱"必需食材总数为 0"
- 3. 无论食材库当前是否含有这些可选食材，该菜谱均判定为可做状态
- 4. 详情页顶部同样显示"准备就绪"提示框

## 场景三

### 场景概述

可做状态下，相关菜谱卡片状态胶囊同步显示"Ready to cook"

### 场景逻辑步骤

- 1. 某菜谱判定为可做状态
- 2. 首页菜谱列表卡片左上角状态胶囊显示"Ready to cook"，并配以成功态配色与勾选图标
- 3. 搜索页、收藏页等复用同一卡片的页面，该菜谱卡片状态胶囊同样显示"Ready to cook"
- 4. 该可做饭菜同时进入首页"Ready to cook"横向推荐区

## 场景四

### 场景概述

食材库变化导致可做状态在"缺少"与"可做"之间切换，详情页与卡片实时刷新

### 场景逻辑步骤

- 1. 当前某菜谱处于可做状态，详情页显示"准备就绪"提示框
- 2. 用户从食材库移除该菜谱的某一必需食材
- 3. 系统重新比对，"缺少数"由 0 变为 1
- 4. 详情页提示框消失，改为显示"缺少 1 个食材"及对应缺少食材列表与"一键补齐"按钮；卡片胶囊改显示对应缺少状态
- 5. 用户再把该必需食材加回食材库，缺少数回到 0，详情页再次回到"准备就绪"，卡片同步回到"Ready to cook"

## 场景五

### 场景概述

"可做状态"的作用范围与不作用范围

### 场景逻辑步骤

- 1. **仅作用于** 以下界面的可做状态展示：菜谱详情页顶部状态区（"准备就绪"提示框）；首页/搜索/收藏页菜谱卡片状态胶囊（"Ready to cook"）；首页"Ready to cook"横滑区
- 2. **不作用于** 以下内容：详情页做菜步骤区、详情页食材清单区不单独再展示可做状态；可选食材是否满足不改变可做状态（参见可选食材需求）

> [说明] 需求验收文案"You're all set! You have everything to make this."与代码实现一致。
