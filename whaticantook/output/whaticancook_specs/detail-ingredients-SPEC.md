# Recipe Detail食材清单SPEC

菜谱详情页展示该菜谱的全部食材清单，包括食材总数、每个食材的名称（含数量与单位）、已满足状态、缺少状态与可选标记，并随食材库变化实时刷新。

## 场景一

### 场景概述

进入菜谱详情页，展示食材清单总数与每个食材的名称及数量

### 场景逻辑步骤

- 1. 用户打开某菜谱详情页（例如 Chicken Fried Rice）
- 2. 滚动到食材区，标题显示"Ingredients"，标题右侧显示该菜谱的食材总数（含必需与可选全部食材）
- 3. 食材总数下方逐行展示每个食材：
  - 3.1 每行中间展示该食材的"数量 + 单位 + 名称"（例如"3 cups  Rice"），名称首字母大写
  - 3.2 食材顺序与该菜谱食材清单的原顺序一致

## 场景二

### 场景概述

已拥有的必需食材显示已满足（选中）状态

### 场景逻辑步骤

- 1. 系统将食材清单中每个食材与当前食材库逐一比对（支持大小写、复数、同义词归一）
- 2. 当某"必需"食材已存在于食材库时：
  - 2.1 该行左侧圆圈显示"已满足"勾选标记
  - 2.2 该行右侧不显示任何状态标记（既不显示 Missing 也不显示 Optional）

## 场景三

### 场景概述

缺少的必需食材显示 Missing 状态

### 场景逻辑步骤

- 1. 当某"必需"食材未存在于食材库时：
  - 1.1 该行左侧圆圈不显示勾选标记
  - 1.2 该行右侧显示"Missing"标记

## 场景四

### 场景概述

可选食材显示 Optional 标记，且该标记不因其是否已拥有而改变

### 场景逻辑步骤

- 1. 当某食材标记为"可选"时：
  - 1.1 该行右侧恒显示"Optional"标记，无论该可选食材是否已在食材库
  - 1.2 若该可选食材已在食材库，该行左侧圆圈额外显示"已满足"勾选标记
  - 1.3 若该可选食材不在食材库，该行左侧圆圈不显示勾选，但仍显示"Optional"标记（不显示 Missing）

## 场景五

### 场景概述

食材库变化时，食材清单的已满足/缺少标记实时刷新

### 场景逻辑步骤

- 1. 当前食材清单各食材显示某一组已满足/缺少标记
- 2. 用户在食材库中增加一个原本缺少的食材
- 3. 系统重新比对，该食材由"缺少（Missing）"变为"已满足（勾选）"，右侧 Missing 标记消失
- 4. 用户移除一个原本已拥有的食材，该食材由"已满足"变回"缺少"，右侧重新显示 Missing
- 5. 可选食材的 Optional 标记在以上变化中保持不变

## 场景六

### 场景概述

"Recipe Detail 食材清单"的作用范围与不作用范围

### 场景逻辑步骤

- 1. **仅作用于**：菜谱详情页食材区（食材总数标题、每个食材行的名称/数量、已满足/缺少/可选三态标记）
- 2. **不作用于**：详情页基础信息区、做菜步骤区不重复展示食材清单；首页/搜索/收藏页卡片不展示食材明细，仅展示匹配计数
- 3. **说明**：食材清单显示的食材总数为"必需 + 可选"全部食材数，与卡片匹配计数分母（仅必需食材数）不同

> [说明] 需求验收"已有食材显示选中状态，缺少食材显示 Missing，可选食材显示 Optional"与代码实现一致；食材行另展示"数量 + 单位 + 名称"，食材总数含可选食材。
