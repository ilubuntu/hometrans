# 食材归一化与同义词匹配SPEC

食材在写入食材库与菜谱食材匹配时，均先归一化为统一规范名再比较，支持大小写、首尾空白、复数与常见同义词的差异；匹配为整词级，避免因子串或文本差异导致可做状态错误。

## 场景一

### 场景概述

食材匹配对大小写与首尾空白不敏感

### 场景逻辑步骤

- 1. 用户手动添加食材，输入带首尾空白或大小写不同的名称（如"  Egg  "或"OLIVE OIL"）
- 2. 应用将名称转为小写、去除首尾与非字母数字字符、压缩多余空白，得到规范名（"egg"、"olive oil"）后写入食材库
- 3. 用户打开含对应食材的菜谱详情
- 4. 菜谱食材名（如"egg"、"olive oil"）经同样的归一化后，与食材库中的规范名比较
- 5. 二者规范名相同即视为该食材已满足，不受原始大小写或空白影响

## 场景二

### 场景概述

食材的复数形式与单数互相匹配

### 场景逻辑步骤

- 1. 用户手动添加复数形式的食材（如"eggs"、"tomatoes"）
- 2. 应用将常见复数归一为单数规范名（"eggs"→"egg"、"tomatoes"→"tomato"）后写入食材库
- 3. 用户打开含对应单数食材的菜谱详情（如含"Egg"的菜谱）
- 4. 菜谱食材"egg"与食材库"egg"规范名相同，视为已满足
- 5. 反向同样成立：食材库中存单数"onion"、菜谱写复数"onions"时也视为匹配

## 场景三

### 场景概述

常见同义词/别称归一为同一食材

### 场景逻辑步骤

- 1. 用户手动添加某食材的同义词或别称（如"scallion"、"spring onion"、"chicken breast"、"ground beef"）
- 2. 应用按同义词表将其归一为规范名（"scallion"/"spring onion"→"green onion"、"chicken breast"→"chicken"、"ground beef"→"beef"）后写入食材库
- 3. 用户打开含对应规范食材的菜谱详情（如含"Green onion"的菜谱）
- 4. 菜谱食材"green onion"与食材库归一化后的"green onion"规范名相同，视为已满足
- 5. 同一食材的多种叫法（葱、青葱、spring onion）因此不会因文本差异而漏匹配

## 场景四

### 场景概述

整词包含匹配更具体的名称，但非整词的子串不匹配

### 场景逻辑步骤

- 1. 食材库中已有较通用的食材（如"rice"），菜谱需要更具体的同名扩展食材（如"basmati rice"）
- 2. 应用按整词集合包含判定：通用名的每个词都出现在具体名中即视为满足（"rice"满足"basmati rice"）
- 3. 反向同样成立（食材库"chicken thigh"满足菜谱"chicken"）
- 4. 但当二者仅是子串关系而非整词关系时不匹配：食材库"egg"不满足菜谱"eggplant"，"water"不满足"watercress"
- 5. 该整词规则避免短名误触发长名，保证可做状态准确

## 场景五

### 场景概述

写入食材库时即归一化，保证同一食材不重复入库

### 场景逻辑步骤

- 1. 用户手动或快速添加食材
- 2. 应用在写入前对食材名归一化（小写、去标点、同义词映射），以规范名作为食材库主键
- 3. 若食材库已存在相同规范名的食材，新添加覆盖原记录而非新增重复条目（如已存"egg"，再添加"eggs"不会产生第二条）
- 4. 快速添加建议区据此过滤：已添加食材的规范名不再出现在建议中
- 5. 删除时同样按归一化名定位，确保删除的是规范名对应的记录

## 场景六

### 场景概述

食材归一化与匹配的作用范围与不作用范围

### 场景逻辑步骤

- 1. **作为底层共享行为作用于** 所有涉及食材比对的位置：写入（手动添加、快速添加）、删除（单个删除、清空）、匹配（首页可做区与卡片状态、详情页可做/缺失、搜索页匹配与可做筛选及排序）
- 2. **不独立展示** 归一化与匹配没有独立界面，其效果通过各页面食材匹配状态的可做/缺失/匹配计数体现
- 3. **不改变展示文本的归一化语义** 已添加区显示的食材名仅做首字母大写展示处理，底层存储与匹配始终使用归一化规范名
- 4. **同义词集合固定** 为应用内置的同义词表，用户不能新增或编辑同义词映射

> [偏差] 需求称支持"大小写、复数和常见同义词"，实际另支持"整词包含"匹配更具体名称（如 rice 匹配 basmati rice），并对非整词子串明确不匹配（egg 不匹配 eggplant），以避免误判（见场景四）。需求验收"eggs 匹配 Egg""scallion 匹配 Green onion"均与代码一致（见场景二、场景三）。
