# 返回导航SPEC

菜谱详情页、设置页等二级页面在顶部提供返回按钮，点击后返回进入该页面之前的上一级页面；返回后尽量保持上一页已有的数据状态。

## 场景一

### 场景概述

从首页打开菜谱详情页后，点击返回回到首页

### 场景逻辑步骤

- 1. 用户在"首页"点击某菜谱卡片，进入该菜谱的"详情页"
- 2. 详情页顶部左侧显示一个返回按钮
- 3. 用户点击该返回按钮
- 4. 应用返回"首页"（即进入详情页之前所在的页面）

## 场景二

### 场景概述

从首页打开设置页后，点击返回回到首页

### 场景逻辑步骤

- 1. 用户在"首页"右上角点击设置图标，进入"设置页"
- 2. 设置页顶部显示返回按钮与标题"Settings"
- 3. 用户点击该返回按钮
- 4. 应用返回"首页"（即进入设置页之前所在的页面）

## 场景三

### 场景概述

返回后保持上一页的数据状态

### 场景逻辑步骤

- 1. 用户在"首页"选择了某个分类筛选（例如"Dinner"），或在"搜索页"输入了某个搜索词
- 2. 用户从该页面进入"详情页"或"设置页"
- 3. 用户点击返回按钮回到原页面
- 4. 原页面此前所选的分类、所输入的搜索词等数据状态保持不变，未丢失
- 5. 列表内容按保留的数据状态（如所选分类）继续呈现；列表滚动位置尽量保持，但不保证精确还原到离开时的位置

## 场景四

### 场景概述

通过系统返回操作回到上一级

### 场景逻辑步骤

- 1. 用户处于"详情页"或"设置页"等二级页面
- 2. 用户使用系统返回手势或系统返回键
- 3. 应用执行与点击页面内返回按钮等效的返回行为，回到进入该页面之前的上一级页面
- 4. 返回后上一页的数据状态保持不变（同场景三）

## 场景五

### 场景概述

"返回导航"的作用范围与不作用范围

### 场景逻辑步骤

- 1. **仅作用于** 以下二级页面: "菜谱详情页"(左上角返回按钮) / "设置页"(顶部返回按钮)
- 2. **返回目标**: 进入该二级页面之前的上一级页面（通常为首页、搜索页或收藏页）
- 3. **不作用于** 以下页面: "首页" / "搜索页" / "食材库页" / "收藏页" 四个主页面 —— 这些页面通过底部导航相互切换，本身不提供"返回上一级"的页面内返回按钮（其顶部标题栏无返回箭头）

> [偏差] 需求要求"列表位置尽量不丢失"；代码保留了分类、搜索词等数据状态，但列表精确滚动位置未以可持久化方式保存，可能回到顶部（见场景三步骤 5）。这是需求"尽量"预期与实现间的潜在差异。需求未提及系统返回；代码支持系统返回手势/返回键等效返回（见场景四）。均为补充说明。
> [交叉引用] 一级主页面之间的切换与状态保留见 `bottom-nav-SPEC.md`。
