# 离线菜谱数据初始化 规格文档

## 需求概述
应用随安装包内置一份完整的菜谱数据文件（共 18 道菜谱，覆盖早餐、午餐、晚餐、甜点、小吃、饮品等分类）。应用首次进入主界面时，将该内置数据初始化写入本地数据存储；此后所有菜谱浏览、搜索、详情展示均基于本地数据完成，不依赖任何网络请求，实现完全离线可用。

## 原子场景

### 场景1: 首次进入主界面初始化本地菜谱库
- **触发条件**: 应用已完成引导（或跳过引导）首次进入主界面，且本地菜谱库为空。
- **动作序列**: 用户进入主首页。
- **预期行为**: 应用检测到本地菜谱库为空，从内置菜谱数据文件读取 18 道菜谱并整批写入本地数据存储；初始化期间首页显示加载态。
- **逻辑步骤**:
  1. 首页加载时启动"数据初始化"流程，进入加载态。
  2. 加锁防止并发重复初始化。
  3. 查询本地菜谱库数量；若为 0 则继续，否则跳过初始化。
  4. 从内置菜谱数据文件（随包发布的静态文件）读取全部菜谱数据并解析。
  5. 将解析后的菜谱整批写入本地数据存储（重复写入时以最新覆盖）。
  6. 初始化成功后切到内容态，加载本地菜谱并渲染首页。

### 场景2: 离线浏览菜谱
- **触发条件**: 本地菜谱库已初始化（非首次），设备处于无网络状态。
- **动作序列**: 用户在无网络下启动应用并浏览首页/搜索/详情。
- **预期行为**: 菜谱数据全部来自本地存储，正常展示 18 道菜谱，不因无网络而失败或空白。
- **逻辑步骤**:
  1. 查询本地菜谱库数量，大于 0，跳过初始化。
  2. 直接从本地存储读取菜谱列表并渲染。
  3. 全程无网络调用。

### 场景3: 初始化失败可重试
- **触发条件**: 内置数据读取或写入过程发生异常。
- **动作序列**: 初始化流程捕获到错误。
- **预期行为**: 首页进入错误态，提供重试入口；用户触发重试后重新执行初始化流程。
- **逻辑步骤**:
  1. 初始化流程用容错方式执行，捕获异常。
  2. 异常时切到错误态（与加载态区分）。
  3. 用户点击重试 → 重新进入加载态并执行场景1 步骤 2-6。

### 场景4: 内置菜谱内容完整性
- **触发条件**: 初始化完成。
- **动作序列**: 校验本地菜谱库内容。
- **预期行为**: 至少包含以下固定菜谱（及其分类）：Chicken Fried Rice（晚餐）、5-Minute Mug Cake（甜点）、Banana Oat Pancakes（早餐）等，共 18 道覆盖多分类的菜谱。
- **逻辑步骤**:
  1. 初始化写入全部 18 道菜谱。
  2. 本地库可按分类、标题等检索到上述菜谱。

## 边界场景
- 已初始化过的设备再次进入首页：跳过写入，直接读取，不重复初始化。
- 并发触发初始化（如快速重建首页）：加锁保证只写入一次。
- 清除应用数据后：本地菜谱库被清空，下次进入首页重新触发场景1 初始化。
- 内置数据文件损坏/缺失：进入错误态（场景3），不崩溃。

## 偏差说明
- 技术实现为 Android 资产文件（recipes.json）+ Room 数据库 + kotlinx.serialization 解析。规格按"随包发布的静态数据文件 + 本地数据存储"表述，迁移 HarmonyOS 时可用 rawfile 资源 + 关系型数据库等价实现，不绑定具体框架。
- 初始化代码含 550ms 人为延迟，仅为让加载态可感知，属体验性设计，非功能必需；规格中表述为"初始化过程显示加载态"，不规定具体延迟时长。
- 菜谱分类在数据中为枚举（早餐/午餐/晚餐/甜点/小吃/饮品），与首页分类筛选（REQ-007）对应。
