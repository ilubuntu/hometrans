# 收藏排序规格文档

## 需求概述
收藏页面应按收藏时间倒序展示菜谱，最近收藏的菜谱排在列表最前面。当用户先后收藏多个菜谱时，最后收藏的菜谱始终出现在列表顶部。重新收藏（取消后再收藏）的菜谱会更新其收藏时间并移至顶部。

## 原子场景

### 场景1: 按收藏时间倒序排列
- **触发条件**: 用户已收藏多个菜谱
- **动作序列**:
  1. 用户先收藏菜谱 A
  2. 用户再收藏菜谱 B
  3. 用户切换到收藏页
- **预期行为**:
  - 列表第一项为菜谱 B（最近收藏）
  - 列表第二项为菜谱 A（较早收藏）
  - 排列顺序为收藏时间从新到旧
- **逻辑步骤**:
  1. 收藏菜谱 A 时记录时间戳 T_A
  2. 收藏菜谱 B 时记录时间戳 T_B（T_B > T_A）
  3. 查询收藏记录按时间戳降序排列
  4. 渲染列表时 B 在前 A 在后

### 场景2: 连续收藏的排序验证
- **触发条件**: 用户依次收藏菜谱 A、B、C
- **动作序列**:
  1. 按顺序收藏 A -> B -> C
  2. 打开收藏页
- **预期行为**:
  - 列表顺序: C（最前）-> B -> A
  - 最后收藏的 C 排在顶部
- **逻辑步骤**:
  1. 每次收藏记录当前时间戳
  2. T_C > T_B > T_A
  3. 按时间戳 DESC 排序

### 场景3: 重新收藏后移至顶部
- **触发条件**: 收藏列表中有菜谱 A（较早收藏）和菜谱 B（最近收藏）
- **动作序列**:
  1. 用户取消收藏菜谱 A
  2. 用户重新收藏菜谱 A
- **预期行为**:
  - 菜谱 A 的收藏时间更新为重新收藏时的时间
  - 菜谱 A 移至列表顶部
  - 列表顺序变为 A -> B
- **逻辑步骤**:
  1. 取消收藏删除记录
  2. 重新收藏写入新记录，时间戳更新为当前时间
  3. 新时间戳 > B 的时间戳
  4. 排序后 A 排在 B 前面

### 场景4: 同时收藏的时间区分
- **触发条件**: 用户在短时间内连续收藏多个菜谱
- **动作序列**:
  1. 快速连续收藏菜谱 X、Y、Z
- **预期行为**:
  - 排序按毫秒级时间戳区分
  - 最后操作的 Z 排在最前
- **逻辑步骤**:
  1. 每次收藏记录精确到毫秒的时间戳
  2. 即使间隔很短，时间戳仍有先后
  3. 按时间戳降序确保正确排列

### 场景5: 收藏列表数据源排序保证
- **触发条件**: 任意收藏列表状态
- **动作序列**:
  1. 系统加载收藏列表
- **预期行为**:
  - 排序在数据查询层面保证（按时间戳降序）
  - UI 层不做额外排序
- **逻辑步骤**:
  1. 数据层查询收藏记录时指定 ORDER BY 时间戳 DESC
  2. 返回的 ID 列表已是降序
  3. 映射为菜谱对象时保持原顺序

## 边界场景
- **仅一个收藏**: 列表只有一项，无需排序
- **所有收藏时间相同（极端情况）**: 按数据返回的稳定顺序排列
- **菜谱被删除后重新出现**: 若菜谱数据存在则按收藏时间排序

## 偏差说明
- 排序由数据层查询保证（ORDER BY savedAt DESC），非 UI 层排序
- 时间戳精确到毫秒，确保先后顺序可区分
- 重复收藏（REPLACE 策略）会更新时间戳，将菜谱移到顶部
