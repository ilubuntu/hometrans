# 加载与错误状态规格文档

## 需求概述
应用首次启动初始化菜谱数据时，首页应显示加载骨架屏（占位动画）。若菜谱数据初始化失败，应显示错误状态页面，包含错误提示文案和「重试」按钮。正常初始化路径不应长时间停留在加载状态；错误路径允许用户重试初始化。

## 原子场景

### 场景1: 首次启动显示加载骨架屏
- **触发条件**: 应用首次安装启动，本地菜谱数据库为空
- **动作序列**:
  1. 应用启动，进入首页
  2. 开始初始化菜谱数据
- **预期行为**:
  - 首页显示加载骨架屏（闪烁动画占位卡片）
  - 骨架屏模拟菜谱卡片布局（图片区 + 标题行 + 副标题行）
  - 加载状态持续短暂时间后切换到内容
- **逻辑步骤**:
  1. 首页 ViewModel 创建时自动触发数据初始化
  2. 初始化状态设为 Loading
  3. 检测本地数据库菜谱数为 0
  4. 执行种子数据加载（含刻意延迟使加载可感知）
  5. 加载期间首页渲染骨架屏组件
  6. 加载完成后状态切换为 Ready -> 渲染实际内容

### 场景2: 正常初始化完成
- **触发条件**: 菜谱数据初始化成功
- **动作序列**:
  1. 种子数据加载完成
  2. 数据写入本地数据库
- **预期行为**:
  - 加载骨架屏消失
  - 首页显示完整内容（问候语、食材库卡片、菜谱列表等）
  - 加载状态不长时间停留
- **逻辑步骤**:
  1. 种子数据解析成功
  2. 批量写入数据库
  3. 初始化状态设为 Ready
  4. 页面重新渲染为内容模式

### 场景3: 初始化失败显示错误状态
- **触发条件**: 菜谱数据初始化过程中发生异常
- **动作序列**:
  1. 种子数据加载抛出异常
- **预期行为**:
  - 加载骨架屏消失
  - 显示居中错误状态视图
  - 显示红色圆形图标
  - 显示标题「Something went wrong」
  - 显示说明文案「We couldn't load your recipes. Please try again.」
  - 显示「Try again」操作按钮
- **逻辑步骤**:
  1. 种子数据加载捕获异常
  2. 初始化状态设为 Error
  3. 页面渲染错误状态组件
  4. 错误组件含图标、标题、说明和重试按钮

### 场景4: 点击重试重新初始化
- **触发条件**: 首页显示错误状态
- **动作序列**:
  1. 用户点击「Try again」按钮
- **预期行为**:
  - 重新执行菜谱数据初始化
  - 页面先切换回加载骨架屏
  - 初始化成功 -> 显示内容
  - 初始化再次失败 -> 再次显示错误状态
- **逻辑步骤**:
  1. 接收重试按钮点击事件
  2. 重新触发数据初始化
  3. 初始化状态设为 Loading -> 渲染骨架屏
  4. 执行种子加载（幂等，数据库为空时才执行）
  5. 成功 -> Ready，失败 -> Error

### 场景5: 非首次启动跳过加载
- **触发条件**: 应用非首次启动，本地已有菜谱数据
- **动作序列**:
  1. 应用启动，进入首页
- **预期行为**:
  - 检测到本地数据库已有菜谱数据
  - 跳过种子数据加载
  - 快速显示首页内容（加载时间极短）
- **逻辑步骤**:
  1. 初始化时检测本地数据库菜谱数
  2. 菜谱数 > 0 -> 跳过种子加载
  3. 初始化状态直接设为 Ready
  4. 页面快速渲染内容

### 场景6: 加载骨架屏视觉
- **触发条件**: 首页处于加载状态
- **动作序列**:
  1. 查看加载骨架屏
- **预期行为**:
  - 显示多个占位卡片
  - 每个卡片模拟菜谱卡片结构：图片区（高矩形）+ 标题行（70%宽）+ 副标题行（45%宽）
  - 所有占位区域有闪烁（shimmer）动画
- **逻辑步骤**:
  1. 渲染骨架卡片组件
  2. 图片区、标题行、副标题行使用 shimmer 修饰符
  3. shimmer 产生从左到右的渐变扫光动画

## 边界场景
- **初始化幂等**: 种子加载仅在数据库为空时执行，重试不会重复写入
- **并发保护**: 初始化操作有互斥锁保护，防止并发重复执行
- **加载时间**: 正常路径有约 0.5 秒刻意延迟使加载可感知，非首次启动几乎无延迟

## 偏差说明
- 加载状态仅在首页首次初始化时出现，其他页面（搜索/收藏/详情）无独立加载骨架
- 错误状态仅针对菜谱数据初始化失败，不覆盖其他运行时错误
- 重试操作重新执行完整初始化流程（Loading -> Ready/Error）
