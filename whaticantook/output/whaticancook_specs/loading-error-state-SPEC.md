# 错误与加载状态SPEC

应用在加载离线数据或出现异常时，各页面提供明确的加载态、错误态或空态，避免长时间白屏或崩溃，并在异常时给出可理解的提示与恢复手段。

## 场景一

### 场景概述

应用启动时避免白屏，数据就绪后再进入主界面

### 场景逻辑步骤

- 1. 用户打开应用
- 2. 应用在首屏数据就绪前展示启动屏占位，不出现长时间白屏
- 3. 应用完成首屏必要数据准备后，启动屏消失，进入主界面（已跳过引导时进入首页）

## 场景二

### 场景概述

首页加载内置菜谱数据时展示骨架占位

### 场景逻辑步骤

- 1. 用户进入首页，内置菜谱数据尚未初始化完成
- 2. 首页展示菜谱卡片的骨架占位（带有微光动效的占位卡片），而非空白
- 3. 内置菜谱数据初始化完成后，骨架占位被实际菜谱内容替换

## 场景三

### 场景概述

首页数据初始化异常时展示错误态并可重试

### 场景逻辑步骤

- 1. 内置菜谱数据初始化过程中出现异常
- 2. 首页展示居中的错误态：
  - 2.1 展示错误图标
  - 2.2 展示标题"Something went wrong"
  - 2.3 展示说明"We couldn't load your recipes. Please try again."
  - 2.4 展示"Try again"重试按钮
- 3. 页面不崩溃，也不长期空白
- 4. 用户点击"Try again"按钮，应用重新尝试初始化菜谱数据
- 5. 若重试成功，首页切换为正常菜谱内容；若仍失败，保持错误态可再次重试

## 场景四

### 场景概述

菜谱详情页加载时占位、菜谱不存在时空态提示

### 场景逻辑步骤

- 1. 用户进入某菜谱详情页，数据尚未就绪时，详情页展示空白占位（短暂过渡，不崩溃）
- 2. 数据就绪且菜谱存在时，展示详情内容
- 3. 若该菜谱不存在（例如已被删除或标识无效），详情页展示空态提示：
  - 3.1 展示图标
  - 3.2 展示标题"Recipe not found"
  - 3.3 展示说明"This recipe is no longer available."

## 场景五

### 场景概述

搜索页、收藏页、食材库页加载态与空结果态

### 场景逻辑步骤

- 1. 用户进入搜索页、收藏页或食材库页，数据首次加载完成前页面处于加载态
- 2. 数据加载完成后：
  - 2.1 搜索页若无匹配菜谱，展示空结果态（不长期空白）
  - 2.2 收藏页若无收藏菜谱，展示空收藏态并提供浏览入口
  - 2.3 食材库页按实际食材数据展示内容或空态
- 3. 各页面均可在数据就绪后恢复展示，不长期空白、不崩溃

## 场景六

### 场景概述

"错误与加载状态"的作用范围与不作用范围

### 场景逻辑步骤

- 1. **仅作用于** 以下页面的加载/错误/空态: 应用启动（启动屏占位）/ 首页（骨架屏 + 错误重试）/ 菜谱详情页（占位 + 找不到菜谱空态）/ 搜索页（加载态 + 空结果态）/ 收藏页（加载态 + 空收藏态）/ 食材库页（加载态）
- 2. **不作用于** 以下页面: 设置页（静态信息，无独立加载/错误数据态）/ 引导页（固定引导内容，无加载/错误数据态）—— 这两个页面不展示骨架屏、错误重试或找不到数据空态

> [偏差] 需求未明确各页加载态的具体形式；代码中首页用骨架屏、详情页用空白占位、搜索/收藏/食材库用加载标志控制（见场景二、四、五）。需求未明确具体提示文案；代码首页异常提示为"Something went wrong…"、详情页菜谱不存在提示为"Recipe not found…"（见场景三、四）。均为实现层面的补充说明，与需求"应有明确状态""显示可理解的提示"一致。
> [交叉引用] 搜索页空结果的详细表现见 `search-empty-SPEC.md`；收藏页空态的详细表现见 `saved-empty-SPEC.md`。
