# 搜索页Fewest missing排序SPEC

搜索页排序项"Fewest missing"按菜谱缺少必需食材的数量从少到多对结果列表排序（升序）；缺少必需食材数量相同时，按匹配度从高到低排列。

## 场景一

### 场景概述

用户选择"Fewest missing"，结果列表按缺少必需食材数量从少到多排序

### 场景逻辑步骤

- 1. 用户在食材库中添加部分食材
- 2. 用户位于搜索页排序条，点击"Fewest missing"项
- 3. "Fewest missing"项进入选中态，其余排序项退出选中态
- 4. 系统计算每道菜谱缺少的"必需"食材数量：
  - 4.1 仅统计必需食材，可选食材不计入缺少数
  - 4.2 将每道菜谱的必需食材与当前食材库逐一比对，未被满足的必需食材计入缺少数
- 5. 结果列表按缺少必需食材数量从少到多重新排列，缺少更少的菜谱排在更前

## 场景二

### 场景概述

缺少必需食材数量相同时，按匹配度从高到低排列

### 场景逻辑步骤

- 1. 当前生效排序为"Fewest missing"
- 2. 当两道或多道菜谱缺少的必需食材数量相同时
- 3. 这些菜谱之间按其匹配度（已满足必需食材数 ÷ 必需食材总数）从高到低排列
- 4. 缺少数不同者仍严格按缺少数从少到多排列

## 场景三

### 场景概述

食材库变化时，"Fewest missing"排序结果实时刷新

### 场景逻辑步骤

- 1. 当前生效排序为"Fewest missing"，结果列表按当前缺少数排列
- 2. 用户在食材库中增加或移除任一食材
- 3. 系统立即重新计算每道菜谱缺少的必需食材数量与匹配度
- 4. 结果列表按新缺少数从少到多重新排列

## 场景四

### 场景概述

"Fewest missing"排序与其他筛选叠加，且只对已过滤结果重排

### 场景逻辑步骤

- 1. 用户在搜索页可能已输入搜索词、选定分类、开启"Cookable"
- 2. 系统先按搜索词、分类、可做筛选过滤出结果集合
- 3. 再对过滤后的结果集合按缺少必需食材数量从少到多重排
- 4. 排序只改变结果集合的顺序，不改变其成员（即不会因排序增减菜谱）

## 场景五

### 场景概述

"Fewest missing"排序的作用范围与不作用范围

### 场景逻辑步骤

- 1. **仅作用于** 搜索页结果列表的排序
- 2. **不作用于** 首页菜谱列表——首页列表有独立的排序，不受搜索页"Fewest missing"影响
- 3. **可与文本搜索、分类筛选、可做筛选叠加**：排序作用于已过滤结果，四者可任意组合
- 4. **仅统计必需食材的缺少数**：可选食材不参与"Fewest missing"的排序计数

> [偏差] 需求描述未明确相同缺少数时的次序，实际同缺少数菜谱按匹配度从高到低排列，为合理的实现细节。
