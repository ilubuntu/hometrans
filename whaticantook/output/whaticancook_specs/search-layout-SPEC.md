# 搜索页布局SPEC

搜索页为用户提供菜谱检索界面，自上而下依次包含页面标题、搜索输入框、分类筛选条、可做筛选与排序控件条，以及下方的搜索结果列表区域；用户可在此页输入关键词、选择分类、切换"可做"筛选并选择排序方式。

## 场景一

### 场景概述

用户进入搜索页后，看到完整的页面布局结构

### 场景逻辑步骤

- 1. 用户点击底部导航的"Search"入口，或点击首页顶部的搜索入口条，进入搜索页
- 2. 搜索页自上而下依次展示以下区域：
  - 2.1 顶部展示页面大标题"Search"
  - 2.2 标题下方为搜索输入框，未输入时框内显示占位文案"Search recipes…"；输入框为单行输入，键盘动作键为"搜索"
  - 2.3 输入框下方为横排可左右滑动的分类筛选条，依次排列"All"项与各菜谱分类项
  - 2.4 分类条下方为一行控件：左侧为"可做(Cookable)"切换项（项前带勾选图标），其后为横排可滑动的排序项
  - 2.5 控件条下方为搜索结果区域，以可滚动的菜谱卡片列表形式展示当前匹配结果

## 场景二

### 场景概述

分类筛选条展示固定分类集合，并单选高亮当前选中分类

### 场景逻辑步骤

- 1. 用户查看分类筛选条
- 2. 分类条依次包含"All"项与以下分类项，每项前带分类图标：Breakfast、Lunch、Dinner、Dessert、Snack、Drinks
- 3. 分类项为单选互斥：任意时刻最多一项处于选中态（"All"或某一具体分类）
- 4. 应用首次进入搜索页时，"All"项默认处于选中态
- 5. 处于选中态的项背景高亮、文字反色并去除边框；未选中项保持底色与描边

## 场景三

### 场景概述

可做筛选为开关切换，排序项为单选；二者并排于同一控件条

### 场景逻辑步骤

- 1. 用户查看分类条下方的控件条
- 2. 排序项依次为："Best match""Quickest""Fewest missing"，横向排列可左右滑动
- 3. 排序项为单选互斥：任意时刻最多一项处于选中态
- 4. 应用首次进入搜索页时，"Best match"项默认处于选中态
- 5. "Cookable"项为开关：点击在"选中"与"未选中"两态间切换；应用首次进入搜索页时默认为未选中
- 6. 处于选中态的项背景高亮、文字反色并去除边框

## 场景四

### 场景概述

搜索结果列表以菜谱卡片形式展示，每张卡片可进入详情并支持收藏切换

### 场景逻辑步骤

- 1. 用户查看控件条下方的结果区
- 2. 当存在匹配结果时，结果以可滚动的菜谱卡片列表展示，每张卡片包含分类、标题、描述、时间、难度、热量、食材匹配计数与心形收藏按钮
- 3. 用户点击某张菜谱卡片，进入该菜谱的详情页
- 4. 用户点击某张卡片右上角的心形按钮，切换该菜谱的收藏状态

## 场景五

### 场景概述

"搜索页布局"的作用范围与不作用范围

### 场景逻辑步骤

- 1. **仅作用于** 搜索页：该页固定包含标题、搜索输入框、分类筛选条、可做筛选+排序条、结果区五部分
- 2. **不与首页共用筛选状态**：首页虽也有分类筛选，但首页与搜索页的分类选择各自独立，在一处选定分类不影响另一处
- 3. **不作用于** 收藏页：收藏页仅展示收藏列表，不提供搜索框、分类筛选条与排序条
- 4. **分类项与排序项集合固定** 为应用内置集合，用户不能新增、编辑或删除

> [偏差] 需求验收列举的分类为"All、Breakfast、Lunch、Dinner"，实际分类项另含 Dessert、Snack、Drinks（共 6 个具体分类 + All），与需求用列举方式描述一致，无冲突，此处补全。
