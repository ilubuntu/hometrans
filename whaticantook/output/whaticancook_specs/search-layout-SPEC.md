# Search 页面基础布局 规格文档

## 需求概述
搜索页是应用的主要菜谱浏览入口之一，提供多维度的菜谱查找能力。页面自上而下包含：标题、搜索输入框、分类筛选条（全部 + 多个菜谱分类）、"仅看可做"开关、排序方式选项（最佳匹配/最快/最少缺失），以及符合筛选条件的菜谱结果列表。页面进入时使用合理的默认筛选（全部分类、最佳匹配排序），底部导航中"搜索"主入口处于高亮状态，让用户清楚当前所处位置。

## 原子场景

### 场景1: 页面整体结构展示
- **触发条件**: 用户切换到搜索主入口。
- **动作序列**: 用户查看搜索页。
- **预期行为**: 页面自上而下展示：标题"搜索"、搜索输入框、横向滚动的分类筛选条、"仅看可做"开关与排序选项行、菜谱结果列表。
- **逻辑步骤**:
  1. 渲染标题"搜索"。
  2. 渲染搜索输入框（见"Search 文本搜索"规格）。
  3. 渲染分类筛选条（见"Search 分类筛选"规格）。
  4. 渲染"仅看可做"开关（见"Search Cookable 筛选"规格）与排序选项（见对应排序规格）。
  5. 渲染结果列表。

### 场景2: 默认筛选态
- **触发条件**: 用户首次进入搜索页。
- **动作序列**: 用户查看筛选控件。
- **预期行为**: 分类筛选默认选中"全部"；排序默认选中"最佳匹配"；"仅看可做"默认关闭。结果列表展示全部菜谱（按最佳匹配排序）。
- **逻辑步骤**:
  1. 初始化分类为"全部"。
  2. 初始化排序为"最佳匹配"。
  3. 初始化"仅看可做"为关闭。
  4. 按默认筛选渲染全部菜谱结果。

### 场景3: 分类筛选条展示
- **触发条件**: 搜索页渲染分类筛选区。
- **动作序列**: 用户查看横向筛选条。
- **预期行为**: 筛选条以可横向滚动的标签组形式展示，包含"全部"与各菜谱分类标签（与首页分类一致，含图标），当前选中分类高亮。
- **逻辑步骤**:
  1. 取菜谱分类集合。
  2. 渲染"全部"标签 + 各分类标签。
  3. 当前选中分类高亮显示。

### 场景4: "仅看可做"开关与排序选项行
- **触发条件**: 搜索页渲染筛选控件行。
- **动作序列**: 用户查看开关与排序行。
- **预期行为**: 该行左侧为"仅看可做"开关标签，右侧为可横向滚动的排序选项标签组（最佳匹配/最快/最少缺失），当前排序高亮。
- **逻辑步骤**:
  1. 渲染"仅看可做"开关标签。
  2. 渲染排序选项标签组。
  3. 当前排序与开关状态高亮显示。

### 场景5: 结果列表展示
- **触发条件**: 存在符合筛选条件的菜谱。
- **动作序列**: 用户查看结果列表。
- **预期行为**: 以全宽卡片列表形式展示符合当前筛选与排序的菜谱；每张卡片复用首页菜谱卡片样式（含匹配胶囊与收藏按钮），点击可进入详情。
- **逻辑步骤**:
  1. 按当前筛选与排序得到结果集。
  2. 非空 → 渲染菜谱卡片列表。
  3. 为空 → 渲染空态（见"Search 空结果"规格）。

### 场景6: 底部导航高亮
- **触发条件**: 用户处于搜索页。
- **动作序列**: 用户查看底部导航。
- **预期行为**: 底部导航中"搜索"主入口处于高亮选中状态，其余主入口非高亮。
- **逻辑步骤**:
  1. 底部导航根据当前路由高亮对应主入口。
  2. 搜索路由 → "搜索"高亮。

## 边界场景
- 无结果：结果列表区显示空态提示（见"Search 空结果"规格），不崩溃、不展示旧结果。
- 食材库变化：结果卡片的匹配胶囊与排序实时刷新。
- 搜索页结果卡片复用首页菜谱卡片组件，样式与交互一致（点击进详情、收藏切换）。
- 分类筛选、排序、开关的详细行为见各自独立规格。

## 偏差说明
- 无。规格按需求描述的布局元素与默认选中态呈现。迁移 HarmonyOS 时应实现等价的分层布局（标题 + 搜索框 + 分类筛选条 + 开关与排序行 + 结果列表）与默认筛选态。
- 本规格聚焦页面布局与默认态；搜索/筛选/排序的具体逻辑见"Search 文本搜索（search-text-SPEC.md）""Search 分类筛选（search-category-SPEC.md）""Search Cookable 筛选（search-cookable-SPEC.md）""Search Best match 排序（search-best-match-SPEC.md）""Search Quickest 排序（search-quickest-SPEC.md）"等独立规格。
