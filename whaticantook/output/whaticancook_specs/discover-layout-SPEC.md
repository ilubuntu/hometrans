# Discover首页基础布局SPEC

Discover 首页自上而下展示问候语、主标题、设置入口、搜索入口、食材库入口、可做菜谱区或空食材提示、分类筛选区与菜谱列表，底部提供四个顶级页面导航入口。

## 场景一

### 场景概述

用户进入 Discover 首页，展示完整的页面布局

### 场景逻辑步骤

- 1. 用户跳过或完成引导后进入首页（或启动时已标记引导完成，直接进入首页）
- 2. 首页菜谱数据装载完成后，自上而下展示以下区段：
  - 2.1 顶部区：左侧问候语 + 主标题"What can I cook?"，右侧设置入口图标
    - 问候语按当前时间段取值：上午显示"Good morning"，下午显示"Good afternoon"，傍晚/晚上显示"Good evening"，深夜显示"Hungry tonight?"
  - 2.2 搜索入口条：展示搜索图标与提示文案"Search recipes, ingredients…"，整体可点击
  - 2.3 食材库入口卡（"Your pantry"卡）：副标题随食材数量变化（见场景二）
  - 2.4 可做菜谱区或空食材提示：
    - 当存在可做菜谱时，展示"Ready to cook"区，横向排列可做菜谱卡片
    - 当无可做菜谱且食材库为空时，展示空食材提示卡（完整判定与文案见 `empty-pantry-hint-SPEC.md`）
  - 2.5 分类筛选区：先展示"Browse recipes"标题，下方横排展示"All"与各分类筛选项（交互见 `category-filter-SPEC.md`）
  - 2.6 菜谱列表区：纵向排列菜谱卡片（卡片内容见 `recipe-card-SPEC.md`）
- 3. 页面底部展示底部导航栏（见场景三）

## 场景二

### 场景概述

"Your pantry"入口卡的副标题随食材库数量动态变化

### 场景逻辑步骤

- 1. 用户位于首页
- 2. 首页读取食材库当前数量
- 3. "Your pantry"卡副标题按数量取值：
  - 3.1 数量为 0 时，副标题显示"Add what you have at home"
  - 3.2 数量为 1 时，副标题显示"1 ingredient ready"
  - 3.3 数量大于 1 时，副标题显示"N ingredients ready"（N 为实际数量）
- 4. 食材库数量变化时，副标题随之刷新
- 5. 用户点击该卡片，进入食材库页

## 场景三

### 场景概述

底部导航栏在首页与其他顶级页展示，当前页对应入口高亮

### 场景逻辑步骤

- 1. 用户位于首页或其他顶级页（搜索、食材库、收藏）
- 2. 页面底部展示底部导航栏，依次排列四个入口：Discover、Search、Pantry、Saved
- 3. 当前所在页对应的导航入口处于高亮态：背景高亮、图标与文字颜色为强调色，并展开显示文字标签
- 4. 其余入口为非高亮态，仅显示图标、不显示文字标签
- 5. 用户点击任一入口，进入对应顶级页，高亮态随之切换

## 场景四

### 场景概述

Discover 首页布局的作用范围与不作用范围

### 场景逻辑步骤

- 1. **仅作用于** Discover 首页：顶部区（问候语 + 主标题 + 设置入口）、搜索入口条、"Your pantry"入口卡、"Browse recipes"标题与分类筛选区、菜谱列表区均为首页专属
- 2. **跨顶级页共享** 底部导航栏：搜索、食材库、收藏四个顶级页与首页共用同一底部导航栏及其高亮逻辑
- 3. **不作用于** 二级页面（菜谱详情页、设置页）：这些页面不展示底部导航栏
- 4. **布局随状态切换**：菜谱数据装载中或装载出错时，首页不展示上述内容布局，改展示加载占位或错误态（详见相关加载/错误状态需求）

> [偏差] 需求问候语仅列"Good morning/evening"，实际问候语另含下午"Good afternoon"与深夜"Hungry tonight?"两段（见场景一步骤2.1）。需求搜索文案"Search recipes, ingredients..."实际为同一含义的省略号写法。
