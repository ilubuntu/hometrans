# Pantry 清空 规格文档

## 需求概述
Pantry 页"我的厨房"区提供"清空全部"入口，用户点击后移除食材库中所有食材。清空后：Pantry 页恢复空态（数量副标题为空态文案、隐藏"我的厨房"区、快速添加区标题改为"添加食材"），并且首页与搜索页的菜谱匹配状态恢复为空食材库状态（如首页食材库卡片显示空态文案、不再显示可做菜谱区）。

## 原子场景

### 场景1: 点击清空全部移除所有食材
- **触发条件**: 食材库已有至少一个食材，处于 Pantry 页。
- **动作序列**: 用户点击"我的厨房"区的"清空全部"入口。
- **预期行为**: 食材库全部食材被删除。
- **逻辑步骤**:
  1. 点击"清空全部"触发清空动作。
  2. 删除食材库中所有食材记录。

### 场景2: Pantry 页恢复空态
- **触发条件**: 清空动作完成后。
- **动作序列**: 用户查看 Pantry 页。
- **预期行为**: 数量副标题显示空态引导文案；"我的厨房"区隐藏；快速添加区标题改为"添加食材"；所有快速添加目录项回到可选状态。
- **逻辑步骤**:
  1. 食材列表为空 → 隐藏"我的厨房"区。
  2. 快速添加区标题切换为"添加食材"。
  3. 去重集合为空 → 全部目录项重新可选。
  4. 数量副标题按 0 渲染空态文案。

### 场景3: 首页匹配状态恢复空态
- **触发条件**: 清空动作完成后，用户回到首页。
- **动作序列**: 用户查看首页。
- **预期行为**: 食材库卡片副标题显示"添加你家里有的食材"（空态文案）；不显示"可做菜谱"区，改显示添加食材引导提示；菜谱卡片匹配状态为空食材库状态。
- **逻辑步骤**:
  1. 食材库为空 → 首页食材库卡片副标题为空态文案。
  2. 可做菜谱集合为空（要求食材库非空）→ 不显示"可做菜谱"区，显示引导提示。
  3. 菜谱匹配按空食材库重算。

### 场景4: 搜索/详情匹配状态恢复空态
- **触发条件**: 清空动作完成后。
- **动作序列**: 用户进入搜索或详情。
- **预期行为**: 菜谱匹配计数恢复为 0/必需数；可做筛选无结果；详情页可做状态降级为缺少食材。
- **逻辑步骤**:
  1. 食材库为空 → 所有菜谱匹配按空库重算。
  2. 搜索可做筛选、详情可做状态随之更新。

## 边界场景
- 食材库本就为空：清空入口不显示（"我的厨房"区隐藏），无可清空内容。
- 清空动作立即生效，无加载延迟。
- 清空后再次添加食材可正常恢复匹配状态。

## 偏差说明
- 实现中清空无二次确认弹窗（点击即清空）。规格以代码行为为准（直接清空）；迁移时如需提升安全性，可考虑增加二次确认，但属可选增强，非必需。
- 另有设置页"清空食材库"入口（对应 REQ-039），与本规格的 Pantry 页入口效果一致；本规格仅描述 Pantry 页"清空全部"。
- 跨页面状态恢复依赖响应式数据流；迁移 HarmonyOS 时需以等价的响应式状态广播机制实现。
