# 首页分类筛选SPEC

首页菜谱列表上方的分类筛选区提供"All"与多个菜谱分类项，用户选择某一分类后，菜谱列表即时过滤为该分类的菜谱；选择"All"恢复展示全部菜谱。

## 场景一

### 场景概述

用户选择一个具体分类，菜谱列表只展示该分类的菜谱

### 场景逻辑步骤

- 1. 用户位于首页菜谱列表上方的分类筛选区，看到横排的"All"项与各分类项
- 2. 各分类项依次为：Breakfast、Lunch、Dinner、Dessert、Snack、Drinks（每项前带分类图标）
- 3. 用户点击某一分类项（如"Dinner"）
- 4. 该分类项进入选中态（背景高亮、文字反色、去除边框）
- 5. 菜谱列表即时过滤，只展示属于该分类（晚餐）的菜谱
- 6. 过滤结果内菜谱按食材匹配度从高到低、标题字母顺序排列

## 场景二

### 场景概述

用户选择"All"，菜谱列表恢复展示全部菜谱

### 场景逻辑步骤

- 1. 用户位于首页，当前已选中某一具体分类（如"Dinner"），列表只显示晚餐菜谱
- 2. 用户点击"All"项
- 3. "All"项进入选中态，原分类项退出选中态
- 4. 菜谱列表取消分类过滤，恢复展示全部菜谱
- 5. 全部菜谱同样按食材匹配度从高到低、标题字母顺序排列

## 场景三

### 场景概述

分类项为单选互斥，任意时刻最多一项处于选中态

### 场景逻辑步骤

- 1. 用户位于首页分类筛选区
- 2. "All"与各具体分类项构成一个单选组
- 3. 任意时刻最多只有一项处于选中态（"All"或某一个具体分类）
- 4. 用户点击任一未选中项后，原选中项自动退出选中态，新点击项进入选中态
- 5. 应用首次进入首页时，"All"项默认处于选中态（即默认展示全部菜谱）

## 场景四

### 场景概述

首页分类筛选的作用范围与不作用范围

### 场景逻辑步骤

- 1. **仅作用于** 首页"Browse recipes"菜谱列表的分类过滤
- 2. **不作用于** 首页"Ready to cook"可做菜谱区——该区由食材匹配独立计算，不受分类筛选影响
- 3. **不与搜索页共用状态** 搜索页虽也有分类筛选，但二者各自独立，在首页选定的分类不影响搜索页，反之亦然
- 4. **分类项集合固定** 为应用内置的菜谱分类，用户不能新增、编辑或删除分类项

> [偏差] 需求分类列举"All、Breakfast、Lunch、Dinner、Snack、Dessert 等"，实际分类项另含"Drinks"（需求用"等"表举例，无冲突）。过滤结果内菜谱按匹配度+标题排序为代码实现细节，需求未明确排序但行为与"按所选分类刷新列表"一致。
