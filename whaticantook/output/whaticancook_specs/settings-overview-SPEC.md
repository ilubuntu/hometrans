# Settings 页面基础信息SPEC

用户从首页右上角设置图标进入"设置"页面，页面按"外观 / 数据 / 关于"三个区块展示主题切换项、清空食材项与关于信息。

## 场景一

### 场景概述

进入"设置"页面后，按"外观 / 数据 / 关于"顺序展示各区块内容

### 场景逻辑步骤

- 1. 用户在"首页"右上角点击设置图标
- 2. 进入"设置"页面，顶部展示标题"Settings"与返回按钮
- 3. 页面按顺序展示以下区块：
  - 3.1 外观区块（标题标签"Appearance"）：
    - 展示主题项标题"Theme"
    - 展示三个主题选项，文案依次为"Light" / "Dark" / "Match system"
  - 3.2 数据区块（标题标签"Data"）：
    - 展示"Clear pantry"项，标题文案为"Clear pantry"
  - 3.3 关于区块（标题标签"About"）：
    - 应用名称行：标签"App"，值为"WhatCanICook"
    - 版本号行：标签"Version"，值为"1.0.0"
    - 构建技术行：标签"Built with"，值为"Jetpack Compose"

## 场景二

### 场景概述

进入"设置"页面时，主题选项反映当前已生效的主题

### 场景逻辑步骤

- 1. 用户在"首页"右上角点击设置图标进入"设置"页面
- 2. 外观区块的三个主题选项中，与当前生效主题一致的那个处于选中高亮状态；其余两个为未选中状态
- 3. 若此前从未设置过主题，则"Match system"处于选中状态（默认主题）

## 场景三

### 场景概述

"设置"页面信息展示的作用范围与不作用范围

### 场景逻辑步骤

- 1. **仅作用于** 以下界面: "设置"页面本身的外观区块、数据区块、关于区块展示
- 2. **仅可通过** 以下入口进入: "首页"右上角设置图标
- 3. **不作用于** 以下位置: "首页"(仅提供入口图标，不展示设置区块内容) / "搜索页" / "食材库页" / "收藏页" / "菜谱详情页" — 这些页面既不展示外观/数据/关于区块内容，也不提供进入"设置"页面的入口

> [偏差] 需求仅列出区块标题与关键字段。代码在外观区块的主题项标题上方还展示区块标签"APPEARANCE"，清空食材项下还展示副标题"Remove all ingredients you've added"，关于区块底部还展示一段离线运行说明小字（参见场景一步骤 3）。这些为补充信息，不影响需求要求的核心展示内容。
> [交叉引用] 主题选项的切换交互与运行时配色变化见 `theme-switch-SPEC.md`；主题选择的持久保存见 `theme-persist-SPEC.md`。
