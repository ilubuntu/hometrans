# 首页菜谱卡片 规格文档

## 需求概述
首页（及可做菜谱区）的菜谱卡片以全宽卡片形式展示菜谱信息。每张卡片包含：菜谱视觉元素（渐变背景 + 食物图形）、匹配状态胶囊（按食材匹配情况显示"可做/缺少 N 个/已拥有 N/M"）、收藏按钮、分类、标题、简介、以及耗时/难度/热量三项元信息。整张卡片可点击进入菜谱详情。

## 原子场景

### 场景1: 卡片完整内容展示
- **触发条件**: 首页或可做菜谱区渲染一张菜谱卡片。
- **动作序列**: 用户查看菜谱卡片。
- **预期行为**: 卡片自上而下展示：视觉元素区（渐变背景 + 居中食物图形，左上角匹配状态胶囊、右上角收藏按钮）；信息区显示分类（图形+分类名）、标题、简介、以及一行元信息（耗时、难度、热量）。
- **逻辑步骤**:
  1. 取菜谱的图形索引与食物图形渲染视觉元素区。
  2. 根据食材匹配结果渲染左上角状态胶囊（场景2）。
  3. 根据收藏状态渲染右上角收藏按钮。
  4. 渲染分类（图形 + 分类名）、标题、简介（标题与简介超长时省略）。
  5. 渲染元信息行：耗时（如 "5 min" 或 "1h 30m"）、难度（如 "Easy"）、热量（如 "400 kcal"）。

### 场景2: 匹配状态胶囊随匹配情况变化
- **触发条件**: 菜谱卡片渲染，食材库与菜谱必需食材比对完成。
- **动作序列**: 食材库内容变化导致匹配状态变化。
- **预期行为**:
  - 必需食材全部满足 → 绿色胶囊显示 "Ready to cook"。
  - 缺少 1–2 个必需食材 → 警告色胶囊显示 "Missing N"（N 为缺少数量）。
  - 缺少超过 2 个必需食材 → 中性色胶囊显示 "已拥有/必需" 计数（如 "0/4"）。
- **逻辑步骤**:
  1. 比对食材库与菜谱必需食材，得到已拥有数 / 必需数 / 缺少数。
  2. 必需数为 0 或无缺少 → 可做状态。
  3. 缺少数 ≤ 2 → 接近可做状态。
  4. 缺少数 > 2 → 探索状态，显示 "已拥有/必需" 计数。

### 场景3: 点击卡片进入详情
- **触发条件**: 用户点击菜谱卡片。
- **动作序列**: 用户点击卡片任意可点击区域。
- **预期行为**: 跳转到该菜谱详情页。
- **逻辑步骤**:
  1. 卡片整体可点击，点击携带菜谱 ID 触发打开详情页导航。

### 场景4: 收藏按钮切换
- **触发条件**: 菜谱卡片右上角收藏按钮可见。
- **动作序列**: 用户点击收藏按钮。
- **预期行为**: 收藏状态在未收藏/已收藏间切换，按钮视觉即时反映；不影响卡片其它区域（点击按钮不进入详情）。
- **逻辑步骤**:
  1. 收藏按钮独立可点击，阻止冒泡到卡片点击。
  2. 切换该菜谱的收藏状态并持久化。
  3. 按钮视觉根据新状态更新。

### 场景5: 验证示例（5-Minute Mug Cake）
- **触发条件**: 食材库为空，渲染 5-Minute Mug Cake 卡片。
- **动作序列**: 查看该卡片。
- **预期行为**: 分类显示 Dessert；耗时 "5 min"；难度 "Easy"；热量 "400 kcal"；匹配胶囊显示 "0/4"。
- **逻辑步骤**:
  1. 该菜谱分类为甜点、耗时 5 分钟、难度 Easy、热量 400。
  2. 必需食材 4 个，食材库为空 → 已拥有 0/4，缺少超过 2 个 → 探索态胶囊 "0/4"。

## 边界场景
- 标题或简介过长：最多显示 2 行并以省略号截断。
- 必需食材数为 0 的菜谱：直接判定为可做，胶囊显示 "Ready to cook"。
- 耗时 ≥ 60 分钟：以 "Nh" 或 "Nh Mm" 形式显示（如 "1h 30m"）。
- 卡片视觉元素区为渐变背景 + 表情图形，无外部图片加载，离线可用且每菜谱稳定一致。

## 偏差说明
- 实现中"图片"为程序生成的品牌渐变背景 + 食物表情（emoji），无真实图片资源。规格按"视觉元素（渐变背景 + 食物图形）"描述，迁移 HarmonyOS 时可用渐变 + emoji 或自绘图形等价实现。
- "匹配计数如 0/4"仅在探索态（缺少超过 2 个必需食材）以"已拥有/必需"形式显示；可做态显示"Ready to cook"，接近态显示"Missing N"。
