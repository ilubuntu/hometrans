# 基础计算器SPEC

计算器主页面基础运算操作

## 场景一

### 场景概述

用户打开应用后默认进入计算器主页面，页面初始状态展示结果为 0

### 场景逻辑步骤

- 1. 用户打开应用
- 2. 默认进入计算器主页面
- 3. 页面上方表达式区为空，下方结果区显示数字 0

## 场景二

### 场景概述

用户在初始状态下点击数字键输入数字，数字正确追加到结果区显示

### 场景逻辑步骤

- 1. 页面结果区当前显示 0
- 2. 用户点击数字键（1-9 中的任意一个）
- 3. 结果区的 0 被替换为所点击的数字并显示
- 4. 用户继续点击数字键（0-9 中的任意一个）
- 5. 新输入的数字追加到结果区末尾，结果区显示连续输入的完整数字（以千分位逗号分隔显示，如输入 4900 则显示 4,900）

## 场景三

### 场景概述

用户在初始状态下连续点击数字 0，结果区始终保持 0 不产生前导零

### 场景逻辑步骤

- 1. 页面结果区当前显示 0
- 2. 用户点击数字 0 键
- 3. 结果区仍显示 0，不出现前导零
- 4. 用户再次点击数字 0 键
- 5. 结果区仍显示 0

## 场景四

### 场景概述

用户点击小数点键输入小数，小数点正确追加到结果区

### 场景逻辑步骤

- 1. 页面结果区当前显示 0
- 2. 用户点击小数点键
- 3. 结果区显示 0. （0 后面紧跟小数点）
- 4. 用户继续点击数字键（如 0、0）
- 5. 数字追加到小数点后，结果区显示完整小数（如 0.00）

## 场景五

### 场景概述

用户重复点击小数点键，结果区仅保留一个小数点

### 场景逻辑步骤

- 1. 结果区当前已含小数点（如显示 0.）
- 2. 用户再次点击小数点键
- 3. 结果区不发生变化，仍只保留一个小数点

## 场景六

### 场景概述

用户在初始状态下（表达式为空且结果为 0）点击运算符，页面无反应

### 场景逻辑步骤

- 1. 页面表达式区为空，结果区显示 0
- 2. 用户点击任意四则运算符（+、-、×、÷）
- 3. 页面不发生变化，表达式区仍为空，结果区仍显示 0

## 场景七

### 场景概述

用户输入数字后点击运算符，数字与运算符组成表达式，结果区归零

### 场景逻辑步骤

- 1. 用户已输入数字（如结果区显示 12）
- 2. 用户点击任意四则运算符（如 +）
- 3. 上方表达式区显示该数字和运算符（如 "12 +"）
- 4. 下方结果区归零显示 0
- 5. 用户输入第二个数字（如 3）
- 6. 结果区显示新输入的数字 3

## 场景八

### 场景概述

用户连续点击不同运算符，仅保留最后点击的运算符

### 场景逻辑步骤

- 1. 用户已输入数字并点击了运算符（如表达式区显示 "12 +"，结果区显示 0）
- 2. 用户在未输入新数字时点击另一个运算符（如 ×）
- 3. 表达式区的运算符被替换为最新点击的运算符（显示 "12 ×"），结果区仍显示 0

## 场景九

### 场景概述

用户输入完整表达式后点击等号，页面计算并展示最终表达式和结果

### 场景逻辑步骤

- 1. 用户依次输入数字、运算符、数字组成完整表达式（如输入 1、+、2，结果区显示 2，表达式区显示 "1 +"）
- 2. 用户点击等号键
- 3. 页面上方表达式区显示完整的计算表达式（如 "1 + 2"）
- 4. 页面下方结果区显示计算结果（如 3.0）
- 5. 同时该计算的表达式和结果被保存到历史记录（详见计算历史记录功能）

## 场景十

### 场景概述

用户输入不完整的表达式（表达式以运算符结尾、缺少第二个操作数）后点击等号，页面不进行计算

### 场景逻辑步骤

- 1. 用户输入数字后点击运算符但未输入第二个数字（如表达式区显示 "1 ÷"，结果区显示 0）
- 2. 用户点击等号键
- 3. 页面不发生变化，表达式区仍显示 "1 ÷"，结果区仍显示 0

## 场景十一

### 场景概述

用户在计算完成后重复点击等号，不会重复计算

### 场景逻辑步骤

- 1. 用户已完成一次计算（如表达式区显示 "1 + 2"，结果区显示 3.0）
- 2. 用户再次点击等号键
- 3. 页面不发生变化，表达式和结果保持上一次计算的结果

## 场景十二

### 场景概述

用户点击 C 键删除结果区的当前输入，每次删除末尾一位

### 场景逻辑步骤

- 1. 结果区当前显示多位数字（如 1234）
- 2. 用户点击 C 键
- 3. 结果区删除末尾一位，显示 123
- 4. 用户继续点击 C 键
- 5. 结果区继续删除末尾一位，显示 12

## 场景十三

### 场景概述

用户持续点击 C 键删除至结果仅剩一位（或负号），结果区归零显示 0

### 场景逻辑步骤

- 1. 结果区当前显示单字符（如 -1）
- 2. 用户点击 C 键
- 3. 结果区归零显示 0（不会显示为空）

## 场景十四

### 场景概述

用户在表达式已含运算符时点击 C 删除结果至零，表达式中的最后一个操作数回退到结果区

### 场景逻辑步骤

- 1. 表达式区显示含运算符和操作数的表达式（如 "12 + -23 -"），结果区显示某数字（如 -1）
- 2. 用户点击 C 键，结果区被删除至 0
- 3. 由于表达式区仍有内容，表达式中的最后一个操作数（如 -23）回退到结果区显示
- 4. 表达式区去掉该操作数及其后的运算符（显示 "12 +"）

## 场景十五

### 场景概述

用户长按 C 键，触发全部清空操作

### 场景逻辑步骤

- 1. 页面当前有输入或计算内容（表达式区和结果区非初始状态）
- 2. 用户长按 C 键
- 3. 等同于点击 AC 键的效果：表达式区清空，结果区归零显示 0

## 场景十六

### 场景概述

用户点击 AC 键清空全部输入和结果

### 场景逻辑步骤

- 1. 页面当前有表达式和结果（如表达式区显示 "1 + 2"，结果区显示 3.0）
- 2. 用户点击 AC 键
- 3. 表达式区清空，结果区归零显示 0

## 场景十七

### 场景概述

用户完成计算后，数字键、小数点键、正负号键、百分号键暂时不响应，需先按运算符或清空键才能继续

### 场景逻辑步骤

- 1. 用户已完成一次计算（表达式区显示完整表达式，结果区显示计算结果，如 3.0）
- 2. 用户点击数字键
- 3. 页面无变化（表达式和结果不变）
- 4. 用户点击小数点键
- 5. 页面无变化
- 6. 此状态下用户只能点击运算符、等号或清空键进行后续操作

## 场景十八

### 场景概述

用户完成计算后点击运算符，将上次计算结果作为新表达式的起始操作数继续运算

### 场景逻辑步骤

- 1. 用户已完成一次计算（如表达式区显示 "1 + 2"，结果区显示 3.0）
- 2. 用户点击任意运算符（如 -）
- 3. 表达式区以上次结果为起始显示新表达式（如 "3.0 -"）
- 4. 结果区归零显示 0
- 5. 用户可继续输入数字进行后续运算

## 场景十九

### 场景概述

用户执行计算后结果为无穷大或非数字（无效结果）时，除 AC 键外所有按键均不响应

### 场景逻辑步骤

- 1. 用户输入会产生无效结果的表达式（如一个极大数字除以一个接近零的小数）
- 2. 用户点击等号键
- 3. 结果区显示无效结果标识（如 Infinity）
- 4. 用户点击任意按键（数字、运算符、小数点、等号等）
- 5. 页面无变化，所有按键均不响应
- 6. 用户点击 AC 键
- 7. 表达式区清空，结果区归零显示 0，恢复正常状态

## 场景二十

### 场景概述

按键区根据计算状态自动切换首个按键：未计算时显示 C（删除），计算完成后切换为 AC（全部清空）

### 场景逻辑步骤

- 1. 页面处于未计算状态（用户正在输入表达式）
- 2. 按键区左上角显示 C 键（删除当前输入）
- 3. 用户完成一次计算（点击等号得到结果）
- 4. 按键区左上角的 C 键自动切换为 AC 键（全部清空）
- 5. 用户点击 AC 键清空后，按键区恢复显示 C 键

## 场景二十一

### 场景概述

"基础计算器"功能的作用范围与不作用范围

### 场景逻辑步骤

- 1. **仅作用于**: 计算器主页面的数字键、四则运算符（+、-、×、÷）、等号、小数点、C/AC 键，以及表达式区和结果区的展示
- 2. **不作用于**: 计算历史记录页面（历史记录的查看、清空有独立入口和逻辑）、主题切换（主题切换入口在主页面顶部，但有独立的切换逻辑）
- 3. 计算器主页面是应用启动后的默认页面，也是从历史记录页面恢复计算后返回的目标页面
# 深色与浅色模式切换SPEC

计算器主页面深色模式与浅色模式切换

## 场景一

### 场景概述

用户打开应用时，页面主题初始跟随系统当前的深色或浅色设置

### 场景逻辑步骤

- 1. 用户打开应用
- 2. 应用读取系统当前的深色/浅色模式设置
- 3. 若系统当前为深色模式，则计算器主页面以深色视觉样式显示（深色背景、浅色文字、深橙色运算符按钮）
- 4. 若系统当前为浅色模式，则计算器主页面以浅色视觉样式显示（浅色背景、深色文字、深橙色运算符按钮）
- 5. 同时，系统状态栏的图标样式也跟随当前主题自动适配

## 场景二

### 场景概述

用户在计算器主页面点击主题切换按钮，页面在深色和浅色视觉样式之间切换，切换以圆形揭示动画展开

### 场景逻辑步骤

- 1. 用户在计算器主页面顶部栏看到主题切换按钮
- 2. 当前若为深色模式，按钮显示"切换到浅色"图标（太阳图标）；当前若为浅色模式，按钮显示"切换到深色"图标（月亮图标）
- 3. 用户点击主题切换按钮
- 4. 页面以用户点击位置为圆心，新主题的视觉样式以圆形逐渐展开覆盖旧主题（约半秒动画）
- 5. 动画结束后，页面完全呈现新的主题视觉样式
- 6. 主题切换按钮的图标更新为对应的反方向切换图标

## 场景三

### 场景概述

用户在输入数字或表达式的过程中切换主题，当前正在输入的数字和已有表达式保持不变

### 场景逻辑步骤

- 1. 用户在结果区已输入数字（如 123），或已组成表达式（如表达式区显示 "12 +"，结果区显示 3）
- 2. 用户点击主题切换按钮
- 3. 页面完成深色与浅色视觉样式的切换
- 4. 切换后，结果区仍显示之前输入的数字（如 123），表达式区仍显示之前的表达式（如 "12 +"），未丢失任何输入内容

## 场景四

### 场景概述

用户在完成计算后切换主题，计算结果保持不变

### 场景逻辑步骤

- 1. 用户已完成一次计算（表达式区显示完整表达式，结果区显示计算结果，如 3.0）
- 2. 用户点击主题切换按钮
- 3. 页面完成深色与浅色视觉样式的切换
- 4. 切换后，表达式区仍显示之前的完整表达式，结果区仍显示之前的计算结果（如 3.0），未丢失计算结果

## 场景五

### 场景概述

用户切换主题后，历史记录不受影响，保持完整

### 场景逻辑步骤

- 1. 用户已有若干条计算历史记录
- 2. 用户点击主题切换按钮
- 3. 页面完成深色与浅色视觉样式的切换
- 4. 切换后，用户进入历史记录页面，历史记录仍然完整存在，未因主题切换而丢失或清空

## 场景六

### 场景概述

用户多次反复切换主题，每次切换均正常生效且数据保持不变

### 场景逻辑步骤

- 1. 用户点击主题切换按钮，页面从深色切换到浅色（或从浅色切换到深色）
- 2. 用户再次点击主题切换按钮，页面切回原来的主题
- 3. 每次切换均以圆形揭示动画展开，数据（输入、表达式、结果、历史记录）始终保持不变

## 场景七

### 场景概述

"深色与浅色模式切换"功能的作用范围与不作用范围

### 场景逻辑步骤

- 1. **仅作用于**: 计算器主页面顶部栏的主题切换按钮入口，以及由此触发的全局视觉样式切换（含计算器主页面和历史记录页面）
- 2. **不作用于**: 当前正在输入的数字、已有表达式、计算结果、历史记录数据 — 这些在切换主题时不受任何影响
- 3. 主题切换入口仅在计算器主页面顶部栏，历史记录页面无主题切换按钮
- 4. 主题偏好仅在当前使用期间有效，重新打开应用后将重新跟随系统设置
# 计算历史记录SPEC

计算器历史记录的保存、查看、恢复与清空

## 场景一

### 场景概述

用户完成一次有效计算后，该计算的表达式、结果和日期被自动保存到历史记录

### 场景逻辑步骤

- 1. 用户在计算器主页面输入完整表达式并点击等号完成计算（如输入 1 + 2 = 得到结果 3.0）
- 2. 计算结果为有效数字（非无穷大或非数字）
- 3. 该计算的表达式（如 "1 + 2"）、结果（如 3.0）和当前日期被自动保存到历史记录中，用户无需进行额外操作
- 4. 若用户再次点击等号重复求值，不会重复保存同一条记录

## 场景二

### 场景概述

用户完成计算但结果为无效值（无穷大或非数字）时，该计算不被保存到历史记录

### 场景逻辑步骤

- 1. 用户输入会产生无效结果的表达式（如一个数字除以零）
- 2. 用户点击等号键
- 3. 结果区显示无效结果（如 Infinity）
- 4. 该计算不被保存到历史记录中

## 场景三

### 场景概述

用户输入不完整的表达式后点击等号，该操作不产生历史记录

### 场景逻辑步骤

- 1. 用户输入数字和运算符但缺少第二个操作数（如表达式以运算符结尾）
- 2. 用户点击等号键
- 3. 页面不进行计算，表达式和结果保持不变
- 4. 该操作不产生历史记录

## 场景四

### 场景概述

用户在计算器主页面点击历史记录入口，进入历史记录页面查看最近的计算记录

### 场景逻辑步骤

- 1. 用户在计算器主页面顶部栏点击历史记录入口按钮
- 2. 进入历史记录页面
- 3. 页面展示所有已保存的计算记录，按日期分组显示
- 4. 每条记录上方显示该计算的表达式（半透明、可水平滚动），下方显示计算结果（可水平滚动）
- 5. 每组记录下方标注日期标签（今天保存的显示"Today"，昨天保存的显示"Yesterday"，更早的显示对应日期）
- 6. 列表默认滚动到最近保存的记录位置（最近记录可见）
- 7. 所有数字以千分位逗号格式显示

## 场景五

### 场景概述

用户进入历史记录页面但没有任何历史记录时，页面展示空状态提示

### 场景逻辑步骤

- 1. 用户在计算器主页面点击历史记录入口按钮
- 2. 进入历史记录页面
- 3. 由于尚无任何已保存的计算记录（或历史记录已被清空），页面中央显示空状态提示文案
- 4. 不显示任何列表项和日期标签

## 场景六

### 场景概述

用户在历史记录页面点击某条计算记录，返回计算器主页面并恢复该记录的表达式和结果

### 场景逻辑步骤

- 1. 用户在历史记录页面浏览计算记录列表
- 2. 用户点击其中某条计算记录（如表达式 "1 + 2"、结果 3.0）
- 3. 页面返回到计算器主页面
- 4. 计算器主页面上方表达式区显示该记录的表达式（如 "1 + 2"）
- 5. 下方结果区显示该记录的结果（如 3.0）
- 6. 恢复后的计算记录不会被重复保存到历史记录中

## 场景七

### 场景概述

用户在历史记录页面点击返回按钮，直接返回计算器主页面，不恢复任何记录

### 场景逻辑步骤

- 1. 用户在历史记录页面顶部栏点击返回按钮
- 2. 页面返回到计算器主页面
- 3. 计算器主页面的表达式和结果保持用户离开时的状态，不恢复任何历史记录

## 场景八

### 场景概述

用户在历史记录页面点击清空按钮，弹出清空确认弹窗，用户点击确认后清空全部历史记录

### 场景逻辑步骤

- 1. 用户在历史记录页面顶部栏点击清空按钮
- 2. 弹出清空确认弹窗，显示标题"Clear"和提示文案"Clear history now?"
- 3. 弹窗底部显示"Cancel"和"Clear"两个按钮
- 4. 用户点击"Clear"确认按钮
- 5. 全部历史记录被清空删除
- 6. 清空完成后，历史记录页面展示空状态（无任何记录，显示空状态提示）

> [偏差] 需求要求清空后历史记录页面展示空状态，但实际代码在点击确认清空后会同时执行清空操作和返回计算器主页面的操作（即清空后直接返回主页面而非停留在历史页面展示空状态）。下次进入历史记录页面时才会看到空状态。

## 场景十

### 场景概述

"计算历史记录"功能的作用范围与不作用范围

### 场景逻辑步骤

- 1. **仅作用于**: 计算器主页面的历史记录入口按钮；历史记录页面的记录查看、点击恢复、清空操作；有效计算完成后自动保存
- 2. **不作用于**: 计算器主页面的数字输入和运算逻辑、主题切换 — 这些有独立的功能逻辑
- 3. 保存触发仅限于等号键且结果为有效数字时；查看入口仅限于计算器顶部栏历史按钮；恢复入口仅限于历史记录页面点击列表项；清空入口仅限于历史记录页面清空按钮（需经确认弹窗确认）
