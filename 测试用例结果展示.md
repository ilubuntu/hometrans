## 简要结论

1. **本地逻辑应用效果最好**：功能边界清晰、状态简单、平台依赖少，HT 可以较完整地还原功能。
2. **本地数据应用具备实验价值**：列表、详情、收藏、筛选、设置等常见流程可以迁移，但 UI 状态反馈和工程质量仍需要人工检查。
3. **复杂在线应用效果明显下降**：当应用依赖 RSS、HTTP、数据库、下载、播放、同步等真实能力时，当前流程容易生成页面原型和 mock 服务，功能闭环不足。
4. **成本随复杂度快速上升**：项目越复杂，耗时和 token 消耗越高，但投入不一定能换来等比例的功能闭环提升。
