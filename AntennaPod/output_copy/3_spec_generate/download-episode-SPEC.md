# 下载单集SPEC

用户可在单集详情页、单集列表(多选批量、侧滑)或由自动下载后台触发, 对单个或多个单集发起下载; 下载中显示进度并可取消, 下载完成后单集标记为已下载、可用于离线播放等场景。移动网络受限时先经用户确认。

## 场景一

### 场景概述

用户在单集详情页点击"下载", 在允许网络下立即开始下载

### 场景逻辑步骤

- 1. 用户进入单集详情页, 该单集未下载且不在下载中
- 2. 详情页右下次操作按钮显示为"下载"
- 3. 用户点击"下载"按钮
- 4. 若当前网络允许下载(如在 Wi-Fi 下, 或已开启移动数据下载), 立即开始下载, 按钮切换为"取消下载"并叠加下载进度环
- 5. 若该单集无媒体或已下载, 点击不产生任何效果(不重复下载)

## 场景二

### 场景概述

移动网络受限时, 用户点击"下载"弹出确认对话框

### 场景逻辑步骤

- 1. 用户在详情页点击"下载", 但当前网络为受限的移动网络(且未开启移动数据下载)
- 2. 弹出确认对话框, 标题为"移动网络下载确认", 提示当前为移动网络, 并提供三个选项: "稍后下载"(等设备条件满足再下载)、"本次允许"(本次立即下载)、"取消"
- 3. 用户选择"稍后下载": 按设备条件排队下载, 按钮切换为"取消下载"
- 4. 用户选择"本次允许": 立即开始下载, 且在随后一段约定时间内再次下载其它单集时不再弹出此对话框
- 5. 用户选择"取消"或点击对话框外部: 不开始下载, 按钮保持"下载"

## 场景三

### 场景概述

下载中显示进度环, 排队等待时显示不确定动画

### 场景逻辑步骤

- 1. 单集进入下载流程后, 详情页次操作按钮与所在列表项的次级动作区显示下载进度环
- 2. 若该单集正在实际下载(运行中), 进度环按下载百分比填充
- 3. 若该单集在下载队列中排队等待, 进度环显示为不确定状态的动画
- 4. 下载过程中, 进度随事件实时刷新(无需用户手动刷新)

## 场景四

### 场景概述

用户在下载过程中取消下载

### 场景逻辑步骤

- 1. 单集处于下载中(运行中或排队等待)
- 2. 详情页次操作按钮显示为"取消下载"
- 3. 用户点击"取消下载", 终止该单集的下载任务
- 4. 同时关闭该单集的自动下载标记(后续不会再自动下载它)
- 5. 按钮恢复为"下载", 进度环消失

## 场景五

### 场景概述

下载完成后, 单集标记为已下载并可用于离线场景

### 场景逻辑步骤

- 1. 下载任务成功完成
- 2. 该单集的媒体被标记为"已下载", 并记录下载完成时间, 持久化到数据库
- 3. 详情页与列表项的次操作按钮切换为"删除下载"(已下载状态)
- 4. 该单集此后可用于离线播放、出现在下载页等场景

## 场景六

### 场景概述

用户在单集列表中通过多选批量下载

### 场景逻辑步骤

- 1. 用户在单集列表(详情页列表/Episodes/搜索结果/下载页等)长按进入多选模式
- 2. 选中一个或多个未下载的单集
- 3. 在多选动作栏中选择"下载"
- 4. 所选单集按设备条件依次触发下载, 各自进入下载中状态(显示进度环)
- 5. 批量下载不弹出单集级的移动网络确认(由批量上下文与设备下载约束统一决定)

## 场景七

### 场景概述

用户在列表项上侧滑触发下载

### 场景逻辑步骤

- 1. 用户在支持侧滑的列表项上向"下载"方向滑动
- 2. 若该单集无媒体, 提示"无媒体", 不下载
- 3. 若该单集已下载, 提示"已下载", 不重复下载
- 4. 否则触发与详情页相同的下载流程(含移动网络确认逻辑), 进入下载中状态

## 场景八

### 场景概述

自动下载在后台触发单集下载

### 场景逻辑步骤

- 1. 当订阅源刷新发现新单集, 且对应播客设置开启了自动下载、且满足自动下载条件时
- 2. 系统在后台对满足条件的单集发起下载(无需用户主动点击)
- 3. 下载中状态同样在列表项与详情页上显示进度环
- 4. 下载完成后与手动下载一致, 标记为已下载

## 场景九

### 场景概述

"下载单集"的作用范围与不作用范围

### 场景逻辑步骤

- 1. **仅作用于** 以下入口的单集下载: 单集详情页下载按钮; 单集列表的多选批量下载; 列表项侧滑下载; 下载日志中的失败重试; 后台自动下载
- 2. **不作用于** 以下位置/关系: 收藏页(下载与收藏相互独立, 下载不改变收藏状态); 桌面小部件(不触发下载); 播放队列(下载完成本身不会自动把单集加入队列, 加入队列是独立动作)
- 3. "下载完成后出现在下载页""下载日志与失败详情"由各自独立规格约束; 本规格仅描述下载这一动作与下载中/完成/取消的状态
