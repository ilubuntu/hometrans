# OPML导入SPEC

用户可通过"添加播客页"或"设置 → 备份与恢复"中的 OPML 导入入口，或通过系统"打开方式 / 分享至应用"动作，把一个 OPML 文件中的若干播客批量加入订阅；导入成功后这些播客出现在订阅列表中。

## 场景一

### 场景概述

用户从"添加播客页"或"设置 → 备份与恢复"启动 OPML 导入，进入"OPML 导入页"

### 场景逻辑步骤

- 1. 用户进入"添加播客页"，点击"OPML 导入"按钮；或进入"设置 → 备份与恢复"，点击"OPML 导入"项
- 2. 应用调用系统文件选择器（不限定文件类型）让用户选择一个 OPML 文件
- 3. 若系统没有可用的文件选择器：应用提示"无法启动系统文件管理器"，导入流程不启动
- 4. 用户在系统文件选择器中选择一个文件后，应用打开"OPML 导入页"
- 5. "OPML 导入页"加载时显示进度条

## 场景二

### 场景概述

用户从系统"打开方式 / 分享"动作直接打开 OPML 文件

### 场景逻辑步骤

- 1. 用户在系统文件管理器、邮件客户端或浏览器中点击一个 OPML 文件或链接（mime 类型为 `text/x-opml` / `text/xml` / `application/xml` 之一）
- 2. 系统弹出"打开方式"，应用出现在候选列表中
- 3. 用户选择用本应用打开
- 4. 应用直接打开"OPML 导入页"并解析该文件
- 5. 该入口与场景一最终汇入同一"OPML 导入页"，后续步骤相同

## 场景三

### 场景概述

"OPML 导入页"解析 OPML 文件后展示可导入的播客列表

### 场景逻辑步骤

- 1. 应用读取所选文件的字节流
- 2. 应用按字节顺序标记探测字符编码；无标记时默认 UTF-8
- 3. 应用解析 OPML 文件：
  - 3.1 解析所有 outline 节点，提取每个节点的订阅源地址与标题
  - 3.2 跳过没有订阅源地址的节点
  - 3.3 当节点没有显式标题时，按"标题优于文本、文本兜底、最终回退为订阅源地址"的顺序确定显示标题
- 4. 解析成功后：
  - 4.1 应用以多选列表形式展示所有解析出的播客标题
  - 4.2 列表初始状态为"全部选中"
  - 4.3 顶部菜单提供"全选 / 取消全选"快速切换
- 5. 用户可：
  - 5.1 取消选中部分条目（仅导入所选条目）
  - 5.2 通过"全选 / 取消全选"切换
  - 5.3 点击"取消"按钮直接放弃导入并关闭页面

## 场景四

### 场景概述

OPML 文件解析失败或文件读取失败时的容错

### 场景逻辑步骤

- 1. 若用户进入"OPML 导入页"时没有任何文件传入：应用弹出"未选择文件"提示
- 2. 若读取文件流失败且错误与读取权限相关：应用申请存储读取权限；用户授权后自动重试解析；用户拒绝时弹出"需要读取权限"提示并允许再次申请或取消关闭页面
- 3. 若 OPML 文件格式不合法导致解析抛出异常：应用弹出"错误"对话框，标题为"OPML 读取错误"，正文附加原始错误详情；用户点击"确定"后关闭页面

## 场景五

### 场景概述

用户确认导入所选项，应用批量写入订阅并刷新

### 场景逻辑步骤

- 1. 用户在"OPML 导入页"列表中确认选择集合后点击"确认"按钮
- 2. 应用按如下分支处理：
  - 2.1 若"父级控制"已开启且"订阅需密码"已启用：先弹出父级控制密码对话框；用户输入正确后才进入下一步；用户取消则放弃导入
  - 2.2 否则直接进入下一步
- 3. 应用显示进度条，对每个被选中的播客：
  - 3.1 用其订阅源地址与标题构造一个新订阅
  - 3.2 把该订阅写入应用的本地订阅库（不写入单集级状态）
- 4. 全部写入完成后，应用立即对所有未刷新的订阅源发起一次刷新，以拉取每个新订阅的元数据与单集
- 5. 应用跳转到主界面（清空回退栈到主页），新订阅出现在"订阅页"中
- 6. 若导入过程中抛出异常：应用弹出 toast 显示异常消息

## 场景六

### 场景概述

"OPML 导入"功能的作用范围与不作用范围

### 场景逻辑步骤

- 1. **仅作用于** 以下入口: "添加播客页"的"OPML 导入"按钮（场景一）；"设置 → 备份与恢复"的"OPML 导入"项（场景一）；系统"打开方式 / 分享"动作（场景二，仅当文件 mime 为 `text/x-opml` / `text/xml` / `application/xml` 之一）
- 2. **仅作用于** 以下数据范围: 批量"新增订阅"（写入订阅库 + 触发订阅源刷新）；不修改任何已存在的订阅
- 3. **不作用于** 以下数据范围: 单集级状态（Queue、Favorites、Playback history、Downloads、播放位置）完全不被 OPML 导入触及；OPML 导入与"数据库导入"（整体恢复应用数据库）是两个独立功能，互不影响
- 4. **不作用于** 以下位置: 单集详情页、播放页、队列页、收件箱页、收藏页、下载页、历史页等均无 OPML 导入入口

> [偏差] 需求未提及"父级控制密码门控"和"批量选择预览"；代码默认弹出多选预览列表并允许用户取消选中部分条目，且在父级控制启用时强制先验证密码。本规格如实保留这两个行为。
