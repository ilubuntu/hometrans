# AddPodcast页面内容SPEC

Add podcast 页面展示搜索框、推荐播客网格，以及 RSS 地址、本地文件夹、Apple Podcasts、fyyd、Podcast Index、OPML 导入等多个添加入口，各入口均可点击进入对应流程

## 场景一

### 场景概述

Add podcast 页面顶部展示搜索框，用户输入关键词或 RSS 地址进行搜索或直达订阅

### 场景逻辑步骤

- 1. 用户进入 Add podcast 页面，顶部展示一个圆角搜索栏，左侧为搜索图标，右侧为输入框，提示文字"Search podcast…"
- 2. 用户在输入框输入内容并点击软键盘的搜索键，或点击搜索图标
- 3. 若输入内容为 http(s) 开头的网址，应用直接按 RSS 地址进入订阅流程
- 4. 若输入内容为非网址关键词，应用进入联合在线搜索结果页，展示匹配的播客
- 5. 搜索完成后输入框被清空，便于下一次输入

## 场景二

### 场景概述

搜索框下方展示推荐播客网格，从在线推荐榜单加载封面与标题

### 场景逻辑步骤

- 1. 搜索框下方区域展示推荐播客网格，每个网格项显示播客封面与标题
- 2. 推荐内容来自在线推荐榜单，依赖网络可达性；网络可用时自动加载
- 3. 当推荐默认配置发生变化时，推荐网格自动刷新
- 4. 用户点击某一推荐项，进入该播客的预览页（预览与订阅详见对应规格）

## 场景三

### 场景概述

用户通过"Add podcast by RSS address"入口，粘贴 RSS 地址添加播客

### 场景逻辑步骤

- 1. 用户点击列表项"Add podcast by RSS address"（左侧带 feed 图标）
- 2. 弹出对话框，标题为"Add podcast by RSS address"，输入框提示"RSS address"
- 3. 若系统粘贴板中存在 http 开头的内容，输入框自动预填该内容
- 4. 用户输入或确认 RSS 地址后点击确认
- 5. 若地址格式不合法，输入框显示"地址无效"错误提示，不关闭对话框
- 6. 若地址合法，对话框关闭，进入该 RSS 地址的订阅流程

## 场景四

### 场景概述

用户通过"Add local folder"入口，选择本地文件夹作为订阅源

### 场景逻辑步骤

- 1. 用户点击列表项"Add local folder"（左侧带文件夹图标）
- 2. 应用调起系统文件夹选择器，用户选择一个本地文件夹
- 3. 应用取得该文件夹的持久化读写权限
- 4. 应用将该文件夹作为本地订阅源进行订阅
- 5. 订阅成功后，跳转到该本地订阅源的单集列表页
- 6. 若系统无文件管理器可用，应用弹出提示"无法启动系统文件管理器"

## 场景五

### 场景概述

用户通过 Apple Podcasts、fyyd 或 Podcast Index 中的任一入口进入对应在线搜索

### 场景逻辑步骤

- 1. 用户点击列表项"Search Apple Podcasts"（或"Search fyyd"、"Search Podcast Index"，均带搜索图标）
- 2. 应用进入对应来源的在线搜索页
- 3. 用户在该搜索页输入关键词，结果仅来自所选来源
- 4. 三个入口的搜索行为一致，仅搜索数据源不同

## 场景六

### 场景概述

用户通过"Import podcast list (OPML)"入口导入 OPML 文件批量添加订阅

### 场景逻辑步骤

- 1. 用户点击列表项"Import podcast list (OPML)"（左侧带下载图标）
- 2. 应用调起系统文件选择器，文件类型为任意
- 3. 用户选择一个 OPML 文件
- 4. 应用启动 OPML 导入流程（导入细节详见对应规格）
- 5. 若系统无文件管理器可用，应用弹出提示"无法启动系统文件管理器"

## 场景七

### 场景概述

"Add podcast 页面内容"功能的作用范围与不作用范围

### 场景逻辑步骤

- 1. **仅作用于** 以下入口: Add podcast 页面内的顶部搜索框、推荐播客网格、6 个添加入口列表项
- 2. **不作用于** 以下位置: 各入口的实际订阅/搜索/导入结果（由预览页、在线搜索页、OPML 导入流程各自承接）；推荐网格与 6 个入口均不在本页内直接写入订阅状态，仅触发下游流程
- 3. 推荐网格内容依赖网络，无网络时推荐区可能为空；6 个添加入口中依赖系统文件管理器（本地文件夹、OPML）的入口在无文件管理器时给出提示

> [偏差] 需求描述推荐播客网格为静态展示，实际推荐内容来自在线推荐榜单，依赖网络且会在推荐配置变化时刷新。需求未提及顶部搜索框的"网址直达"行为，实际输入 http(s) 网址时会直接进入 RSS 订阅流程而非在线搜索。需求列举的 6 个添加入口（RSS、本地文件夹、Apple Podcasts、fyyd、Podcast Index、OPML）与代码完全一致。
