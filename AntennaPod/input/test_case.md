# AntennaPod 验收测试用例

说明：以下用例面向 HomeTrans 迁移后的 HarmonyOS 应用自测和人工验收。AntennaPod 是播客应用，核心数据来自 podcast RSS feed。为保证无人值守测试稳定，建议测试前准备一个稳定可访问的 podcast RSS feed，或准备一个固定 OPML 文件。默认 Android 包名为 `de.danoeh.antennapod`；HarmonyOS bundle name 以后续目标工程实际配置为准。

测试数据建议：
- 稳定 podcast RSS feed：使用一个公开、长期可访问、单集数量充足的 feed。
- 至少保证 feed 中有 3 个以上单集，且单集标题、描述、音频地址可解析。
- 如果网络不稳定，优先使用本地可访问的测试 RSS/OPML 服务。

### Scenario: 首次启动进入空库首页
- 覆盖需求：REQ-001, REQ-002
- 前置条件：清除应用数据后首次启动。
- 动作：打开应用。
- 预期结果：应用正常进入主界面，显示 AntennaPod 标识或首页标题；空库状态下没有崩溃，并能看到添加播客/订阅相关入口。

### Scenario: 打开主导航并切换 Queue
- 覆盖需求：REQ-002, REQ-016
- 前置条件：应用已启动。
- 动作：打开主导航或底部导航，点击 Queue。
- 预期结果：进入 Queue 页面；页面标题或高亮状态指向 Queue；空队列时显示空态。

### Scenario: 打开 Add podcast 页面
- 覆盖需求：REQ-003
- 前置条件：应用已启动。
- 动作：从首页或导航点击添加播客入口。
- 预期结果：进入 Add podcast 页面，显示搜索播客、RSS feed URL 输入、OPML 导入等入口。

### Scenario: 输入 RSS feed 并解析播客
- 覆盖需求：REQ-004, REQ-039
- 前置条件：准备稳定 podcast RSS feed。
- 动作：在 Add podcast 页面输入 RSS feed URL，点击确认/搜索。
- 预期结果：应用展示播客预览信息，包含播客标题、描述、封面或单集列表；没有停留在无限加载。

### Scenario: 确认订阅播客
- 覆盖需求：REQ-004, REQ-005
- 前置条件：RSS feed 已解析出播客预览。
- 动作：点击 Subscribe 或确认订阅按钮。
- 预期结果：订阅成功，播客进入 Subscriptions；可从订阅列表看到该播客。

### Scenario: 订阅列表进入播客详情
- 覆盖需求：REQ-005, REQ-006
- 前置条件：至少已订阅 1 个播客。
- 动作：进入 Subscriptions，点击已订阅播客。
- 预期结果：进入播客详情页，显示播客标题、描述、封面、设置入口和单集列表。

### Scenario: 播客详情滚动单集列表
- 覆盖需求：REQ-006, REQ-007
- 前置条件：处于播客详情页，feed 至少有 3 个单集。
- 动作：向下滚动单集列表。
- 预期结果：单集列表可正常滚动；每个单集展示标题、时间/日期、播放或下载相关状态。

### Scenario: 打开单集详情页
- 覆盖需求：REQ-008
- 前置条件：处于播客详情页。
- 动作：点击任意单集。
- 预期结果：进入单集详情页，展示单集标题、播客名、描述、播放、下载、加入队列、收藏等操作。

### Scenario: 单集详情返回播客详情
- 覆盖需求：REQ-006, REQ-008
- 前置条件：处于单集详情页。
- 动作：点击返回按钮。
- 预期结果：返回原播客详情页，列表位置和订阅状态保持合理。

### Scenario: 播放单集并显示迷你播放器
- 覆盖需求：REQ-009, REQ-011
- 前置条件：处于单集详情页，单集有可播放音频。
- 动作：点击 Play。
- 预期结果：播放开始；页面或底部显示当前播放单集标题；播放按钮状态变为暂停。

### Scenario: 暂停并继续播放
- 覆盖需求：REQ-011
- 前置条件：已有单集正在播放。
- 动作：点击 Pause，再点击 Play。
- 预期结果：播放状态在暂停和播放之间切换，按钮状态同步变化。

### Scenario: 打开完整播放页
- 覆盖需求：REQ-010
- 前置条件：已有单集正在播放并显示迷你播放器。
- 动作：点击迷你播放器。
- 预期结果：进入完整播放页，显示封面、标题、进度条、播放/暂停、快进、后退、播放速度等控件。

### Scenario: 播放页快进后退
- 覆盖需求：REQ-012
- 前置条件：处于完整播放页，单集正在播放或暂停。
- 动作：点击快进按钮，再点击后退按钮。
- 预期结果：播放进度或时间文本发生变化；页面不崩溃。

### Scenario: 修改播放速度
- 覆盖需求：REQ-013
- 前置条件：处于完整播放页。
- 动作：点击播放速度入口，选择一个非 1.0x 的速度。
- 预期结果：速度弹窗可打开；选择后播放速度显示更新。

### Scenario: 打开睡眠定时器
- 覆盖需求：REQ-014
- 前置条件：处于完整播放页。
- 动作：点击 Sleep timer 入口。
- 预期结果：显示睡眠定时器弹窗或设置页，可选择倒计时。

### Scenario: 将单集加入队列
- 覆盖需求：REQ-015, REQ-016
- 前置条件：处于单集详情页。
- 动作：点击 Add to queue 或更多菜单中的加入队列。
- 预期结果：提示加入成功；进入 Queue 页面可以看到该单集。

### Scenario: 队列移动单集到顶部或底部
- 覆盖需求：REQ-017
- 前置条件：Queue 中至少有 2 个单集。
- 动作：打开某个队列项更多菜单，选择 Move to top 或 Move to bottom。
- 预期结果：队列中该单集位置发生变化。

### Scenario: 清空队列
- 覆盖需求：REQ-018
- 前置条件：Queue 中至少有 1 个单集。
- 动作：点击 Queue 页面菜单中的 Clear queue 并确认。
- 预期结果：Queue 变为空态；原单集的队列标记同步移除。

### Scenario: Episodes 聚合列表展示订阅单集
- 覆盖需求：REQ-019
- 前置条件：至少已订阅 1 个播客。
- 动作：进入 Episodes 页面。
- 预期结果：页面展示来自订阅播客的单集列表；排序/筛选入口可见。

### Scenario: Episodes 筛选或排序入口
- 覆盖需求：REQ-019
- 前置条件：处于 Episodes 页面。
- 动作：打开筛选或排序菜单。
- 预期结果：弹出筛选/排序选项；选择后列表刷新或选中状态变化。

### Scenario: Inbox 查看新单集
- 覆盖需求：REQ-020
- 前置条件：订阅播客后存在未处理的新单集。
- 动作：进入 Inbox 页面。
- 预期结果：显示新单集列表；列表项可进入详情或执行更多操作。

### Scenario: 从 Inbox 移除单集
- 覆盖需求：REQ-020
- 前置条件：Inbox 中至少有 1 个单集。
- 动作：对某个单集执行 Remove from inbox。
- 预期结果：该单集从 Inbox 列表移除，但不从播客单集列表删除。

### Scenario: 收藏单集
- 覆盖需求：REQ-021
- 前置条件：处于单集详情页。
- 动作：点击 Favorite 或星标/收藏按钮。
- 预期结果：收藏状态变为已收藏；进入 Favorites 页面可看到该单集。

### Scenario: 取消收藏单集
- 覆盖需求：REQ-021
- 前置条件：Favorites 页面已有 1 个收藏单集。
- 动作：打开该单集并取消收藏，或在列表菜单中取消收藏。
- 预期结果：该单集从 Favorites 页面移除；列表为空时显示空态。

### Scenario: 播放历史记录
- 覆盖需求：REQ-022
- 前置条件：至少播放过一个单集。
- 动作：进入 Playback History 页面。
- 预期结果：页面显示刚播放过的单集。

### Scenario: 清空播放历史
- 覆盖需求：REQ-022
- 前置条件：Playback History 中至少有 1 条记录。
- 动作：点击清空历史并确认。
- 预期结果：播放历史变为空态。

### Scenario: 下载单集
- 覆盖需求：REQ-025, REQ-023
- 前置条件：处于单集详情页，网络可用。
- 动作：点击 Download。
- 预期结果：下载状态显示进行中并最终完成；Completed Downloads 页面出现该单集。

### Scenario: 删除已下载单集
- 覆盖需求：REQ-026
- 前置条件：已有下载完成的单集。
- 动作：在单集详情或 Completed Downloads 中删除下载。
- 预期结果：本地下载被删除；单集仍保留在播客列表中。

### Scenario: 查看下载日志
- 覆盖需求：REQ-024
- 前置条件：执行过下载或制造过一次下载失败。
- 动作：进入 Download Log 页面。
- 预期结果：列表显示下载记录；点击失败记录可看到详细错误。

### Scenario: 应用内搜索单集
- 覆盖需求：REQ-027
- 前置条件：至少已订阅 1 个播客。
- 动作：进入 Search，输入某个已知单集标题关键词。
- 预期结果：搜索结果展示匹配单集；点击结果进入单集详情。

### Scenario: 播客内搜索单集
- 覆盖需求：REQ-028
- 前置条件：处于某个播客详情页。
- 动作：点击该页搜索入口，输入该播客内某个单集关键词。
- 预期结果：结果只来自当前播客；点击结果进入对应单集详情。

### Scenario: 打开播客设置
- 覆盖需求：REQ-029
- 前置条件：处于播客详情页。
- 动作：点击播客设置入口。
- 预期结果：进入该播客设置页，显示自动下载、新单集处理、标签、播放速度等设置。

### Scenario: 修改播客标签
- 覆盖需求：REQ-030
- 前置条件：处于播客设置页。
- 动作：打开标签设置，为播客添加一个标签并保存。
- 预期结果：返回订阅列表后可看到或通过筛选使用该标签。

### Scenario: 导入 OPML
- 覆盖需求：REQ-031
- 前置条件：准备固定 OPML 文件。
- 动作：从 Add podcast 或 Settings 的 Import/Export 入口选择 OPML 文件导入。
- 预期结果：OPML 中的播客被加入订阅列表。

### Scenario: 导出 OPML
- 覆盖需求：REQ-032
- 前置条件：至少已订阅 1 个播客。
- 动作：进入 Settings -> Import/Export，点击导出订阅。
- 预期结果：触发文件保存流程或导出确认流程。

### Scenario: 打开外观设置
- 覆盖需求：REQ-033
- 前置条件：应用已启动。
- 动作：进入 Settings，打开 User interface 或 Appearance 相关设置。
- 预期结果：可看到主题、导航或界面相关配置。

### Scenario: 切换主题
- 覆盖需求：REQ-033, REQ-040
- 前置条件：处于外观设置页。
- 动作：切换到深色或浅色主题。
- 预期结果：应用主题发生变化；重启应用后选择保持。

### Scenario: 修改播放设置
- 覆盖需求：REQ-034
- 前置条件：进入 Settings。
- 动作：打开 Playback 设置，修改快进或后退秒数。
- 预期结果：设置保存；播放页快进/后退显示或行为跟随变化。

### Scenario: 修改下载设置
- 覆盖需求：REQ-035
- 前置条件：进入 Settings。
- 动作：打开 Downloads 设置，切换移动网络下载或自动下载相关开关。
- 预期结果：开关状态更新并保存。

### Scenario: 同步设置未配置不影响本地功能
- 覆盖需求：REQ-036
- 前置条件：未登录任何同步服务。
- 动作：进入同步设置页后返回，继续打开订阅和播放页面。
- 预期结果：同步未配置不影响本地订阅、队列和播放功能。

### Scenario: 查看统计页面
- 覆盖需求：REQ-037
- 前置条件：至少播放过一个单集。
- 动作：进入 Statistics 页面。
- 预期结果：页面展示收听统计或播放时间相关信息。

### Scenario: 分享单集
- 覆盖需求：REQ-038
- 前置条件：处于单集详情页。
- 动作：点击 Share。
- 预期结果：打开系统分享面板或等效分享弹窗；取消后返回单集详情。

### Scenario: 输入无效 RSS feed 显示错误
- 覆盖需求：REQ-039
- 前置条件：处于 Add podcast 页面。
- 动作：输入无效 URL 或不可解析的 RSS 地址并确认。
- 预期结果：页面显示错误信息和返回/重试入口，不崩溃。

### Scenario: 重启后数据保持
- 覆盖需求：REQ-040
- 前置条件：已订阅播客、收藏单集、加入队列并修改过一个设置。
- 动作：关闭并重新打开应用。
- 预期结果：订阅、收藏、队列和设置仍然存在。
