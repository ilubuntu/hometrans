REQ-001 首次启动与空库首页
应用首次启动后进入 Home 页面。空库状态下应显示 Home 标题、搜索入口、更多菜单入口、底部导航，以及引导用户添加播客或管理订阅的空状态内容。
步骤: 打开应用 -> 进入 Home -> 观察页面标题、底部导航和空状态提示。
验收重点: 应用可正常启动；Home 不空白、不崩溃；能看到添加播客、Subscriptions 或 More 菜单等入口。

REQ-002 底部导航基础结构
应用底部导航包含 Home、Queue、Inbox、Subscriptions 和 More。Home、Queue、Inbox、Subscriptions 是可切换的主要页面；More 是溢出菜单入口，不是独立内容页。
步骤: 分别点击 Home、Queue、Inbox、Subscriptions -> 再点击 More。
验收重点: 四个主页面切换后标题和选中状态正确；点击 More 时在当前页面上弹出菜单，不应进入一个只显示 More 的空页面。

REQ-003 More 溢出菜单
用户点击底部 More 后，应在当前页面上弹出菜单列表。菜单至少包含 Episodes、Downloads、Playback history、Favorites、Statistics、Add podcast、Customize navigation、Settings 等入口。
步骤: 进入任意主页面 -> 点击底部 More -> 查看弹出的菜单项。
验收重点: 菜单覆盖在当前页面之上；当前页面内容不被替换；菜单项文字和图标清晰可见。

REQ-004 More 菜单跳转设置页
用户可通过 More 菜单进入 Settings 页面。进入 Settings 后页面有返回按钮、标题、搜索框和设置分类列表。
步骤: 点击底部 More -> 点击 Settings -> 进入 Settings 页面。
验收重点: Settings 页面可打开；展示 User interface、Playback、Downloads、Synchronization、Backup & restore、Notifications 等设置分类；点击返回可回到原页面。

REQ-005 Add podcast 页面入口
用户可以从 Subscriptions 页的加号按钮、Home 的添加入口或 More 菜单中的 Add podcast 进入 Add podcast 页面。
步骤: 进入 Subscriptions -> 点击右下角加号；或点击 More -> Add podcast。
验收重点: 成功进入 Add podcast 页面；页面标题为 Add podcast；有返回按钮。

REQ-006 Add podcast 页面内容
Add podcast 页面应展示搜索框、推荐播客网格和多个添加入口，包括按 RSS 地址添加、本地文件夹、Apple Podcasts 搜索、fyyd 搜索、Podcast Index 搜索、OPML 导入。
步骤: 进入 Add podcast 页面 -> 查看页面内容 -> 向下滚动。
验收重点: 搜索框可见；推荐播客封面以网格展示；RSS、搜索、OPML 等入口可见且可点击。

REQ-007 播客预览与订阅
用户在 Add podcast 页面选择推荐播客或搜索结果后，应打开播客预览页或弹层，展示播客封面、标题、作者、描述、Subscribe 按钮和 Episodes preview 列表。
步骤: 进入 Add podcast -> 点击一个推荐播客 -> 查看预览内容 -> 点击 Subscribe。
验收重点: 预览页内容完整；Subscribe 按钮可点击；订阅过程有明确结果反馈。

REQ-008 订阅成功后的数据刷新
用户订阅播客成功后，Subscriptions 页面应出现该播客；Home 页面应从空库状态更新为有内容状态，展示订阅相关推荐或单集摘要。
步骤: 完成一次 Subscribe -> 返回 Subscriptions -> 返回 Home。
验收重点: Subscriptions 中出现播客卡片；Home 显示 Continue listening、See what's new、Get surprised、Check your classics 或类似分区；数据状态与订阅结果一致。

REQ-009 Subscriptions 页面
Subscriptions 页面展示用户已订阅的播客列表。每个播客项展示封面、标题或基础状态，支持点击进入播客详情。
步骤: 已完成至少一个订阅 -> 点击底部 Subscriptions -> 点击播客卡片。
验收重点: 已订阅播客可见；点击后进入对应播客详情；空订阅时显示合理空态和添加入口。

REQ-010 播客详情页
播客详情页展示播客标题、封面、描述、订阅状态、设置或更多入口，以及该播客的单集列表。
步骤: 进入 Subscriptions -> 点击一个播客 -> 进入播客详情页。
验收重点: 详情信息完整；单集列表可滚动；返回后仍回到原来的订阅上下文。

REQ-011 单集列表项
播客详情页或 Episodes 页面中的单集列表项应展示封面、标题、日期、大小或时长、播放/下载/队列等状态，并支持点击进入单集详情。
步骤: 进入播客详情 -> 查看单集列表 -> 点击一个单集。
验收重点: 单集列表信息清晰；列表可滚动；点击单集进入详情页。

REQ-012 单集详情页
单集详情页展示播客名、单集标题、封面或缩略图、发布时间、时长、描述，以及 Stream/Play、Download、Cancel download、分享、更多等操作。
步骤: 从单集列表点击单集 -> 进入单集详情页。
验收重点: 单集详情内容完整；操作按钮可见；长文本描述可滚动；返回后列表上下文保持。

REQ-013 下载单集
用户可以在单集详情或列表中下载单集。下载中应显示进行中或 Cancel download 状态，下载完成后状态更新。
步骤: 进入单集详情 -> 点击 Download -> 等待状态变化。
验收重点: 下载按钮状态正确变化；下载过程中可取消；下载完成后该单集可用于离线或队列场景。

REQ-014 Queue 数据来源
Queue 页面数据来自用户对单集执行下载、加入队列或类似排队操作。订阅播客后进入播客详情，选择单集下载或加入队列，Queue 应出现对应单集。
步骤: 完成订阅 -> 进入播客详情 -> 打开单集详情 -> 下载或加入队列 -> 点击底部 Queue。
验收重点: Queue 中出现被操作的单集；数量和剩余时长更新；不是只显示空态或占位文本。

REQ-015 Queue 页面展示
Queue 页面展示待播放单集列表，顶部显示总数量和剩余播放时长。每个队列项展示拖拽手柄、封面、日期/大小、标题、时长和移除按钮。
步骤: Queue 中已有至少一个单集 -> 进入 Queue 页面。
验收重点: 顶部统计正确；队列项内容完整；移除按钮和拖拽手柄可见。

REQ-016 Queue 移除单集
用户可以从 Queue 移除某个单集。移除后列表数量和剩余时长同步更新。
步骤: 进入 Queue -> 点击某个队列项的移除按钮。
验收重点: 被移除单集从 Queue 消失；顶部统计更新；其他单集顺序保持合理。

REQ-017 Queue 排序与移动
用户可以通过拖拽或菜单调整 Queue 中单集顺序，也可以将单集移动到顶部或底部。
步骤: Queue 中至少有两个单集 -> 拖拽或使用菜单移动一个单集。
验收重点: 操作后列表顺序变化；刷新或切换页面后顺序保持。

REQ-018 清空 Queue
用户可以清空 Queue。清空后 Queue 显示空态，原单集仍保留在播客详情或 Episodes 中。
步骤: Queue 中已有单集 -> 打开 Queue 菜单 -> 选择 Clear queue 并确认。
验收重点: Queue 为空；相关队列标记同步移除；不会删除订阅或单集元数据。

REQ-019 Episodes 总列表
Episodes 页面展示所有订阅播客中的单集，支持排序、筛选和批量操作。Episodes 入口通过 More 菜单进入。
步骤: 已订阅至少一个播客 -> 点击 More -> Episodes。
验收重点: 页面展示订阅来源的单集；筛选或排序入口可见；单集可进入详情。

REQ-020 Inbox 数据含义
Inbox 展示订阅源刷新后发现的新单集或未处理单集。它不是所有单集列表，也不是下载列表、收藏列表或播放队列。
步骤: 订阅播客并刷新 feed -> 点击底部 Inbox。
验收重点: 有新单集时显示新单集列表；没有新单集时显示合理空态；页面不应误显示 Queue、Downloads 或 Favorites 数据。

REQ-021 Inbox 单集处理
用户可以从 Inbox 查看新单集，并将单集从 Inbox 移除。移除后该单集不再显示在 Inbox，但仍保留在播客详情或 Episodes 中。
步骤: Inbox 中存在单集 -> 对某个单集执行 Remove from inbox。
验收重点: 单集从 Inbox 消失；不会删除原始单集；切换页面后状态保持。

REQ-022 Favorites 收藏入口
用户可以在单集详情或播放页点击星标收藏单集。收藏后该单集出现在 Favorites 页面。
步骤: 打开单集详情或播放页 -> 点击星标 -> 点击 More -> Favorites。
验收重点: 收藏状态切换正确；Favorites 页面显示收藏单集；再次取消后从 Favorites 移除。

REQ-023 Favorites 页面
Favorites 通过 More 菜单进入。没有收藏时显示 No favorites 空态；有收藏时显示收藏单集列表。
步骤: 点击 More -> Favorites。
验收重点: 页面标题为 Favorites；空态内容正确；有数据时列表可见并可进入单集详情。

REQ-024 播放单集
用户点击单集的 Stream/Play 后进入播放状态，并可打开完整播放页。播放页面向播客语音单集，展示封面、播客名、日期、标题和播放控制。
步骤: 打开单集详情 -> 点击 Stream 或 Play -> 打开播放页。
验收重点: 播放页展示当前单集信息；播放/暂停按钮可见；底部导航状态保持。

REQ-025 播放页控制
播放页支持播放/暂停、后退、快进、播放速度、进度条、睡眠定时器、收藏、分享、更多菜单等操作。
步骤: 进入播放页 -> 点击播放/暂停 -> 点击后退和快进 -> 打开播放速度或睡眠定时器。
验收重点: 控件状态有反馈；进度或时间文本合理变化；页面不崩溃。

REQ-026 播放页 Show notes
播放页提供 Shownotes 入口，用于查看当前单集说明。点击后应展示或展开单集说明内容。
步骤: 进入播放页 -> 点击 Shownotes。
验收重点: 当前单集说明可打开；返回播放页后播放状态保持。

REQ-027 Playback history
播放过的单集应进入 Playback history。Playback history 通过 More 菜单进入。
步骤: 播放一个单集 -> 返回主界面 -> 点击 More -> Playback history。
验收重点: 播放过的单集出现在历史列表；可从历史进入单集详情；支持清空历史。

REQ-028 Downloads 页面
Downloads 通过 More 菜单进入，用于查看下载中或已下载的单集。已下载单集可播放或删除下载。
步骤: 下载一个单集 -> 点击 More -> Downloads。
验收重点: 下载单集出现在 Downloads；下载状态正确；删除下载后列表更新。

REQ-029 Download log
下载失败或下载完成后，应能查看下载日志和失败详情。失败记录应提供可理解的错误信息。
步骤: 触发一次下载成功或失败 -> 进入下载日志入口。
验收重点: 日志列表展示记录；失败详情可打开；错误状态不导致应用崩溃。

REQ-030 应用内搜索
用户可以通过顶部搜索入口搜索播客或单集。输入关键词后应显示匹配结果，点击结果进入对应详情。
步骤: 点击顶部搜索图标 -> 输入关键词 -> 查看结果 -> 点击结果。
验收重点: 搜索框可输入；结果按关键词刷新；结果可进入播客或单集详情。

REQ-031 播客内搜索
在某个播客详情页中，用户可以搜索当前播客内的单集，搜索结果范围限制在该播客。
步骤: 进入播客详情 -> 点击该页搜索入口 -> 输入单集关键词。
验收重点: 搜索结果只来自当前播客；点击结果进入对应单集详情。

REQ-032 播客设置
每个订阅播客应有独立设置页，支持新单集处理方式、自动下载、播放速度、标签、认证、源地址等设置。
步骤: 进入播客详情 -> 打开设置或更多入口 -> 修改一个设置后返回。
验收重点: 设置页可打开；修改后保存；返回播客详情后状态合理。

REQ-033 标签与分组
用户可以给订阅播客设置标签，并在订阅列表中查看或筛选标签分组。
步骤: 打开播客设置 -> 添加标签 -> 返回 Subscriptions。
验收重点: 标签保存成功；订阅列表可以展示或使用该标签。

REQ-034 OPML 导入
用户可以从 Add podcast 或 Settings 的备份/导入入口导入 OPML 文件，批量添加订阅。
步骤: 打开 Add podcast 或 Settings -> 选择 Import podcast list (OPML) -> 选择文件。
验收重点: 导入流程可启动；导入成功后多个播客出现在 Subscriptions。

REQ-035 OPML 导出
用户可以从 Settings 的备份/导出入口导出当前订阅列表为 OPML。
步骤: 点击 More -> Settings -> Backup & restore -> 导出订阅。
验收重点: 导出流程可启动；能进入文件保存或确认流程；不会破坏当前订阅。

REQ-036 Settings 页面分类
Settings 页面展示主要设置分类，包括 User interface、Playback、Downloads、Synchronization、Backup & restore、Notifications、Parental Controls 以及项目相关入口。
步骤: 点击 More -> Settings -> 浏览设置列表。
验收重点: 分类列表完整；页面可滚动；点击分类可进入二级设置页。

REQ-037 外观设置
User interface 或 Appearance 设置支持主题、订阅显示、锁屏或导航相关配置。修改后界面应立即或重启后生效。
步骤: Settings -> User interface -> 修改一个外观选项。
验收重点: 设置项可打开和保存；界面状态或下次启动状态与选择一致。

REQ-038 播放设置
Playback 设置支持耳机控制、快进/后退秒数、播放完成行为、队列相关选项等。
步骤: Settings -> Playback -> 修改快进或后退秒数。
验收重点: 设置保存成功；播放页对应控制或行为跟随变化。

REQ-039 下载设置与同步设置
Downloads 设置支持更新间隔、移动网络、自动下载、自动删除等选项；Synchronization 设置在未配置账号时显示配置入口，且不影响本地订阅和播放。
步骤: Settings -> Downloads 修改一个开关；Settings -> Synchronization 查看未配置状态。
验收重点: 开关状态可更新并保存；同步未配置时本地功能仍可使用。

REQ-040 数据持久化
订阅、Queue、Favorites、Playback history、Downloads 状态和用户设置应持久化。应用关闭并重新打开后，主要数据和设置仍存在。
步骤: 完成订阅、加入 Queue、收藏或修改设置 -> 关闭应用 -> 重新打开。
验收重点: 订阅仍在；Queue/Favorites/History/设置按预期保持；不会回到错误的全空状态。
