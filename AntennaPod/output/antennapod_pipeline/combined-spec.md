# SPEC: REQ-003 添加播客入口

## 需求概述

用户可以从首页或主导航进入"添加播客"页面。该页面应提供完整的播客添加入口，包括：播客搜索（支持多个搜索源）、RSS Feed URL 手动输入、OPML 文件导入、以及本地文件夹添加。各入口布局清晰、可见可操作。

## 原子场景

### 场景 3.1：进入添加播客页面（Happy Path）
**触发**: 用户通过主导航或首页入口进入添加播客页面
**执行步骤**:
1. 用户点击主导航中的"添加播客"入口，或空库状态的引导入口，或订阅列表的浮动添加按钮
2. 加载添加播客页面
3. 页面显示以下入口区域：
   - 搜索输入框（统一搜索）
   - 多个搜索平台快捷按钮
   - RSS URL 输入按钮
   - OPML 导入按钮
   - 本地文件夹添加按钮

### 场景 3.2：统一搜索 - 关键词搜索（Happy Path）
**触发**: 用户在搜索输入框输入关键词并触发搜索
**执行步骤**:
1. 用户在搜索输入框输入播客名称或关键词
2. 点击搜索按钮或按下搜索键
3. 判断输入是否为 URL 格式：
   - 若为 URL（以 http:// 或 https:// 开头）→ 直接进入 RSS 订阅流程
   - 若非 URL → 进入联合搜索结果页面
4. 联合搜索聚合多个搜索源的结果
5. 隐藏键盘，清除搜索框焦点

### 场景 3.3：搜索平台快捷入口（Happy Path）
**触发**: 用户点击某个搜索平台按钮
**执行步骤**:
1. 页面提供多个搜索平台入口（如 iTunes、Fyyd、Podcast Index）
2. 用户点击某个平台按钮
3. 进入该平台的搜索页面
4. 用户可输入关键词在该平台搜索播客

### 场景 3.4：RSS URL 输入 - 正常流程（Happy Path）
**触发**: 用户点击"按 URL 添加"按钮
**执行步骤**:
1. 弹出 URL 输入对话框
2. 对话框提示"RSS 地址"
3. 检查系统剪贴板：
   - 若剪贴板内容以 "http" 开头 → 自动预填到输入框
4. 用户输入或确认 URL
5. 点击确认按钮
6. 校验 URL 格式：
   - 格式合法 → 关闭对话框，进入 RSS Feed 预览订阅流程
   - 格式非法 → 输入框显示错误提示，不关闭对话框

### 场景 3.5：RSS URL 输入 - 格式校验失败（异常场景）
**触发**: 用户输入无效 URL 并确认
**执行步骤**:
1. 用户输入非 URL 格式的文本（如纯文字）
2. 点击确认
3. 输入框下方显示"地址无效"错误提示
4. 对话框保持打开，用户可修正后重新提交
5. 点击取消 → 关闭对话框，不执行任何操作

### 场景 3.6：OPML 文件导入入口（Happy Path）
**触发**: 用户点击 OPML 导入按钮
**执行步骤**:
1. 点击 OPML 导入按钮
2. 调用系统文件选择器
3. 用户选择 OPML 文件
4. 进入 OPML 导入流程页面

### 场景 3.7：OPML 导入 - 文件选择器不可用（异常场景）
**触发**: 系统没有可用的文件选择器
**执行步骤**:
1. 点击 OPML 导入按钮
2. 系统无法启动文件选择器
3. 显示错误提示消息（"无法启动系统文件管理器"）
4. 页面不崩溃

### 场景 3.8：本地文件夹添加入口（Happy Path）
**触发**: 用户点击添加本地文件夹按钮
**执行步骤**:
1. 点击本地文件夹按钮
2. 调用系统文件夹选择器
3. 用户选择一个文件夹
4. 系统获取该文件夹的持久访问权限
5. 将文件夹创建为"本地播客"并保存到数据库
6. 触发该本地播客的媒体文件扫描
7. 成功后导航到该播客的详情页

### 场景 3.9：本地文件夹添加失败（异常场景）
**触发**: 本地文件夹添加过程中出错
**执行步骤**:
1. 文件夹选择或访问失败
2. 显示错误提示消息
3. 页面不崩溃，用户可重试

### 场景 3.10：从订阅列表浮动按钮进入（Happy Path）
**触发**: 用户在订阅列表页面点击浮动添加按钮
**执行步骤**:
1. 用户在订阅列表页面看到浮动添加按钮
2. 点击按钮
3. 进入添加播客页面
4. 页面布局与从导航进入时一致

## 验收要点
- 添加播客页面布局完整，所有入口可见
- 搜索框可输入并触发搜索（关键词或 URL 自动识别）
- 多个搜索平台快捷按钮可点击
- RSS URL 输入支持剪贴板预填和格式校验
- OPML 导入入口可触发系统文件选择器
- 本地文件夹添加功能正常
- 文件选择器不可用时显示友好错误提示，不崩溃

# REQ-015 加入播放队列 - 需求规格

## 概述
用户可以将剧集加入播放队列。加入队列的操作可以从剧集列表的上下文菜单、多选批量操作、或滑动操作中触发。加入后，剧集会根据用户配置的插入位置添加到队列中，队列页面实时显示更新。

## 原子场景

### 场景 1：从剧集上下文菜单加入队列（正常流程）
**触发**：用户在剧集列表中长按某剧集，选择"加入队列"

**执行步骤**：
1. 检查该剧集是否已有媒体且不在队列中
2. 若已在队列中：菜单项不可见
3. 若无媒体：菜单项不可见
4. 执行加入队列操作：
   a. 读取当前播放队列
   b. 根据用户配置的插入位置（队列顶部/底部/当前播放项之后）计算插入索引
   c. 将剧集插入到计算的位置
   d. 为剧集标记"队列"标签
   e. 若剧集是新状态，将其标记为未播放
   f. 若开启了队列自动排序，重新排序队列
5. 保存队列到数据库
6. 发送队列变更事件，队列页面实时更新列表
7. 剧集列表中的队列标记状态更新

### 场景 2：通过滑动操作加入队列
**触发**：用户在剧集列表中向左/右滑动某剧集

**执行步骤**：
1. 检测到滑动操作配置为"加入队列"
2. 若该剧集已在队列中：执行从队列移除操作
3. 若该剧集无媒体：显示"无媒体"提示消息
4. 否则：执行加入队列操作（同场景 1 步骤 4-7）

### 场景 3：多选批量加入队列
**触发**：用户在多选模式下选中多个剧集，点击"加入队列"操作

**执行步骤**：
1. 从选中的剧集中过滤出有媒体且不在队列中的
2. 批量执行加入队列操作
3. 计算每个剧集的插入位置并依次插入
4. 保存队列到数据库
5. 显示提示消息："已添加 N 集到队列"
6. 队列页面实时更新

### 场景 4：队列开启自动排序时加入
**触发**：用户开启了队列"保持排序"，然后加入新剧集

**执行步骤**：
1. 将剧集添加到队列
2. 根据排序规则（如日期、标题等）自动重新排序整个队列
3. 发送排序事件，队列页面整体刷新
4. 新添加的剧集显示在排序后的正确位置

### 场景 5：重复加入已在队列中的剧集
**触发**：用户尝试将已在队列中的剧集再次加入

**执行步骤**：
1. 检测到该剧集已标记"队列"标签
2. 跳过该剧集，不做任何修改
3. 剧集在队列中的位置不变

# REQ-018 清空队列 - 需求规格

## 概述
用户可以一键清空整个播放队列。清空操作需要二次确认以防误操作。清空后队列页面显示空状态，所有剧集的队列标记被移除，信息栏更新为零。

## 原子场景

### 场景 1：清空队列（正常流程）
**触发**：用户在队列页面点击"清空队列"按钮并确认

**执行步骤**：
1. 弹出确认对话框，显示确认消息
2. 用户点击确认按钮：
   a. 清空数据库中的所有队列记录
   b. 发送队列清空事件
3. 队列页面接收事件：
   a. 清空内存中的队列列表
   b. 列表适配器更新为空列表
   c. 刷新工具栏状态
   d. 刷新信息栏
4. 显示空状态视图（图标 + 标题 + 提示消息）
5. 信息栏显示："0 集 • 剩余 0m"
6. 拖拽排序状态更新

### 场景 2：取消清空操作
**触发**：用户在确认对话框中点击取消

**执行步骤**：
1. 弹出确认对话框
2. 用户点击取消按钮或点击对话框外部
3. 对话框关闭
4. 队列保持不变

### 场景 3：清空队列后的剧集列表状态
**触发**：队列被清空后查看剧集列表

**执行步骤**：
1. 剧集列表在下次加载/刷新时重新查询队列状态
2. 之前在队列中的剧集不再显示"队列"标记
3. 剧集的"加入队列"操作按钮重新变为可用
4. 滑动操作恢复为"加入队列"（而非"从队列移除"）

### 场景 4：清空队列后当前播放不受影响
**触发**：清空队列时有剧集正在播放

**执行步骤**：
1. 清空队列操作不影响当前正在播放的媒体
2. 当前播放继续进行
3. 迷你播放器和播放详情页正常显示
4. 当前剧集播放完毕后，不会自动播放下一集（队列已空）
5. 播放服务根据空队列状态决定后续行为

### 场景 5：清空空队列
**触发**：队列为空时用户点击"清空队列"

**执行步骤**：
1. 弹出确认对话框
2. 用户确认后执行清空操作
3. 由于队列已为空，操作为空操作（无实际影响）
4. 队列页面保持空状态显示

### 场景 6：清空队列后从收件箱恢复
**触发**：清空队列后，收件箱中有新剧集

**执行步骤**：
1. 空状态视图检测到收件箱有新剧集
2. 显示提示消息："收件箱中有新剧集"
3. 显示"前往收件箱"按钮
4. 用户点击按钮跳转到收件箱页面
5. 用户可从收件箱重新加入队列

# 已下载单集SPEC

已下载页集中展示所有下载完成的单集（含已退订节目中的下载），并在顶部展示正在下载的单集，支持按多维度排序、查看下载日志、删除已播下载、单项播放或删除，以及多选批量操作。

## 场景一

### 场景概述

用户进入"已下载"页，系统一次性加载并展示所有已下载完成的单集，并在顶部展示正在下载的单集。

### 场景逻辑步骤

- 1. 用户通过抽屉导航中的"下载"入口进入"已下载页"
- 2. 系统从数据库中读取所有标记为已下载的单集（包含已退订节目中的下载），按当前保存的排序方式一次性全部加载
- 3. 同时读取当前正在下载的单集，置于列表顶部
- 4. 在列表中展示已下载单集；当没有任何已下载单集时，展示"暂无已下载内容"的空状态文案
- 5. 每个已下载单集项展示其下载状态与操作按钮

## 场景二

### 场景概述

用户通过"排序"入口选择已下载单集的排序维度（日期、时长、标题或大小）及升降序，选择后持久化并刷新列表。

### 场景逻辑步骤

- 1. 用户在"已下载页"顶部工具栏点击"排序"
- 2. 弹出排序选择面板，展示可选维度：日期（旧→新 / 新→旧）、时长（短→长 / 长→短）、标题（A→Z / Z→A）、大小（小→大 / 大→小）
- 3. 用户点击某一排序选项（首次选择某维度取默认升降序，再次点击同维度切换升降序）
- 4. 系统将所选排序方式持久化保存，并触发已下载列表刷新

## 场景三

### 场景概述

用户在已下载单集项上通过操作按钮播放或删除该单集下载，按钮行为随设置而异。

### 场景逻辑步骤

- 1. 用户在"已下载页"查看某条已下载单集项
- 2. 该单集项右侧展示操作按钮；按钮行为取决于下载设置：
   - 若设置"下载按钮点击即播放"，则按钮为"播放"，点击直接开始播放该单集
   - 否则按钮为"删除"，点击删除该单集的本地下载（本地节目删除前会给出警告确认）
- 3. 用户也可点击单集进入其详情页进行播放或其他操作

## 场景四

### 场景概述

用户一键删除所有"已下载且已播"的单集下载。

### 场景逻辑步骤

- 1. 用户在"已下载页"顶部工具栏点击"删除已播下载"
- 2. 系统弹出确认对话框，提示将删除所有已下载且已播的单集
- 3. 用户点击"确认"后，系统查询所有"已下载且已播"的单集并删除其本地媒体文件，列表随之刷新；用户也可点击"取消"放弃操作

## 场景五

### 场景概述

用户在"已下载页"查看下载日志。

### 场景逻辑步骤

- 1. 用户在"已下载页"顶部工具栏点击"下载日志"
- 2. 系统弹出下载日志面板，展示历次下载的成功与失败记录
- 3. 用户可查看失败原因等详情后关闭日志面板

## 场景六

### 场景概述

用户在"已下载页"长按单集进入多选模式，对所选单集批量执行操作。

### 场景逻辑步骤

- 1. 用户在"已下载页"长按任一单集
- 2. 系统进入多选模式，浮出悬浮操作菜单，菜单包含可对所选单集批量执行的若干操作（删除、下载、标记已播/未播、加入/移出队列、加入/取消收藏、分享、重置进度等）
- 3. 用户选择若干单集后点击某个操作；若未选中任何单集则提示"未选择任何项"
- 4. 系统对该批所选单集执行操作并写入数据库，完成后退出多选模式

## 场景七

### 场景概述

下载状态变化时，已下载页自动刷新。

### 场景逻辑步骤

- 1. 当某单集下载完成、开始下载或下载被删除时
- 2. 系统更新进行中下载集合与已下载列表：新完成的单集加入列表、已删除的单集移出列表、正在下载的单集显示在顶部
- 3. "已下载页"在打开或再次打开时自动反映最新状态

## 场景八

### 场景概述

"已下载"功能的作用范围与不作用范围。

### 场景逻辑步骤

- 1. **仅作用于** 以下入口进入的"已下载页"：抽屉导航中的"下载"入口
- 2. **不作用于** 以下位置：单集总列表页、收件箱页、收藏页等不限定为"已下载"，也不提供"删除已播下载""下载日志"专属入口；这些页面不展示已下载列表

# SPEC: REQ-040 数据持久化

## 需求概述

应用的所有用户数据在应用重启后必须完整保持，包括：订阅的播客列表、播放队列（含排序顺序）、收藏列表、播放历史、已下载的文件记录、各播客的偏好设置、全局用户设置（主题/播放/下载/同步）、播放位置和进度。数据存储在本地数据库中，支持完整备份导出和恢复。系统级自动备份确保数据安全。

## 原子场景

### 场景 40.1：应用重启后恢复订阅列表（Happy Path）
**触发**: 用户关闭并重新打开应用
**执行步骤**:
1. 用户上次添加了多个播客订阅
2. 用户完全关闭应用后重新打开
3. 应用从本地数据库加载所有订阅数据
4. 订阅列表完整显示所有已订阅播客
5. 每个播客的封面、标题、描述完整恢复
6. 每个播客的单集列表完整恢复

### 场景 40.2：应用重启后恢复播放队列（Happy Path）
**触发**: 重启后恢复播放队列
**执行步骤**:
1. 用户上次排好了播放队列顺序
2. 重启应用
3. 应用从数据库加载队列数据
4. 播放队列完整恢复，排序顺序与上次一致
5. 队列中的每集状态（已播/未播）保持

### 场景 40.3：应用重启后恢复播放位置（Happy Path）
**触发**: 重启后继续播放
**执行步骤**:
1. 用户上次播放到某单集的 15:30 位置
2. 播放服务定期保存播放位置到数据库
3. 重启应用
4. 应用从数据库加载该单集的播放位置
5. 用户继续播放时从 15:30 位置恢复
6. 播放进度准确

### 场景 40.4：应用重启后恢复收藏列表（Happy Path）
**触发**: 重启后查看收藏
**执行步骤**:
1. 用户之前收藏了多集单集

# 删除下载SPEC

用户可删除单集已下载的本地媒体文件，删除后该单集恢复为未下载状态，但单集本身与媒体元数据（标题、描述、下载地址等）均予保留，可再次下载。支持单项删除、批量删除、滑动删除，以及自动删除。

## 场景一

### 场景概述

用户对已下载的单集通过单集长按菜单删除其本地下载。

### 场景逻辑步骤

- 1. 用户在任意单集列表（如总列表、收件箱、收藏、已下载、节目详情、队列等）中长按某条已下载（或正在下载）的单集
- 2. 弹出单集操作菜单，其中显示"删除"项（仅当该单集已下载或下载中时可见）
- 3. 用户点击"删除"
- 4. 若该单集属于本地媒体源节目，系统弹出确认对话框，提示删除本地媒体源单集的后果，用户确认后继续；非本地节目则直接执行删除
- 5. 系统删除该单集的本地媒体文件（及关联的字幕文件），将单集标记为"未下载"，清除本地文件路径与内嵌图片信息，并写入数据库
- 6. 该单集本身与媒体元数据（标题、描述、下载地址等）予以保留，列表项更新为"未下载"状态；若设置了"删除时移出队列"，则该单集一并移出播放队列

## 场景二

### 场景概述

用户在已下载页或单集详情页通过"删除"按钮删除已下载单集。

### 场景逻辑步骤

- 1. 用户在已下载页或单集详情页查看某条已下载单集
- 2. 该单集提供"删除"按钮（仅当已下载时可见）
- 3. 用户点击"删除"
- 4. 若为本地媒体源节目，系统弹出确认对话框，用户确认后继续
- 5. 系统删除该单集的本地下载并置为"未下载"，元数据保留，列表相应更新

## 场景三

### 场景概述

用户在多选模式下批量删除所选单集的下载。

### 场景逻辑步骤

- 1. 用户在单集列表长按进入多选模式，选择若干单集
- 2. 用户点击悬浮操作菜单中的"删除"
- 3. 若所选单集中有本地媒体源节目，系统先弹出确认对话框，用户确认后继续
- 4. 系统对所选单集逐条处理：已下载的删除其本地文件并置为"未下载"；正在下载的取消其下载并移除部分文件
- 5. 系统提示"已删除 N 集"，对应单集状态更新，元数据保留

## 场景四

### 场景概述

用户通过列表项滑动操作删除已下载单集。

### 场景逻辑步骤

- 1. 用户在配置了"删除"滑动动作的列表（如已下载列表）中对某条已下载单集执行滑动
- 2. 系统对该已下载单集执行删除（含本地节目的二次确认），过程与单集长按删除一致

## 场景五

### 场景概述

单集接近播放完成时自动删除其本地下载（当自动删除开启时）。

### 场景逻辑步骤

- 1. 当"自动删除"设置开启，且某单集播放进度接近完成（接近播放完阈值）
- 2. 系统自动删除该单集的本地下载，将其置为"未下载"，元数据保留
- 3. 本地媒体源节目仅在"自动删除本地节目"也开启时才参与自动删除

## 场景六

### 场景概述

删除恰为当前正在播放的单集时，停止播放。

### 场景逻辑步骤

- 1. 用户删除的某单集恰好是当前正在播放的媒体
- 2. 系统在删除其本地文件的同时，停止当前播放
- 3. 该单集置为"未下载"状态，播放状态被清除

## 场景七

### 场景概述

存储空间不足时自动清理未受保护的已下载单集。

### 场景逻辑步骤

- 1. 当系统因存储空间不足触发已下载单集的自动清理
- 2. 系统按清理策略删除部分已下载单集的本地文件（被标记为收藏的单集受保护，不被清理）
- 3. 被清理的单集恢复为"未下载"，元数据保留，仍可再次下载

## 场景八

### 场景概述

"删除下载"功能的作用范围与不作用范围。

### 场景逻辑步骤

- 1. **仅作用于** 以下入口：任意单集列表的单集长按菜单"删除"、已下载页/单集详情页的"删除"按钮、多选批量"删除"菜单、列表项滑动删除、自动删除触发
- 2. **不作用于** 以下情况：删除下载不会删除单集本身与媒体元数据记录（单集仍保留在节目列表中，可再次下载）；未下载的单集不显示"删除"项（滑动删除对未下载项无效）

# 下载单集SPEC

用户可对未下载的单集发起下载（单集、批量或滑动），系统在后台下载媒体文件并实时展示下载进度，下载完成后将单集状态变更为"已下载"。在受限网络（如移动网络）下下载需用户确认；本地媒体源节目不提供下载。

## 场景一

### 场景概述

用户对未下载的单集通过下载按钮发起下载，当前网络允许直接下载。

### 场景逻辑步骤

- 1. 用户在任意单集列表或单集详情页，对一条未下载且非本地媒体源的单集点击"下载"按钮
- 2. 系统校验该单集媒体存在、当前未在下载且未已下载
- 3. 在当前网络允许下载（如非受限网络，或在已确认的免提示窗口内）时，系统将该单集加入下载队列并开始后台下载
- 4. 列表项以环形进度条实时展示该单集的下载进度（每秒更新一次）
- 5. 下载期间系统在通知栏展示下载进度通知
- 6. 下载完成后，系统将该单集标记为"已下载"，写入媒体文件信息（本地路径、大小、时长，并解析章节与字幕），列表项状态随之更新

## 场景二

### 场景概述

用户在受限网络（如移动网络）下发起下载时，系统弹出确认对话框让用户选择处理方式。

### 场景逻辑步骤

- 1. 用户对未下载的单集点击"下载"按钮
- 2. 系统检测当前为受限网络且下载未被允许直接进行
- 3. 系统弹出"确认移动网络下载"对话框，提供三种选择：
   - "稍后下载"：本次按受限网络约束排队等待（待符合网络条件时再下载）
   - "本次允许"：本次立即下载（忽略受限网络约束），并在约 5 分钟内对后续下载免再次提示
   - "取消"：放弃本次下载
- 4. 若用户选择"稍后下载"或"本次允许"，系统按对应方式处理下载

## 场景三

### 场景概述

用户在多选模式下批量下载所选单集。

### 场景逻辑步骤

- 1. 用户在单集列表长按进入多选模式，选择若干未下载的单集
- 2. 用户点击悬浮操作菜单中的"下载"
- 3. 系统对所选每条有媒体且未下载的单集依次加入下载队列，并提示"正在下载 N 集"
- 4. 各单集按队列依次下载，列表项分别展示各自的下载进度

## 场景四

### 场景概述

用户通过列表项滑动操作发起下载。

### 场景逻辑步骤

- 1. 用户在配置了"下载"滑动动作的单集列表中对某条未下载单集执行滑动
- 2. 系统对该单集发起下载，过程与点击下载按钮一致（含受限网络确认逻辑）

## 场景五

### 场景概述

下载进度实时展示在列表项与通知栏。

### 场景逻辑步骤

- 1. 某单集处于下载中时，其列表项右侧以环形进度条展示当前下载百分比
- 2. 若该单集仍在下载队列中排队等待（尚未真正开始传输），进度条以不确定态展示
- 3. 系统同时在通知栏展示下载进度通知，随下载推进持续更新

## 场景六

### 场景概述

下载完成后单集状态变更为"已下载"。

### 场景逻辑步骤

- 1. 某单集媒体下载完成后，系统将其标记为"已下载"
- 2. 系统写入该媒体的本地文件路径、文件大小、时长，并解析其内嵌章节与字幕（若有）
- 3. 该单集对应的自动下载标记被关闭（不再重复自动下载）
- 4. 列表中该单集项更新为"已下载"状态（环形进度置满），下载完成信息通过无障碍播报
- 5. 已下载完成后若设置了"下载后加入队列"，该单集自动加入播放队列

## 场景七

### 场景概述

下载失败或被取消时的处理。

### 场景逻辑步骤

- 1. 当某单集下载失败（如网络错误、空间不足、未授权等），系统删除已下载的部分文件，并记录一条失败下载日志（含失败原因）
- 2. 该单集恢复为未下载状态，环形进度归零
- 3. 若下载中途被取消，系统移除部分文件；若该单集是因下载而自动加入队列的，则一并将其移出队列
- 4. 失败与取消记录可在下载日志中查看

## 场景八

### 场景概述

下载受网络条件与自动触发影响。

### 场景逻辑步骤

- 1. 是否允许在移动网络下下载由设置控制：关闭时下载需非按流量计费网络；开启时任意可用网络均可下载
- 2. 当网络从不满足条件恢复为满足条件时（如从移动网络切换到无线网络），符合约束的排队下载自动恢复
- 3. 当满足自动下载条件时（如接入电源、自动下载设置开启），系统自动对符合条件的单集发起下载

## 场景九

### 场景概述

"下载单集"功能的作用范围与不作用范围。

### 场景逻辑步骤

- 1. **仅作用于** 以下入口：任意单集列表项的"下载"按钮/单集详情页下载按钮、多选批量下载菜单、列表项滑动下载、自动下载触发
- 2. **不作用于** 以下位置：本地媒体源节目不提供下载按钮（其媒体已在本地，无需下载）

# 下载日志SPEC

下载日志以底部弹层集中展示历次下载（节目源更新与单集媒体下载）的成功与失败记录，支持查看失败详情、对失败项一键重试，以及清空全部日志。每条记录包含类型、标题、完成时间与失败原因。

## 场景一

### 场景概述

用户在"已下载页"打开下载日志，系统展示历次下载的成功与失败记录。

### 场景逻辑步骤

- 1. 用户进入"已下载页"，点击工具栏中的"下载日志"
- 2. 系统以底部弹层打开"下载日志"面板，从数据库中读取全部下载记录
- 3. 在列表中按完成时间展示每条记录，每条包含：下载类型（节目源或单集媒体）、标题、相对完成时间、以及成功或失败状态
- 4. 当没有任何下载记录时，展示"暂无下载日志"的空状态文案
- 5. （当"已下载页"打开时检测到存在失败记录，系统会自动弹出下载日志面板）

## 场景二

### 场景概述

下载日志中的成功记录以成功标识展示，不提供重试与详情入口。

### 场景逻辑步骤

- 1. 用户在"下载日志"面板查看某条成功记录
- 2. 该记录以"成功"图标标识，并标注"下载成功"
- 3. 成功记录不展示失败原因、不展示"点击查看详情"提示，也不提供重试按钮

## 场景三

### 场景概述

下载日志中的失败记录以失败标识展示，并标注具体的失败原因。

### 场景逻辑步骤

- 1. 用户在"下载日志"面板查看某条失败记录
- 2. 该记录以"失败"图标标识（重复解析异常以"提示"图标标识）
- 3. 该记录展示具体的失败原因文案，如连接错误、IO 错误、存储空间不足、未知主机、未授权、禁止访问、未找到、证书错误、被拦截等
- 4. 该记录同时展示"点击查看详情"提示

## 场景四

### 场景概述

用户点击某条失败记录查看下载错误详情。

### 场景逻辑步骤

- 1. 用户在"下载日志"面板点击某条记录
- 2. 系统弹出"下载错误详情"对话框，展示该下载所属的节目名、单集名、文件下载地址、以及技术性失败原因
- 3. 若该记录为成功，则详情中展示"下载成功"
- 4. 用户可点击文件地址或技术原因将其复制到剪贴板；若能定位到对应节目，详情中提供"前往节目"按钮，点击可跳转至该节目页

## 场景五

### 场景概述

用户对失败记录一键重试下载（节目源重试更新或单集重试下载）。

### 场景逻辑步骤

- 1. 用户在"下载日志"面板查看某条失败记录
- 2. 若该文件不存在更新一次成功的记录，则该条目提供"重试"按钮
- 3. 用户点击"重试"：若为节目源下载失败，系统重新触发该节目源更新；若为单集媒体下载失败，系统重新发起该单集下载，并提示"正在下载"
- 4. 若该文件后续已有一次成功的下载记录，则不再提供重试按钮

## 场景六

### 场景概述

用户一键清空全部下载日志。

### 场景逻辑步骤

- 1. 用户在"下载日志"面板顶部点击"清空"
- 2. 系统清除全部下载日志记录，列表随即清空并展示空状态

## 场景七

### 场景概述

下载日志随新下载结果的产生自动刷新。

### 场景逻辑步骤

- 1. 当某次节目源更新或单集下载完成（无论成功、失败或取消），系统将该次结果写入下载日志
- 2. 若"下载日志"面板处于打开或再次打开状态，系统自动重新加载日志列表，最新结果出现在列表中

## 场景八

### 场景概述

"下载日志"功能的作用范围与不作用范围。

### 场景逻辑步骤

- 1. **仅作用于** 以下入口：已下载页工具栏中的"下载日志"按钮（及已下载页打开时自动弹出）
- 2. **不作用于** 以下位置：单集总列表页、收件箱页、收藏页、播放历史页均不提供下载日志入口，也不展示下载记录

**执行步骤**:
1. 弹出操作菜单/底部弹窗
2. 菜单选项包括：
   - 播放/流式播放
   - 下载/删除下载
   - 加入队列/移出队列
   - 收藏/取消收藏
   - 分享
   - 标记已播放/标记未播放
   - 访问网站
3. 选择操作后执行并关闭菜单
4. 列表项状态相应更新

### 场景 7.14：无媒体单集的列表项（边界场景）
**触发**: 某单集没有关联的媒体文件
**执行步骤**:
1. 隐藏时长、进度条、播放状态相关 UI

# 单集总列表SPEC

聚合展示用户订阅的所有节目下的全部单集，支持分页加载、按维度排序、按状态筛选、以及多选批量操作；并提供下拉刷新与单项操作入口。

## 场景一

### 场景概述

用户进入"单集"总列表页，系统聚合所有订阅节目的单集，按默认排序分页加载首屏数据并展示。

### 场景逻辑步骤

- 1. 用户通过抽屉导航中的"单集"入口，或首页"惊喜单集"卡片上的"更多"入口，进入"单集总列表页"
- 2. 系统从数据库中聚合读取所有订阅节目下的单集，按当前保存的排序方式与筛选条件，读取第 1 页数据（每页 150 条）
- 3. 同时读取满足当前筛选条件的单集总数
- 4. 在列表中展示首屏单集；当筛选条件为空时展示"暂无单集"空状态文案，当筛选条件非空且无匹配项时展示"筛选后无单集"空状态文案
- 5. 首屏加载完成后恢复上次离开时的滚动位置

## 场景二

### 场景概述

用户在"单集总列表页"向下滚动到底部时，系统自动加载下一页单集并追加到列表末尾。

### 场景逻辑步骤

- 1. 用户在"单集总列表页"持续向下滚动
- 2. 当滚动接近列表底部，且仍存在未加载的更多数据时，系统自动发起下一页加载（每页 150 条）
- 3. 在加载过程中列表末尾展示一个加载占位项
- 4. 新一页数据加载完成后追加到列表末尾并移除占位项；若返回数据不足一页（少于 150 条），则标记为已无更多数据，不再继续分页加载
- 5. 加载失败时移除占位项并记录错误，列表保持已有内容

## 场景三

### 场景概述

用户通过"排序"入口选择单集的排序维度（日期或时长）及升降序，选择后持久化并刷新列表。

### 场景逻辑步骤

- 1. 用户在"单集总列表页"顶部工具栏点击"排序"
- 2. 弹出排序选择面板，展示可选维度：日期（旧→新 / 新→旧）、时长（短→长 / 长→短）
- 3. 用户点击某一排序选项（首次选择某维度时取该维度的默认升降序，再次点击同维度则切换升降序）
- 4. 系统将所选排序方式持久化保存，并触发明细列表刷新，按新排序重新读取单集数据

## 场景四

### 场景概述

用户通过"筛选"入口按状态筛选单集，可多组叠加；更改后重置到第 1 页并刷新，支持一键重置全部筛选。

### 场景逻辑步骤

- 1. 用户在"单集总列表页"顶部工具栏点击"筛选"，或点击列表上方的筛选信息条
- 2. 弹出筛选面板，展示以下可切换的状态分组，每组为二选一或清除：已播 / 未播、暂停 / 未暂停、收藏 / 非收藏、有媒体 / 无媒体、在队列 / 不在队列、已下载 / 未下载
- 3. 用户切换任意分组的状态（可同时启用多个分组，每个分组内可选其中一项或不选）
- 4. 系统立即按当前所选筛选条件重新从数据库读取第 1 页数据，并刷新筛选信息条以反映已生效的筛选；同时更新单集总数
- 5. 若用户在筛选面板点击"重置"，则清除全部筛选条件、恢复为展示全部单集，并刷新列表

## 场景五

### 场景概述

用户在"单集总列表页"长按单集进入多选模式，对所选单集批量执行操作（如标记已播、下载等）。

### 场景逻辑步骤

- 1. 用户在"单集总列表页"长按任一单集
- 2. 系统进入多选模式，浮出悬浮操作菜单，菜单包含可对所选单集批量执行的若干操作（删除、下载、标记未播、标记已播、移出队列、加入队列、分享、移出收件箱、加入收藏、取消收藏、重置播放进度等）
- 3. 用户继续点击其他单集以追加选择（或在懒加载模式下选择后续所有未加载页）
- 4. 用户点击悬浮菜单中的某个操作
- 5. 系统对该批所选单集执行对应操作并写入数据库，完成后退出多选模式，并通过提示展示操作结果（如已加入队列 N 集）

## 场景六

### 场景概述

用户在多选模式下对大批量单集执行"标记已播"或"标记未播"时，系统进行二次确认以防误操作。

### 场景逻辑步骤

- 1. 用户在多选模式下选择了较多单集（达到 25 集或以上），或在懒加载模式下勾选了后续所有未加载页
- 2. 用户点击悬浮菜单中的"标记已播"或"标记未播"
- 3. 系统弹出二次确认对话框，提示将对大量单集执行该操作
- 4. 用户点击"确认"后系统对该批单集执行操作并写入数据库；若用户取消则不执行任何操作

## 场景七

### 场景概述

用户在多选模式下未选择任何单集即点击悬浮操作菜单中的操作时，系统提示未选择项。

### 场景逻辑步骤

- 1. 用户进入多选模式但未选中任何单集
- 2. 用户点击悬浮操作菜单中的某个操作
- 3. 系统弹出提示"未选择任何项"，不执行该操作

## 场景八

### 场景概述

用户在"单集总列表页"通过下拉刷新或工具栏"刷新"入口触发节目更新。

### 场景逻辑步骤

- 1. 用户在"单集总列表页"顶部下拉，或点击工具栏"刷新"
- 2. 系统触发节目源更新（必要时征得用户同意），并在刷新过程中展示刷新指示
- 3. 节目更新完成后，单集列表自动重新加载以反映最新数据

## 场景九

### 场景概述

"单集总列表"功能的作用范围与不作用范围。

### 场景逻辑步骤

- 1. **仅作用于** 以下入口进入的"单集总列表页"：抽屉导航中的"单集"入口、首页"惊喜单集"卡片上的"更多"入口
- 2. **不作用于** 以下位置：首页"惊喜单集"卡片本身（仅展示随机单集子集，不展示全部单集，也不提供本页的排序/筛选/批量操作）；其他节目详情页、队列页、下载页等独立页面

> [偏差] 多选悬浮菜单中"移到顶部""移到底部"两项在本页默认不展示（仅在队列等特定页面可见），故未纳入本页场景。

# SPEC: REQ-039 错误状态与重试

## 需求概述

应用在各类操作中（下载、Feed 解析、网络请求、同步）遇到错误时，应提供清晰的错误信息显示和重试机制。错误信息包含错误类型、用户可读的描述和详细技术信息。单集下载失败自动重试最多 3 次，不可恢复错误立即失败。所有错误记录到下载日志中，用户可查看历史和重试。

## 原子场景

### 场景 39.1：单集下载失败自动重试（Happy Path）
**触发**: 单集下载过程中遇到可恢复错误
**执行步骤**:
1. 应用尝试下载单集
2. 下载失败（如网络波动、HTTP 数据错误）
3. 应用显示"正在重试"提示
4. 自动重试下载（最多 3 次尝试）
5. 若重试中某次成功 → 下载完成
6. 若 3 次均失败 → 发送下载失败通知

### 场景 39.2：不可恢复错误立即失败（边界场景）
**触发**: 下载遇到不可恢复的错误
**执行步骤**:
1. 下载请求返回特定错误（禁止访问、资源不存在、未授权、连接被阻止）
2. 应用判断为不可恢复错误
3. 不执行重试，立即标记为失败
4. 发送下载失败通知
5. 记录到下载日志

### 场景 39.3：HTTP 416 范围请求错误（边界场景）
**触发**: 下载请求了无效的文件范围
**执行步骤**:
1. 服务器返回 HTTP 416（范围请求无效）
2. 应用删除已下载的部分文件
3. 从文件开头重新下载
4. 自动重试

### 场景 39.4：查看下载日志（Happy Path）
**触发**: 用户查看下载历史和错误
**执行步骤**:
1. 用户打开下载日志页面
2. 列表显示所有下载记录（成功和失败）
3. 每条记录显示：时间、单集标题、状态
4. 失败记录显示错误类型和用户可读描述
5. 用户可点击查看详细错误信息

### 场景 39.5：从下载日志重试（Happy Path）
**触发**: 用户从下载日志中重试失败的下载
**执行步骤**:
1. 用户在下载日志中找到失败记录
2. 点击重试按钮
3. 应用重新发起下载请求
4. 下载过程中正常执行重试机制

### 场景 39.6：RSS Feed 解析失败（异常场景）
**触发**: Feed 内容下载成功但无法解析
**执行步骤**:
1. Feed 内容下载完成
2. XML 解析器报解析异常
3. 记录解析错误（ERROR_PARSER_EXCEPTION）
4. 错误信息记录到下载日志
5. 用户可查看错误详情

### 场景 39.7：Feed 订阅时的网络错误（异常场景）
**触发**: 添加新订阅时网络不可达
**执行步骤**:
1. 用户输入 Feed URL 尝试订阅
2. 网络请求失败（连接超时、未知主机）
3. 显示错误对话框，提示网络错误
4. 提供重试入口
5. 用户可重试或编辑 URL

### 场景 39.8：下载空间不足（异常场景）
**触发**: 设备存储空间不足无法下载
**执行步骤**:
1. 下载时检测到存储空间不足
2. 记录 ERROR_NOT_ENOUGH_SPACE 错误
3. 显示"空间不足"错误提示
4. 建议用户清理空间或修改缓存设置
5. 记录到下载日志

### 场景 39.9：需认证的下载（异常场景）
**触发**: 下载需要认证但未提供凭证
**执行步骤**:
1. 下载请求返回 HTTP 401（未授权）
2. 记录 ERROR_UNAUTHORIZED 错误
3. 显示"需要认证"错误提示
4. 引导用户在播客设置中配置用户名密码

### 场景 39.10：证书错误（异常场景）
**触发**: 下载时 SSL 证书验证失败
**执行步骤**:
1. HTTPS 连接时证书验证失败
2. 记录 ERROR_CERTIFICATE 错误
3. 显示"证书错误"提示
4. 不自动重试，需用户检查网络环境

### 场景 39.11：同步失败（异常场景）
**触发**: 同步服务连接失败
**执行步骤**:
1. 同步任务执行失败
2. 记录同步错误状态
3. 同步设置页面显示"同步失败"
4. 显示上次同步时间
5. 用户可手动重试同步

### 场景 39.12：未知错误类型（边界场景）
**触发**: 下载遇到未映射的错误代码
**执行步骤**:
1. 错误代码不在已知映射表中
2. 显示"未知错误"提示
3. 记录到下载日志
4. 调试模式下可能抛出异常

## 验收要点
- 下载失败自动重试最多 3 次
- 不可恢复错误（禁止/不存在/未授权）立即失败不重试
- HTTP 416 错误自动删除部分文件重试
- 所有下载结果（成功/失败）记录到日志
- 每种错误有用户可读的本地化描述
- 下载日志可查看历史和详细错误信息
- 可从下载日志重试失败下载
- Feed 解析失败有清晰错误提示
- 空间不足、认证缺失、证书错误等特殊场景有专属错误提示
- 同步失败状态可见且可重试

# REQ-012 快进与后退 - 需求规格

## 概述
用户在播放过程中可以使用快进和后退按钮快速调整播放位置。快进和后退的秒数可通过长按按钮进行配置，配置后立即生效并反映在按钮旁边的秒数标签上。

## 原子场景

### 场景 1：快进播放位置（正常流程）
**触发**：用户在播放详情页点击快进按钮

**执行步骤**：
1. 读取用户配置的快进秒数（默认 30 秒）
2. 计算新位置：当前位置 + 快进秒数
3. 调用播放服务 seek 到新位置
4. 进度条和位置文字更新为新位置
5. 如果新位置越过章节边界，章节信息同步更新

### 场景 2：后退播放位置（正常流程）
**触发**：用户在播放详情页点击后退按钮

**执行步骤**：
1. 读取用户配置的后退秒数（默认 10 秒）
2. 计算新位置：当前位置 - 后退秒数（最小为 0）
3. 调用播放服务 seek 到新位置
4. 进度条和位置文字更新为新位置
5. 如果新位置回到上一个章节，章节信息同步更新

### 场景 3：配置快进秒数
**触发**：用户长按快进按钮

**执行步骤**：
1. 弹出秒数选择对话框，显示可选秒数列表（如 10、15、30、45、60 秒等）
2. 当前选中的秒数高亮显示
3. 用户选择新的秒数值
4. 将新值保存到用户偏好中
5. 快进按钮旁边的秒数标签更新为新值
6. 关闭对话框

### 场景 4：配置后退秒数
**触发**：用户长按后退按钮

**执行步骤**：
1. 弹出秒数选择对话框，显示可选秒数列表
2. 当前选中的秒数高亮显示
3. 用户选择新的秒数值
4. 将新值保存到用户偏好中
5. 后退按钮旁边的秒数标签更新为新值
6. 关闭对话框

### 场景 5：在暂停状态下快进/后退
**触发**：播放处于暂停状态时用户点击快进/后退按钮

**执行步骤**：
1. 快进/后退操作在播放和暂停状态下均可执行
2. 读取配置秒数，计算新位置
3. seek 到新位置
4. 进度条更新，播放保持暂停状态

### 场景 6：快进到接近媒体结尾
**触发**：快进位置超过媒体总时长

**执行步骤**：
1. 计算新位置超出媒体时长
2. 播放器处理到末尾的行为（可能结束播放并跳到下一集）
3. 界面根据播放状态变化更新

# 收藏单集SPEC

用户可对单集进行收藏与取消收藏，所有被收藏的单集（含已退订节目中的收藏）集中在"收藏页"展示，并支持分页加载、滑动切换、单项操作与多选批量操作。收藏项受到自动清理保护。

### 场景概述

用户进入"收藏页"，系统展示所有被标记为收藏的单集，按总列表排序分页加载首屏。

### 场景逻辑步骤

- 1. 用户通过抽屉导航中的"收藏"入口进入"收藏页"
- 2. 系统从数据库中读取所有被标记为收藏的单集（包含已退订节目中的收藏单集），按当前总列表的排序方式读取第 1 页数据（每页 150 条）
- 3. 同时读取收藏单集总数
- 4. 在列表中展示收藏单集，每项以收藏星标标识；当没有任何收藏单集时，展示"暂无收藏单集"的空状态文案

## 场景二

### 场景概述

用户在"收藏页"向下滚动到底部时，系统自动加载下一页收藏单集并追加到列表末尾。

### 场景逻辑步骤

- 1. 用户在"收藏页"持续向下滚动
- 2. 当滚动接近列表底部且仍存在未加载的更多收藏单集时，系统自动发起下一页加载（每页 150 条）
- 3. 加载过程中列表末尾展示加载占位项；加载完成后追加数据并移除占位项；若返回数据不足一页则不再继续分页加载

## 场景三

### 场景概述

用户在任意单集列表中通过单集长按菜单对单集加入或取消收藏。

### 场景逻辑步骤

- 1. 用户在任意单集列表（如总列表、收件箱、下载页、节目详情等）中长按某条单集
- 2. 弹出单集操作菜单；根据该单集是否已收藏，菜单中相应显示"加入收藏"或"取消收藏"（二者互斥，不会同时显示）
- 3. 用户点击对应项：未收藏则加入收藏、已收藏则取消收藏
- 4. 系统更新该单集的收藏状态并写入数据库，相应列表项的收藏星标即时更新（加入时显示、取消时隐藏）

## 场景四

### 场景概述

用户在任意单集列表中对单集执行滑动操作以切换其收藏状态。

### 场景逻辑步骤

- 1. 用户在配置了"收藏"滑动动作的单集列表中对某条单集执行滑动
- 2. 系统切换该单集的收藏状态：未收藏则加入收藏、已收藏则取消收藏
- 3. 该列表项的收藏星标即时更新；在收藏页或按收藏筛选时，取消收藏的项会从列表中移除

## 场景五

### 场景概述

用户在多选模式下批量将所选单集加入或取消收藏，并查看操作数量提示。

### 场景逻辑步骤

- 1. 用户在单集列表长按进入多选模式，选择若干单集
- 2. 用户点击悬浮操作菜单中的"加入收藏"或"取消收藏"
- 3. 系统对所选单集批量更新收藏状态并写入数据库，并提示本次实际加入（或取消）的单集数量
- 4. 对应列表项的收藏星标即时更新；在收藏页或按收藏筛选时列表相应增减

## 场景六

### 场景概述

收藏单集受到自动清理保护，不会因存储空间自动清理而被删除。

### 场景逻辑步骤

- 1. 当系统因存储空间不足触发已下载单集的自动清理时
- 2. 被标记为收藏的已下载单集不参与自动清理，其本地媒体文件被保留
- 3. 仅未收藏的已下载单集可能被自动清理

## 场景七

### 场景概述

"收藏"功能的作用范围与不作用范围。

### 场景逻辑步骤

- 1. **仅作用于** 以下入口：抽屉导航"收藏"页（展示收藏单集）；任意单集列表的单集长按菜单、滑动动作、多选批量菜单（加入/取消收藏）
- 2. **不作用于** 以下位置：单集总列表页默认不限定为收藏项，仅当用户在总列表筛选面板选中"收藏"时才显示收藏项；其他与单集收藏无关的页面（如设置页、统计页）不显示收藏入口

# SPEC: REQ-001 首次启动与空库首页

## 需求概述

应用首次启动（或数据库中无任何单集数据）时，应正常进入主界面并展示空库状态。空库状态下页面需清楚展示：应用名称/标识、主导航入口、添加播客入口，以及当前没有订阅/没有内容的空状态提示。用户能通过空状态页面找到添加播客或导入订阅的入口。

## 原子场景

### 场景 1.1：首次安装后启动应用（Happy Path）
**触发**: 用户首次安装并打开应用
**执行步骤**:
1. 系统初始化应用，加载主界面框架
2. 应用查询本地数据库中的单集总数
3. 查询结果为 0（无任何单集数据）
4. 主界面显示空状态欢迎视图（包含应用标识、空状态提示文案）
5. 空状态视图显示添加播客入口指引（箭头指向主导航中的添加入口）
6. 首页内容分区列表被隐藏
7. 用户可看到主导航结构（底部导航栏或侧边导航面板，取决于用户设置）

### 场景 1.2：空库状态下添加播客入口可见性验证
**触发**: 首次启动后用户查看空状态页面
**执行步骤**:
1. 空状态页面展示应用名称或标识
2. 空状态页面包含"添加播客"的引导提示文本
3. 引导提示包含方向指引，指向主导航中的添加入口：
   - 若使用底部导航模式，箭头指向底部导航栏的添加按钮
   - 若使用侧边导航模式，箭头指向侧边导航面板的添加入口
4. 用户点击该入口后可进入添加播客页面

### 场景 1.3：非首次启动但无单集数据（边界场景）
**触发**: 用户曾安装应用、清除了数据或从未成功订阅过播客，重新启动应用
**执行步骤**:
1. 应用启动，加载主界面
2. 查询单集总数，结果为 0
3. 显示空状态欢迎视图（与首次启动行为一致）
4. 首页内容分区被隐藏
5. 主导航正常可用

### 场景 1.4：订阅播客后首页从空状态切换（状态转换）
**触发**: 用户从空状态完成首次订阅播客操作
**执行步骤**:
1. 用户通过添加入口订阅了一个播客
2. 播客订阅数据写入本地数据库
3. 单集列表同步到本地后，单集总数 > 0
4. 首页收到数据更新通知
5. 空状态欢迎视图隐藏
6. 首页内容分区列表显示（队列、收件箱、订阅、下载等分区）
7. 下拉刷新功能恢复可用

### 场景 1.5：空状态页面滚动行为（边界场景）
**触发**: 用户在空状态页面尝试滚动
**执行步骤**:
1. 空状态页面的滚动位置固定在顶部（不允许向下滚动）
2. 页面不崩溃，不产生异常

### 场景 1.6：空状态下的导航模式适配（边界场景）
**触发**: 用户在设置中切换导航模式（底部导航 ↔ 侧边导航）后回到首页
**执行步骤**:
1. 应用检测到导航模式变更
2. 空状态页面的入口指引箭头方向更新：
   - 底部导航模式 → 箭头指向底部
   - 侧边导航模式 → 箭头指向侧边
3. 主导航结构按新模式重新渲染

### 场景 1.7：应用启动不崩溃（健壮性验证）
**触发**: 应用启动过程中数据库初始化等耗时操作
**执行步骤**:
1. 应用启动，初始化本地数据库
2. 查询单集总数的操作在后台异步执行
3. 查询期间页面显示默认状态（不崩溃、不卡死）
4. 查询完成后正确切换到空状态或内容状态
5. 若查询出错，记录错误日志，不影响应用基本功能

## 验收要点
- 应用可正常启动，空库状态下不崩溃
- 空库状态页面清晰展示应用标识和空状态提示
- 添加播客入口可见且可达
- 入口指引方向与当前导航模式一致
- 订阅播客后首页自动从空状态切换到内容列表

# 收件箱（新单集）SPEC

收件箱集中展示用户所有订阅节目中标记为"新（未播）"的单集，支持按日期或时长排序、滑动移除单集、批量移除，以及一键清空全部新单集（移出收件箱）。

- 4. 在列表中展示新单集；当没有任何新单集时，展示"收件箱为空"的空状态文案

## 场景二

用户在"收件箱页"向下滚动到底部时，系统自动加载下一页新单集并追加到列表末尾。

- 1. 用户在"收件箱页"顶部工具栏点击"排序"
- 2. 弹出排序选择面板，展示可选维度：日期（旧→新 / 新→旧）、时长（短→长 / 长→短）
- 3. 用户点击某一排序选项（首次选择某维度取默认升降序，再次点击同维度切换升降序）
- 4. 系统将所选排序方式持久化保存，并触发明细列表刷新

## 场景四

### 场景概述

用户在"收件箱页"对单个新单集进行滑动移除（标记为非新），并提供撤销。

- 1. 用户在"收件箱页"列表中对某条新单集执行滑动操作
- 2. 系统将该单集从"新"状态改为"非新（已播未完成）"，使其移出收件箱

# SPEC: REQ-002 主导航结构

## 需求概述

应用应提供完整的主导航结构，包含首页（Home）、队列（Queue）、收件箱（Inbox）、全部单集（Episodes）、订阅（Subscriptions）、下载（Downloads）、播放历史（Playback History）、收藏（Favorites）、统计（Statistics）和添加播客（Add Feed）等入口。导航可采用底部导航栏或侧边导航面板两种模式，由用户在设置中选择。导航项的选中高亮状态必须与当前展示的页面保持一致。切换页面后页面标题也相应更新。

## 原子场景

### 场景 2.1：底部导航模式 - 导航项展示（Happy Path）
**触发**: 用户使用底部导航模式，查看主界面
**执行步骤**:
1. 底部导航栏显示主要导航入口（首页、队列、收件箱、全部单集、订阅等）
2. 超出底部栏容量的导航项折叠到"更多"入口
3. 点击"更多"弹出列表，展示剩余导航项（下载、播放历史、收藏、统计、添加播客等）
4. 每个导航项显示图标和简短标签

### 场景 2.2：侧边导航模式 - 导航项展示（Happy Path）
**触发**: 用户使用侧边导航模式，打开侧边导航面板
**执行步骤**:
1. 侧边导航面板显示所有导航入口（首页、队列、收件箱、全部单集、订阅、下载、播放历史、收藏、统计、添加播客）
2. 每个导航项显示图标和完整标签
3. 导航项下方显示已订阅播客列表（带新单集计数 badge）
4. 用户可通过手势或按钮打开/关闭侧边面板

### 场景 2.3：导航项切换与选中状态同步（Happy Path）
**触发**: 用户点击某个导航项切换页面
**执行步骤**:
1. 用户点击导航项（如"队列"）
2. 主内容区加载对应页面
3. 页面标题更新为"队列"
4. 被点击的导航项高亮显示（选中态）
5. 其他导航项恢复非选中态
6. 系统记录最后访问的导航项

### 场景 2.4：导航项计数 Badge 显示（Happy Path）
**触发**: 有新单集或下载完成时查看导航
**执行步骤**:
1. 导航项（如收件箱、下载、订阅）显示对应的计数 badge
2. 收件箱：显示新单集数量
3. 队列：显示队列中待播放单集数量
4. 订阅列表中每个播客：显示该播客的新单集计数
5. 单集状态变化后计数实时更新

### 场景 2.5：底部导航"更多"弹窗操作（Happy Path）
**触发**: 用户在底部导航模式点击"更多"入口
**执行步骤**:
1. 弹出列表显示溢出的导航项
2. 每项显示图标和标签
3. 点击某项 → 关闭弹窗，切换到对应页面
4. 返回主界面后底部导航栏保持可见

### 场景 2.6：应用重启后恢复最后导航位置（Happy Path）
**触发**: 用户在某页面退出后重新打开应用
**执行步骤**:
1. 应用启动，读取上次保存的最后导航项
2. 自动加载该导航项对应的页面
3. 导航选中状态与展示页面一致

### 场景 2.7：导航模式切换（边界场景）
**触发**: 用户在设置中将导航模式从底部导航切换为侧边导航（或反向）
**执行步骤**:
1. 用户进入设置 → 界面设置
2. 切换导航模式开关
3. 主界面导航结构重建：
   - 从底部导航栏切换为侧边导航面板（或反向）
4. 所有导航入口仍然可达
5. 当前页面保持不变，导航选中状态正确

### 场景 2.8：导航项对应页面不存在或加载失败（异常场景）
**触发**: 导航项对应页面加载时发生异常
**执行步骤**:
1. 页面加载失败时显示错误状态或空白
2. 应用不崩溃
3. 用户可切换到其他导航项恢复正常

### 场景 2.9：设置入口可达性（边界场景）
**触发**: 用户需要进入应用设置
**执行步骤**:
1. 设置入口通过工具栏菜单或导航溢出项进入
2. 设置入口始终可达，不受导航模式影响
3. 进入设置页不影响当前导航选中状态

### 场景 2.10：导航面板中的订阅列表交互（Happy Path）
**触发**: 用户在侧边导航面板中点击某个订阅播客
**执行步骤**:
1. 点击订阅播客项
2. 主内容区加载该播客的详情页
3. 订阅列表保持可见（面板关闭后可重新打开）
4. 新单集计数 badge 在查看后更新

## 验收要点
- 所有主要导航入口可达（首页、队列、收件箱、全部单集、订阅、下载、播放历史、收藏、统计、添加播客）
- 导航选中高亮状态与当前页面一致
- 切换页面后标题正确更新

# SPEC: REQ-032 OPML导出

## 需求概述

用户可以将已订阅的播客列表导出为标准 OPML 2.0 格式文件，方便迁移到其他播客应用或备份。导出文件包含所有已订阅播客的标题、Feed URL、网页链接和类型信息。用户通过系统文件选择器选择导出路径。

## 原子场景

### 场景 32.1：导出全部订阅为 OPML（Happy Path）
**触发**: 用户在设置中点击"OPML 导出"
**执行步骤**:
1. 用户进入设置 → 存储 → OPML 导出
2. 应用打开系统文件选择器，让用户选择保存位置和文件名
3. 用户选择目标路径并确认
4. 应用获取所有已订阅播客列表
5. 生成标准 OPML 2.0 格式 XML 文档
6. 文档头部包含标题和创建日期
7. 每个已订阅播客以 outline 元素写入
8. 写入完成后提示用户导出成功

### 场景 32.2：导出文件格式正确性（Happy Path）
**触发**: 验证导出的 OPML 文件格式
**执行步骤**:
1. 导出文件为标准 XML 格式，UTF-8 编码
2. 根元素为 `<opml version="2.0">`

5. 每个 outline 包含：text、title、type、xmlUrl、htmlUrl 属性
6. 文件有缩进，便于阅读

**执行步骤**:
1. 应用遍历所有 Feed 数据
2. 仅导出状态为"已订阅"的播客
3. 已退订或未完成的播客不包含在导出文件中

### 场景 32.4：无订阅时导出（边界场景）
**触发**: 用户没有任何订阅时执行导出
**执行步骤**:
1. 应用获取订阅列表为空
2. 仍然生成有效的 OPML 文件
3. `<body>` 部分为空
4. 文件结构完整，可正常打开

### 场景 32.5：播客缺少网页链接（边界场景）
**触发**: 某些播客没有网页链接（htmlUrl）
**执行步骤**:
1. 导出时检测每个 Feed 的 link 字段
2. 有 link 的播客写入 htmlUrl 属性
3. 无 link 的播客省略 htmlUrl 属性
4. xmlUrl 和 title 始终写入

### 场景 32.6：取消导出（边界场景）
**触发**: 用户在文件选择器中取消操作
**执行步骤**:
1. 用户在系统文件选择器中按返回键
2. 不执行任何导出操作
3. 返回设置页面

### 场景 32.7：导出路径不可写（异常场景）
**触发**: 用户选择的目标路径不可写入
**执行步骤**:
1. 应用尝试写入文件
2. 写入失败，抛出 IO 异常
3. 显示错误提示信息
4. 用户可重新选择路径

## 验收要点
- 可通过系统文件选择器选择导出路径
- 导出标准 OPML 2.0 格式文件
- 仅导出已订阅状态的播客
- 每个 outline 包含标题、Feed URL，有网页链接时包含 htmlUrl
- 文件使用 UTF-8 编码，有缩进
- 文件头部包含标题和创建日期
- 无订阅时可生成有效的空 OPML 文件

- 4. 在列表中展示历史单集；当没有任何历史单集时，展示"暂无播放历史"的空状态文案，并隐藏"清空历史"入口

## 场景二

### 场景概述

用户在"播放历史页"向下滚动到底部时，系统自动加载下一页历史单集并追加到列表末尾。

### 场景逻辑步骤

- 1. 用户在"播放历史页"持续向下滚动
- 2. 当滚动接近列表底部且仍存在未加载的更多历史单集时，系统自动发起下一页加载（每页 150 条）
- 3. 加载过程中列表末尾展示加载占位项；加载完成后追加数据并移除占位项；若返回数据不足一页则不再继续分页加载

## 场景三

### 场景概述

用户在"播放历史页"一键清空全部播放历史。

### 场景逻辑步骤

- 1. 用户在"播放历史页"顶部工具栏点击"清空历史"（仅当历史列表非空时该入口可见）
- 2. 系统弹出确认对话框，提示将清空全部播放历史
- 3. 用户点击"确认"后，系统清除所有单集的"在历史中"标记，清空后历史列表变为空状态并隐藏"清空历史"入口；用户也可点击"取消"放弃操作

## 场景四

### 场景概述

单集在播放完成后自动进入播放历史，历史页随之刷新。

### 场景逻辑步骤

- 1. 用户播放某单集至完成（或该单集被标记为已播/完成）
- 2. 系统将该单集写入播放历史（标记为"在历史中"）
- 3. 若"播放历史页"处于打开或再次打开状态，系统自动重新加载历史列表并更新工具栏（清空入口随之显示）

## 场景五

### 场景概述

"播放历史"功能的作用范围与不作用范围。

### 场景逻辑步骤

- 1. **仅作用于** 以下入口进入的"播放历史页"：抽屉导航中的"播放历史"入口
- 2. **不作用于** 以下位置：单集总列表页、收件箱页、收藏页均不限定为"在历史中"，也不提供"清空历史"入口；这些页面不展示播放历史的清空功能

# SPEC: REQ-034 播放设置

## 需求概述

## 原子场景

### 场景 34.1：设置快进/快退秒数（Happy Path）
**触发**: 用户修改快进或快退的默认秒数
**执行步骤**:
1. 用户进入设置 → 播放
2. 点击"快退秒数"设置项
3. 弹出秒数选择对话框，列出可选值（如 10、15、30 秒）
4. 用户选择一个值
5. 保存设置
6. 播放器的快退按钮使用新的秒数
7. 同理设置快进秒数

### 场景 34.2：设置全局播放速度（Happy Path）
**触发**: 用户修改默认播放速度
**执行步骤**:
1. 用户点击"播放速度"设置项
2. 弹出播放速度对话框
3. 显示预设速度选项（如 1.0x、1.25x、1.5x、2.0x）
4. 支持自定义速度
5. 用户选择后保存
6. 后续播放默认使用该速度（单播客自定义速度优先）

### 场景 34.3：设置入队位置（Happy Path）

**执行步骤**:
1. 用户点击"入队位置"设置项

5. 后续添加新剧集到队列时使用新位置

### 场景 34.4：设置智能标记已播阈值（Happy Path）

1. 用户点击"智能标记已播"设置项

3. 秒数自动格式化为"X 秒"或"X 分钟"
4. 用户选择一个值
5. 播放接近结尾时（剩余时长 < 阈值），自动标记该集为已播

### 场景 34.5：关闭智能标记已播（边界场景）

2. 不再自动标记为已播
3. 用户需手动标记播放完成

### 场景 34.6：开启耳机自动恢复播放（Happy Path）
**触发**: 用户开启耳机插入后自动恢复播放
**执行步骤**:
1. 用户开启"耳机插入恢复播放"开关
2. 后续有线耳机插入时自动恢复暂停的播放
3. 注意：特定系统版本以上此选项由系统管理，不再显示

### 场景 34.7：开启蓝牙自动恢复播放（Happy Path）
**触发**: 用户开启蓝牙设备连接后自动恢复播放
**执行步骤**:

2. 后续蓝牙耳机/音箱连接时自动恢复播放
3. 注意：特定系统版本以上此选项由系统管理

### 场景 34.8：耳机/蓝牙选项在高版本不可用（边界场景）
**触发**: 特定系统版本以上查看耳机/蓝牙设置
**执行步骤**:
1. 检测系统版本高于阈值
2. 耳机恢复播放和蓝牙恢复播放选项不显示
3. 这些功能由系统直接管理

## 验收要点
- 可设置快进和快退的秒数（独立设置）
- 可设置全局播放速度
- 可设置入队位置（队尾/队首/当前播放之后）
- 可设置智能标记已播阈值，支持禁用
- 可设置耳机/蓝牙连接后自动恢复播放（版本受限）
- 设置修改后立即应用到播放器
- 秒数选项显示格式化（秒/分钟）

## 概述
用户可以调整音频播放速度。通过弹出速度选择面板，用户可以使用滑动条选择精确速度，或从预设速度列表中快速选择。速度设置会立即应用到播放并持久化保存。同时还提供跳过静音功能。

## 原子场景

### 场景 1：通过滑动条调整播放速度（正常流程）
**触发**：用户点击播放页速度按钮，在弹出的面板中拖动速度滑动条

**执行步骤**：
1. 弹出底部速度选择面板
2. 面板显示当前播放速度，滑动条定位到当前速度值
3. 用户拖动滑动条，实时回调当前速度值
4. 每次速度变化立即保存到用户偏好
5. 每次速度变化立即应用到播放服务（实时改变播放速率）
6. 播放页的速度文字同步更新为新速度值

### 场景 2：从预设速度中选择
**触发**：用户在速度面板中点击某个预设速度

**执行步骤**：
1. 面板展示用户保存的预设速度网格（3 列布局）
2. 用户点击某个预设速度
3. 将该速度保存为当前播放速度
4. 应用到播放服务
5. 面板自动关闭
6. 播放页速度文字更新

### 场景 3：添加当前速度为预设
**触发**：用户在速度面板中点击"添加当前速度"按钮

**执行步骤**：
1. 获取滑动条当前的速度值
2. 检查该速度是否已存在于预设列表中
3. 若已存在：显示提示"预设已存在"
4. 若不存在：添加到预设列表、排序、保存到偏好
5. 预设网格刷新显示

### 场景 4：删除预设速度
**触发**：用户长按某个预设速度

**执行步骤**：
1. 从预设列表中移除该速度
2. 保存更新后的列表到偏好
3. 预设网格刷新显示

### 场景 5：开启/关闭跳过静音
**触发**：用户在速度面板中切换"跳过静音"开关

**执行步骤**：
1. 记录开关状态
2. 保存到用户偏好
3. 应用到播放服务（开启后播放时自动跳过静音部分）

### 场景 6：速度对时间显示的影响
**触发**：播放速度非 1.0 时查看进度

**执行步骤**：
1. 进度条位置和时长文字按当前播放速度转换显示
2. 例如 2.0x 速度下，30 分钟的媒体显示为 15 分钟
3. 队列页的剩余时间也考虑播放速度（若用户启用了此选项）

### 场景 29.9：修改来源 URL（Happy Path）

**执行步骤**:

# REQ-016 队列页面 - 需求规格

## 概述
队列页面展示用户加入播放队列的所有剧集，以列表形式呈现。页面提供剩余集数和总时长的汇总信息，支持排序、清空和锁定操作，以及实时反映队列变更事件。

## 原子场景

### 场景 1：查看队列列表（正常流程）
**触发**：用户导航到队列页面

**执行步骤**：
1. 异步从数据库加载队列数据
2. 显示加载进度指示器
3. 加载完成后以列表形式展示队列中的所有剧集
4. 每个剧集项显示：封面、标题、播客名、播放状态、下载状态等
5. 信息栏显示队列汇总："N 集 • 剩余 Xh Ym"
6. 剩余时长按各剧集未播放部分累加（可选择按播放速度计算）
7. 恢复上次的滚动位置

### 场景 2：队列为空时的显示
**触发**：队列中没有剧集

**执行步骤**：
1. 显示空状态视图（图标 + 标题 + 提示消息）
2. 若收件箱中有新剧集：显示"前往收件箱"按钮
3. 点击按钮跳转到收件箱页面

### 场景 3：排序队列
**触发**：用户点击工具栏的"排序"按钮

**执行步骤**：
1. 弹出排序选择对话框
2. 显示可选排序方式：日期（新→旧/旧→新）、标题（A→Z/Z→A）、时长（长→短/短→长）、随机、智能随机
3. 显示"保持排序"复选框
4. 用户选择排序方式：
   a. 若选择"随机"，自动取消"保持排序"
   b. 其他选择时恢复"保持排序"状态
5. 立即对队列执行排序并保存
6. 列表刷新显示排序后的顺序

### 场景 4：清空队列
**触发**：用户点击工具栏的"清空队列"按钮

**执行步骤**：
1. 弹出确认对话框："确定要清空队列吗？"
2. 用户点击确认：
   a. 清空数据库中的队列
   b. 发送队列清空事件
   c. 列表清空，显示空状态
3. 用户点击取消：不做任何操作

### 场景 5：锁定/解锁队列
**触发**：用户点击工具栏的"锁定"按钮

**执行步骤**：
1. 若队列当前未锁定且首次使用：
   a. 弹出锁定警告对话框（解释锁定后无法手动排序）
   b. 提供"不再显示"复选框
   c. 用户确认后锁定队列
2. 若队列当前已锁定：直接解锁
3. 锁定后：禁用拖拽排序和滑动操作
4. 解锁后：恢复拖拽排序和滑动操作
5. 若开启了"保持排序"：隐藏锁定按钮

### 场景 6：实时更新队列列表
**触发**：其他页面修改了队列（加入/移除/排序/清空）

**执行步骤**：
1. 队列页面通过事件总线接收队列变更事件
2. 根据事件类型更新内存中的队列列表
3. 通知适配器刷新对应的列表项（增/删/移/整体刷新）
4. 更新信息栏的集数和时长汇总
5. 更新工具栏状态（锁定按钮等）

### 场景 7：更新当前播放项进度
**触发**：播放进度更新事件

**执行步骤**：
1. 在队列列表中找到当前正在播放的剧集项
2. 更新该项的进度显示（进度条/位置文字）
3. 不影响其他剧集项的显示

### 场景 8：下拉刷新
**触发**：用户在队列页面下拉

**执行步骤**：
1. 触发播客源更新（检查是否有新剧集）
2. 显示刷新动画
3. 更新完成后隐藏刷新动画

# REQ-017 队列排序与移动 - 需求规格

## 概述
用户可以在队列页面通过拖拽或上下文菜单重新排列队列中剧集的顺序。支持拖拽到任意位置，以及快速移动单个或多个剧集到队列顶部或底部。队列锁定或保持排序模式下禁用手动排序。

## 原子场景

### 场景 1：拖拽移动剧集到任意位置（正常流程）
**触发**：用户长按某剧集的拖拽手柄并上下拖动

**执行步骤**：
1. 检测到拖拽手柄的按下事件
2. 启动拖拽模式，该剧集项变为可拖动状态
3. 用户上下拖动时：
   a. 实时在内存列表中移动该剧集的位置
   b. 列表实时反映新的排列顺序
4. 松手时：
   a. 检测拖拽起始位置和目标位置是否不同
   b. 将移动操作持久化到数据库
   c. 发送移动事件通知其他组件

### 场景 2：通过封面区域拖拽
**触发**：用户长按某剧集封面区域的左半部分

**执行步骤**：
1. 检测到封面左半区域的按下事件
2. 启动拖拽模式（同场景 1）
3. 封面右半区域的按下不触发拖拽（避免误触）

### 场景 3：通过上下文菜单移动到顶部
**触发**：用户长按某剧集，在弹出菜单中选择"移到顶部"

**执行步骤**：
1. 弹出上下文菜单
2. 若该剧集已在顶部：隐藏"移到顶部"选项
3. 若队列开启保持排序：隐藏移动选项
4. 选择"移到顶部"：
   a. 从当前位置移除该剧集
   b. 插入到队列索引 0 的位置
   c. 持久化到数据库
   d. 列表更新显示
5. 信息栏更新集数和时长汇总

### 场景 4：通过上下文菜单移动到底部
**触发**：用户长按某剧集，在弹出菜单中选择"移到底部"

**执行步骤**：
1. 弹出上下文菜单
2. 若该剧集已在底部：隐藏"移到底部"选项
3. 若队列开启保持排序：隐藏移动选项
4. 选择"移到底部"：
   a. 从当前位置移除该剧集
   b. 追加到队列末尾
   c. 持久化到数据库
   d. 列表更新显示

### 场景 5：多选批量移动到顶部
**触发**：用户在多选模式下选中多个剧集，点击"移到顶部"

**执行步骤**：
1. 系统检查选中项是否可以移到顶部（canMove 检查）
   a. 若选中项已连续位于顶部：隐藏"移到顶部"按钮
   b. 若选中项数量等于队列总数：隐藏移动按钮
2. 执行批量移动：
   a. 反转选中项顺序（保持原始相对顺序）
   b. 从队列中移除所有选中项
   c. 依次插入到队列顶部
3. 持久化到数据库
4. 列表刷新显示

### 场景 6：多选批量移动到底部
**触发**：用户在多选模式下选中多个剧集，点击"移到底部"

**执行步骤**：
1. 系统检查选中项是否可以移到底部
   a. 若选中项已连续位于底部：隐藏"移到底部"按钮
2. 执行批量移动：
   a. 从队列中移除所有选中项
   b. 依次追加到队列末尾
3. 持久化到数据库
4. 列表刷新显示

### 场景 7：队列锁定时禁用排序
**触发**：队列处于锁定状态时用户尝试拖拽

**执行步骤**：
1. 拖拽手柄不可见
2. 封面区域的拖拽触摸监听被移除
3. 上下文菜单中的移动选项隐藏
4. 用户无法手动排序

### 场景 8：队列保持排序时禁用排序
**触发**：队列处于"保持排序"模式时

**执行步骤**：
1. 拖拽手柄不可见
2. 上下文菜单中的移动选项隐藏
3. 队列顺序由排序规则自动管理

# 订阅内搜索 需求规格说明（SPEC）

## 1. 场景概述

本特性为播客应用提供「订阅内搜索」能力。当用户进入某个已订阅节目的单集列表页后，可通过工具栏的「搜索」入口打开搜索页，此时搜索范围被限定在该节目内。

用户输入关键词后，应用仅在该节目的单集中，按「标题」与「描述」进行关键词匹配，多词以「与」关系过滤，按发布时间倒序返回最多 300 条。与全库搜索不同，订阅内搜索：

- **不展示节目（订阅）结果区**：结果仅包含单集，不会出现节目横排列表。
- **覆盖该节目下所有状态的单集**：包括已归档的单集，均可被搜索到。
- **禁用在线搜索入口**：不提供网络搜索节目的能力。
- **顶部显示节目范围标签**：以当前节目名称作为可移除的筛选标签展示。

用户可点击节目范围标签的关闭按钮，将搜索范围从「当前节目」切换回「全库」，此时应用以同一关键词重新执行全库搜索（节目结果区恢复展示，在线搜索入口恢复可用）。其余行为（输入防抖、空状态提示、无结果提示、点击单集结果进入详情翻页、状态变化自动刷新）与全库搜索一致，详见「搜索页」特性。

---

## 2. 场景逻辑步骤

### 场景 2.1 从节目单集列表页进入订阅内搜索

**前置条件**
- 用户已进入某个已订阅节目的单集列表页。
- 该节目至少拥有一条单集。

**主流程**
1. 用户点击单集列表页工具栏的「搜索」入口。
2. 应用打开搜索页，搜索范围被设定为当前节目。
3. 搜索输入框自动获取焦点并弹出输入键盘。
4. 顶部显示以当前节目名称命名的范围标签。
5. 节目结果区不展示（仅单集结果区可用）。
6. 在线搜索入口被隐藏。
7. 此时未输入关键词，页面显示空状态提示「输入以搜索」。

**后置条件**
- 订阅内搜索页已展示，范围限定为当前节目，等待用户输入。

---

### 场景 2.2 输入关键词并触发防抖搜索

**前置条件**
- 订阅内搜索页已打开，输入框已聚焦。

**主流程**
1. 用户在输入框输入关键词。
2. 应用对输入内容进行分词处理。
3. 若输入为空、以空格结尾，或距上次输入变化已超过防抖阈值（1500 毫秒），则立即触发搜索。
4. 否则，应用延迟 750 毫秒后触发搜索，以减少频繁查询。
5. 搜索完成后，单集结果区刷新。

**边界与异常**
- 输入被清空时：页面恢复空状态提示「输入以搜索」。
- 连续快速输入时：仅以最后一次有效输入为准触发搜索。

**后置条件**
- 搜索已执行，结果区已更新。

---

### 场景 2.3 提交搜索（手动确认）

**前置条件**
- 用户已在输入框输入关键词。

**主流程**
1. 用户点击键盘上的「搜索/确认」按钮提交查询。
2. 应用显示进度条，表示正在搜索。
3. 搜索完成后，隐藏进度条并展示结果。

**后置条件**
- 结果已展示，进度条已隐藏。

---

### 场景 2.4 展示订阅内单集搜索结果

**前置条件**
- 用户已输入关键词并触发搜索，搜索范围为当前节目。

**主流程**
1. 应用在当前节目的所有单集（含已归档单集）中，判断每条单集的「标题」或「描述」是否包含全部搜索词（多词以「与」关系匹配）。
2. 匹配的单集按发布时间倒序排列。
3. 最多返回 300 条单集结果。
4. 单集结果以纵向列表展示，每条显示标题、发布日期、播放状态、下载状态等信息。
5. 节目结果区保持为空（不展示）。

**边界与异常**
- 无匹配单集：页面显示空状态提示「未找到相关结果」。

**后置条件**
- 单集结果已渲染，节目结果区为空。

---

### 场景 2.5 点击单集结果进入详情

**前置条件**
- 单集结果列表已展示。

**主流程**
1. 用户点击某条单集结果。
2. 应用打开单集详情页，并以当前搜索结果集作为翻页上下文。
3. 用户可在详情页内前后翻页浏览同属本次结果的相邻单集。

**后置条件**
- 单集详情页已打开。

---

### 场景 2.6 移除节目范围标签切换为全库搜索

**前置条件**
- 订阅内搜索页顶部展示了以节目名称命名的范围标签。

**主流程**
1. 用户点击节目范围标签的关闭按钮。
2. 应用将搜索范围从「当前节目」切换为「全库」。
3. 应用显示进度条并重新执行搜索。
4. 搜索完成后：
   - 单集结果按全库范围刷新（不再限定于原节目）。
   - 节目结果区恢复展示（按标题、作者、描述匹配的节目）。
   - 在线搜索入口恢复可用。
5. 节目范围标签从顶部移除。

**后置条件**
- 搜索已切换为全库模式，结果已刷新（后续行为归属「搜索页」特性）。

---

### 场景 2.7 状态变化后自动刷新结果

**前置条件**
- 订阅内搜索页已展示结果。

**主流程**
1. 后台发生单集状态变化（如播放进度更新、播放状态切换）、下载进度变化、订阅列表变化等事件。
2. 应用接收事件后，自动以当前关键词与当前节目范围重新执行搜索或增量更新对应结果项的状态。

# 搜索页 需求规格说明（SPEC）

## 1. 场景概述

本特性为播客应用提供全库搜索页。用户通过单集列表页（总列表、收件箱、收藏、播放历史、已下载、节目详情等）工具栏的「搜索」入口进入搜索页，在输入框输入关键词后，应用在全库范围内检索：

- **单集**：按「标题」与「描述」进行关键词匹配，多词以「与」关系过滤，按发布时间倒序返回最多 300 条。
- **节目**：按「标题」「自定义标题」「作者」「描述」进行关键词匹配，按标题升序返回最多 300 条。

搜索结果分两个区域展示：单集以纵向列表呈现，节目以横向滚动列表呈现。用户可点击单集结果进入单集详情页（支持在结果集内翻页浏览），点击节目结果进入节目主页。搜索页同时提供「在线搜索」入口，用于在网络中查找尚未订阅的节目；并提供可移除的筛选标签（如「队列」「归档」等），用于缩小结果范围。输入即触发搜索（带防抖机制），无输入时提示「输入以搜索」，无结果时提示「未找到相关结果」。

本页面的订阅内搜索（限定单个节目）行为，归属「订阅内搜索」特性，不在本规格范围内。

2. 应用打开搜索页，搜索输入框自动获取焦点并弹出输入键盘。

# SPEC: REQ-038 分享

## 需求概述

用户可以通过系统分享面板分享播客订阅链接和单集信息。支持分享播客订阅深度链接、单集网页链接、单集媒体文件链接（带或不带播放位置）、以及已下载的媒体文件。分享文本自动组装播客名称、单集标题、网页链接和媒体链接。长文本支持缩写。

## 原子场景

### 场景 38.1：分享单集网页链接（Happy Path）
**触发**: 用户在单集详情页选择分享网页链接
**执行步骤**:
1. 用户在单集详情页点击分享按钮
2. 弹出分享选项对话框
3. 用户选择"分享单集链接"
4. 应用组装分享文本：播客名称 + 单集标题 + 网页链接
5. 打开系统分享面板
6. 用户选择分享目标应用
7. 链接发送到目标应用

### 场景 38.2：分享带播放位置的媒体链接（Happy Path）
**触发**: 用户选择分享带当前播放位置的媒体链接
**执行步骤**:
1. 用户在单集详情页点击分享
2. 选择"分享媒体链接（带位置）"
3. 应用组装分享文本：
   - 播客名称 + 单集标题
   - 起始位置：HH:MM:SS
   - 媒体文件 URL + 播放位置锚点 (#t=秒数)
4. 打开系统分享面板
5. 接收方可通过链接跳转到指定播放位置

### 场景 38.3：分享播客订阅链接（Happy Path）
**触发**: 用户在播客详情页分享播客
**执行步骤**:
1. 用户在播客详情页点击分享按钮
2. 应用生成订阅深度链接
3. 链接格式包含播客 Feed URL 和标题
4. 打开系统分享面板
5. 接收方可通过链接快速订阅该播客

### 场景 38.4：分享已下载的媒体文件（Happy Path）
**触发**: 用户分享已下载的音频/视频文件
**执行步骤**:
1. 单集已下载且用户选择"分享文件"
2. 应用获取文件的临时读取 URI
3. 打开系统分享面板，附带文件流
4. 分享类型为媒体 MIME 类型
5. 接收方可直接接收文件

### 场景 38.5：单集没有网页链接（边界场景）
**触发**: 单集没有可分享的网页链接
**执行步骤**:
1. 应用检查单集是否有网页链接
2. 若无网页链接，分享文本中不包含链接部分
3. 仅分享播客名称和标题（及媒体链接如有）

### 场景 38.6：长标题缩写（边界场景）
**触发**: 单集标题或链接过长
**执行步骤**:
1. 分享文本组装时检测长度
2. 超过 50 字符的标题或链接截断
3. 添加省略号（…）
4. 保持分享文本简洁

### 场景 38.7：没有可下载媒体的单集（边界场景）
**触发**: 单集没有媒体文件下载地址
**执行步骤**:
1. 应用检查单集是否有媒体文件
2. 若无媒体文件，分享文本不包含媒体链接
3. 仅分享标题和网页链接

### 场景 38.8：系统分享面板未弹出（异常场景）
**触发**: 系统中没有可接收分享的应用
**执行步骤**:
1. 应用尝试打开系统分享面板
2. 若无应用可处理分享 Intent
3. 系统显示"没有应用可执行此操作"提示
4. 不崩溃

## 验收要点
- 可分享单集网页链接，文本包含播客名称和标题
- 可分享带播放位置的媒体链接（时间锚点）
- 可分享播客订阅深度链接
- 可分享已下载的媒体文件
- 长文本自动缩写（50 字符截断加省略号）
- 无网页链接时不包含链接部分
- 无媒体文件时不包含媒体链接
- 通过系统分享面板选择目标应用

# REQ-014 睡眠定时器 - 需求规格

## 概述
睡眠定时器允许用户设定一个时间或集数，到达后自动暂停播放。支持按分钟倒计时和按集数停止两种模式。还提供延长定时器、自动启用、摇晃重置、结束震动等辅助功能。

## 原子场景

### 场景 1：设置时间倒计时定时器（正常流程）
**触发**：用户在播放页工具栏中选择"设置睡眠定时器"

**执行步骤**：
1. 弹出睡眠定时器底部面板
2. 定时器类型选择为"分钟"
3. 输入框预填上次使用的分钟数
4. 用户输入或调整分钟数
5. 点击"设置"按钮
6. 验证播放服务正在运行（若无播放则提示"无媒体播放"）
7. 保存设定值到偏好
8. 向播放服务发送定时器指令
9. 面板切换为倒计时显示模式，实时显示剩余时间
10. 播放页工具栏菜单项切换为"禁用睡眠定时器"

### 场景 2：设置按集数停止定时器
**触发**：用户在睡眠定时器面板中切换到"集数"类型

**执行步骤**：
1. 从类型下拉框选择"集数"
2. 输入框显示上次使用的集数
3. 用户输入想要播放的集数
4. 系统检查输入集数是否超过队列剩余集数
5. 若超过：显示提示信息
6. 若未超过或连续播放已开启：允许设置
7. 点击"设置"按钮，发送定时器指令
8. 面板显示剩余集数

### 场景 3：延长正在运行的定时器
**触发**：定时器正在运行时用户点击延长按钮

**执行步骤**：
1. 面板显示三个延长选项（如 +5、+10、+30 分钟/集数）
2. 用户点击某个延长按钮
3. 计算新的定时器结束时间（当前剩余 + 延长值）
4. 向播放服务发送延长指令
5. 面板更新显示新的剩余时间

### 场景 4：手动禁用定时器
**触发**：用户点击"禁用"按钮

**执行步骤**：
1. 用户在面板中点击禁用按钮
2. 向播放服务发送禁用指令
3. 定时器停止倒计时
4. 面板切换回设置模式
5. 播放页工具栏菜单项切换回"设置睡眠定时器"

### 场景 5：定时器自然结束
**触发**：倒计时归零

**执行步骤**：
1. 定时器到达零
2. 播放服务自动暂停播放
3. 若开启了震动选项，设备震动提示
4. 面板切换回设置模式
5. 播放页工具栏菜单项切换回"设置睡眠定时器"

### 场景 6：自动启用定时器
**触发**：用户配置了自动启用时间段

**执行步骤**：
1. 用户勾选"自动启用"选项
2. 设置自动启用的时间范围（如 22:00 - 07:00）
3. 在指定时间段内开始播放时，自动启用定时器
4. 若时间范围覆盖大部分时间且为集数类型，显示确认对话框

### 场景 7：连续播放未开启时的集数限制
**触发**：集数类型定时器且连续播放未开启

**执行步骤**：
1. 检测到连续播放（自动播放下一集）未开启
2. 集数输入框禁用，锁定为 1 集
3. 显示提示信息："连续播放未开启时只能播放 1 集"
4. 提供跳转到播放设置的按钮

# SPEC: REQ-004 通过稳定 RSS feed 订阅播客

1. 用户通过添加播客页面输入 RSS URL
2. 应用检查该 URL 对应的播客是否已订阅
3. 若未订阅，开始下载 Feed 内容
4. 下载完成后解析 Feed（提取标题、封面、描述、单集列表）
5. 显示 Feed 预览页面：播客名称、封面、描述、单集列表

8. 导航到该播客的详情页
9. 新订阅出现在订阅列表中

### 场景 4.2：Feed URL 为网页而非直接 Feed（Happy Path）

**执行步骤**:
1. 应用下载 URL 内容
2. 检测到内容不是有效的 RSS/Atom Feed
3. 尝试在页面内容中搜索 Feed 链接

### 场景 4.3：通过搜索反查 Feed URL（Happy Path）
**触发**: 输入的 URL 无法直接找到 Feed，但可通过搜索匹配
**执行步骤**:
1. 直接 URL 查找失败
2. 应用尝试通过搜索平台按网页标题/曲名搜索对应播客
3. 找到匹配的播客及其 Feed URL
4. 使用找到的 Feed URL 继续下载和解析流程

### 场景 4.4：多 Feed 发现与选择（Happy Path）
**触发**: 下载的页面包含多个 Feed 链接
**执行步骤**:
1. 应用解析页面发现多个 Feed 链接
2. 弹出 Feed 发现对话框，列出所有找到的 Feed
3. 每个 Feed 显示标题和 URL
4. 用户选择其中一个
5. 使用选中的 Feed URL 继续解析和预览流程

### 场景 4.5：订阅前预览确认（Happy Path）
**触发**: Feed 解析成功后用户查看预览
**执行步骤**:
1. 预览页面显示播客标题
2. 预览页面显示播客封面
3. 预览页面显示播客描述
4. 预览页面显示单集列表（标题、日期等）
5. 预览页面提供"订阅"按钮
6. 用户确认后才正式订阅（不自动订阅）

### 场景 4.6：已订阅的 Feed 重复订阅（边界场景）
**触发**: 用户输入一个已订阅播客的 Feed URL
**执行步骤**:
1. 应用检查到该 Feed 已在订阅列表中
2. 提示用户该播客已订阅
3. 不执行重复订阅操作
4. 可选择直接跳转到该播客详情页

### 场景 4.7：无效 URL 格式（异常场景）
**触发**: 用户输入格式无效的 URL
**执行步骤**:
1. URL 格式校验失败
2. 显示"地址无效"错误提示
3. 不发起网络请求
4. 用户可修正 URL 后重试

### 场景 4.8：网络请求失败（异常场景）
**触发**: RSS Feed URL 有效但网络不可达
**执行步骤**:
1. 应用尝试下载 Feed 内容
2. 网络请求超时或连接失败
3. 显示错误对话框，提示网络错误
4. 提供重试入口和返回入口
5. 应用不崩溃

### 场景 4.9：Feed 解析失败（异常场景）
**触发**: Feed 内容下载成功但无法解析

2. 解析器报错
3. 显示错误对话框，提示解析失败
4. 提供"编辑 URL"入口，允许用户修正 URL 后重试
5. 提供返回入口

### 场景 4.10：未找到播客（异常场景）
**触发**: URL 处理后未找到任何可用的播客 Feed
**执行步骤**:
1. 所有查找方式（直接下载、搜索反查、Feed 发现）均失败
2. 显示"未找到播客"错误提示
3. 提供"编辑 URL"入口
4. 提供返回入口

3. 确认后使用新 URL 重新执行下载和解析流程

## 验收要点

- 订阅前可预览 Feed 信息，确认后才订阅
- 订阅成功后进入播客详情页，新订阅出现在列表中
- 网页 URL 能自动查找 Feed 链接
- 多 Feed 页面可选择具体 Feed
- 无效 URL、网络失败、解析失败有错误提示和重试入口

- 未登录时显示"选择同步服务"入口

- 可修改登录密码
- 可登出并清除凭证

- 登出后恢复未登录状态

# SPEC: REQ-030 标签与分组

## 需求概述

用户可以为订阅的播客设置一个或多个自定义标签，实现分组管理。标签在导航抽屉中作为分组显示，用户可按标签过滤和浏览订阅列表。支持添加新标签、从已有标签中选择、删除标签，以及批量为多个播客设置标签。标签数据持久化存储。

## 原子场景

### 场景 30.1：为单个播客添加标签（Happy Path）
**触发**: 用户在播客设置中为播客添加标签
**执行步骤**:
1. 用户在播客设置页点击"标签"设置项
2. 弹出标签编辑对话框
3. 显示该播客已有的标签（芯片形式排列）
4. 用户在输入框中输入新标签名称
5. 输入时从已有标签列表自动补全提示
6. 确认添加后标签以芯片形式显示在列表中
7. 用户可继续添加多个标签
8. 点击确认后保存所有标签到本地

### 场景 30.2：删除已有标签（Happy Path）
**触发**: 用户移除播客的某个标签
**执行步骤**:
1. 标签编辑对话框中显示已有标签芯片
2. 每个芯片有删除按钮
3. 用户点击删除按钮移除标签
4. 芯片从列表中消失
5. 确认后保存更新后的标签集合

### 场景 30.3：为多个播客批量设置标签（Happy Path）
**触发**: 用户在订阅列表中多选播客后设置标签
**执行步骤**:
1. 用户在订阅列表中进入多选模式
2. 选择多个播客
3. 点击"设置标签"操作
4. 弹出标签编辑对话框
5. 仅显示所有选中播客的公共标签（交集）
6. 如有多个播客，显示提示信息说明仅显示公共标签
7. 用户添加或删除标签
8. 确认后所有选中播客更新标签（移除原公共标签，添加新标签）

### 场景 30.4：按标签过滤浏览（Happy Path）
**触发**: 用户在导航抽屉中点击某个标签
**执行步骤**:
1. 导航抽屉中按标签分组显示订阅
2. 每个标签显示标题和关联播客数量
3. 用户点击某标签
4. 展开显示该标签下的所有播客
5. 用户可选择某个播客进入详情页

### 场景 30.5：设置根标签（Happy Path）
**触发**: 用户将播客标记为根级显示
**执行步骤**:
1. 标签编辑对话框中有"根文件夹"复选框
2. 仅在非底部导航模式下显示该选项
3. 用户勾选后该播客显示在导航根目录
4. 确认后保存

### 场景 30.6：标签名称重复校验（边界场景）
**触发**: 用户尝试添加已存在的标签
**执行步骤**:
1. 用户输入的标签名已存在于当前列表中
2. 不添加重复标签
3. 输入框保持焦点等待用户修正

### 场景 30.7：未分类标签显示（边界场景）
**触发**: 播客没有设置任何标签
**执行步骤**:
1. 系统自动将无标签播客归入"未分类"分组
2. 导航抽屉中显示"未分类"条目
3. 用户可展开查看所有未设置标签的播客

### 场景 30.8：空标签名称校验（边界场景）
**触发**: 用户尝试添加空名称的标签
**执行步骤**:
1. 用户输入为空或纯空格
2. 不添加标签
3. 输入框保持焦点

## 验收要点
- 可为单个播客添加、删除多个自定义标签
- 支持多选播客批量设置标签（公共标签交集处理）
- 导航抽屉按标签分组显示，每组显示播客数量
- 点击标签可展开查看该标签下的播客
- 支持根标签（在非底部导航模式下）
- 标签输入支持自动补全已有标签
- 无标签播客自动归入"未分类"
- 标签数据持久化存储

# SPEC: REQ-033 主题与外观设置

## 需求概述

用户可以自定义应用的外观和导航方式，包括：主题模式（跟随系统/浅色/深色）、纯黑主题、动态取色（基于壁纸）、剩余时间/已播放时间显示切换、导航抽屉条目显隐、底部导航/侧边抽屉切换、通知栏快捷按钮自定义、剧集列表全局排序方式、滑动手势配置、优先流式播放/下载按钮选择。外观修改后立即生效。

## 原子场景

### 场景 33.1：切换主题模式（Happy Path）
**触发**: 用户更改应用主题
**执行步骤**:
1. 用户进入设置 → 用户界面 → 主题
2. 显示主题选择器，可选：跟随系统、浅色、深色
3. 用户选择一个主题
4. 应用立即重建页面，应用新主题
5. 所有页面颜色立即更新

### 场景 33.2：开启纯黑主题（Happy Path）
**触发**: 用户在深色模式下开启纯黑背景
**执行步骤**:
1. 用户切换"纯黑主题"开关
2. 应用立即重建页面
3. 深色模式下背景变为纯黑
4. 节省 OLED 屏幕电量

### 场景 33.3：开启动态取色（Happy Path）
**触发**: 用户（Android 12+）开启基于壁纸的动态取色
**执行步骤**:
1. 用户切换"动态取色"开关
2. 应用立即重建页面
3. 主题颜色根据系统壁纸动态调整
4. 注意：仅特定系统版本以上显示此选项

### 场景 33.4：切换时间显示方式（Happy Path）
**触发**: 用户将时间显示从"已播放时间"切换为"剩余时间"
**执行步骤**:
1. 用户切换"显示剩余时间"开关
2. 应用广播状态更新事件
3. 所有剧集列表项的时间标签立即刷新
4. 播放器界面时间显示同步更新
5. 显示剩余时长而非已播放时长

### 场景 33.5：自定义导航抽屉条目（Happy Path）
**触发**: 用户隐藏不需要的导航抽屉条目
**执行步骤**:
1. 用户点击"导航抽屉条目"设置
2. 弹出对话框，列出所有可用导航条目
3. 每个条目有显示/隐藏开关
4. 用户勾选要隐藏的条目
5. 确认后导航抽屉仅显示选中的条目

### 场景 33.6：自定义通知栏快捷按钮（Happy Path）
**触发**: 用户设置播放通知栏的快捷按钮
**执行步骤**:
1. 用户点击"通知栏按钮"设置
2. 弹出多选对话框，列出可选按钮：跳过、下一章节、播放速度、睡眠定时器
3. 用户必须选择恰好 2 个按钮
4. 若选择数量不等于 2，显示错误提示
5. 选择 2 个后确认保存
6. 播放通知栏显示自定义的 2 个快捷按钮

### 场景 33.7：设置全局默认排序（Happy Path）
**触发**: 用户设置剧集列表的默认排序方式
**执行步骤**:
1. 用户点击"默认排序"设置
2. 弹出排序选项对话框
3. 用户选择排序方式（如按日期、标题、时长等）
4. 确认后所有剧集列表使用新的默认排序

### 场景 33.8：切换底部导航和侧边抽屉（Happy Path）
**触发**: 用户切换导航模式
**执行步骤**:
1. 用户切换"底部导航"开关
2. 若关闭，显示弃用警告对话框
3. 确认后切换导航模式
4. 开启时显示底部导航栏，"返回键打开抽屉"设置变为不可用
5. 关闭时使用侧边抽屉导航

### 场景 33.9：切换优先流式播放（Happy Path）
**触发**: 用户更改默认操作按钮偏好
**执行步骤**:
1. 用户切换"优先流式播放"开关
2. 应用刷新所有剧集列表
3. 开启时列表项默认按钮变为"流式播放"
4. 关闭时默认按钮变为"下载"
5. 不再显示流式播放建议提示

### 场景 33.10：配置滑动手势（Happy Path）
**触发**: 用户自定义列表项滑动手势动作
**执行步骤**:
1. 用户点击"滑动手势"设置
2. 打开滑动手势配置子页面
3. 分别配置左滑和右滑的动作
4. 可选动作：加入队列、加入收藏、标记已播等
5. 确认后列表项滑动手势使用新配置

### 场景 33.11：动态取色在低版本不可用（边界场景）
**触发**: 低版本系统用户查看动态取色选项
**执行步骤**:
1. 检测系统版本低于阈值
2. "动态取色"选项不显示
3. 用户无法启用该功能

## 验收要点
- 支持跟随系统、浅色、深色三种主题模式，切换后立即生效
- 支持纯黑主题和动态取色（版本受限）
- 可切换时间显示方式（剩余/已播放），全局立即刷新
- 可自定义导航抽屉显示条目
- 可自定义通知栏 2 个快捷按钮（必须恰好选 2 个）
- 可设置全局默认排序方式和滑动手势
- 可切换底部导航和侧边抽屉
- 可切换默认操作按钮（流式播放/下载）
- 外观修改后立即生效，无需重启应用
