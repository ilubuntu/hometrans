# SPEC: REQ-034 播放设置

## 需求概述

用户可以配置全局播放行为，包括：快进和快退的秒数、全局播放速度、入队位置（新单集加入队列的位置）、智能标记已播阈值（接近播放完成多少秒时自动标记为已播）、耳机/蓝牙连接后自动恢复播放、通话结束后恢复播放。设置修改后立即应用到播放器。

## 原子场景

### 场景 34.1：设置快进/快退秒数（Happy Path）
**触发**: 用户修改快进或快退的默认秒数
**执行步骤**:
1. 用户进入设置 → 播放
2. 点击"快退秒数"设置项
3. 弹出秒数选择对话框，列出可选值（如 10、15、30 秒）
4. 用户选择一个值
5. 保存设置
6. 播放器的快退按钮使用新的秒数
7. 同理设置快进秒数

### 场景 34.2：设置全局播放速度（Happy Path）
**触发**: 用户修改默认播放速度
**执行步骤**:
1. 用户点击"播放速度"设置项
2. 弹出播放速度对话框
3. 显示预设速度选项（如 1.0x、1.25x、1.5x、2.0x）
4. 支持自定义速度
5. 用户选择后保存
6. 后续播放默认使用该速度（单播客自定义速度优先）

### 场景 34.3：设置入队位置（Happy Path）
**触发**: 用户更改新剧集加入播放队列的位置
**执行步骤**:
1. 用户点击"入队位置"设置项
2. 弹出选项：队尾、队首、当前播放项之后
3. 用户选择一个选项
4. 设置摘要更新为当前选择
5. 后续添加新剧集到队列时使用新位置

### 场景 34.4：设置智能标记已播阈值（Happy Path）
**触发**: 用户设置接近播放完成时自动标记为已播的阈值
**执行步骤**:
1. 用户点击"智能标记已播"设置项
2. 弹出选项列表：禁用、15秒、30秒、1分钟等
3. 秒数自动格式化为"X 秒"或"X 分钟"
4. 用户选择一个值
5. 播放接近结尾时（剩余时长 < 阈值），自动标记该集为已播

### 场景 34.5：关闭智能标记已播（边界场景）
**触发**: 用户选择禁用智能标记已播
**执行步骤**:
1. 用户选择"禁用"选项
2. 不再自动标记为已播
3. 用户需手动标记播放完成

### 场景 34.6：开启耳机自动恢复播放（Happy Path）
**触发**: 用户开启耳机插入后自动恢复播放
**执行步骤**:
1. 用户开启"耳机插入恢复播放"开关
2. 后续有线耳机插入时自动恢复暂停的播放
3. 注意：特定系统版本以上此选项由系统管理，不再显示

### 场景 34.7：开启蓝牙自动恢复播放（Happy Path）
**触发**: 用户开启蓝牙设备连接后自动恢复播放
**执行步骤**:
1. 用户开启"蓝牙连接恢复播放"开关
2. 后续蓝牙耳机/音箱连接时自动恢复播放
3. 注意：特定系统版本以上此选项由系统管理

### 场景 34.8：耳机/蓝牙选项在高版本不可用（边界场景）
**触发**: 特定系统版本以上查看耳机/蓝牙设置
**执行步骤**:
1. 检测系统版本高于阈值
2. 耳机恢复播放和蓝牙恢复播放选项不显示
3. 这些功能由系统直接管理

## 验收要点
- 可设置快进和快退的秒数（独立设置）
- 可设置全局播放速度
- 可设置入队位置（队尾/队首/当前播放之后）
- 可设置智能标记已播阈值，支持禁用
- 可设置耳机/蓝牙连接后自动恢复播放（版本受限）
- 设置修改后立即应用到播放器
- 秒数选项显示格式化（秒/分钟）
