# 单集总列表SPEC

聚合展示用户订阅的所有节目下的全部单集，支持分页加载、按维度排序、按状态筛选、以及多选批量操作；并提供下拉刷新与单项操作入口。

## 场景一

### 场景概述

用户进入"单集"总列表页，系统聚合所有订阅节目的单集，按默认排序分页加载首屏数据并展示。

### 场景逻辑步骤

- 1. 用户通过抽屉导航中的"单集"入口，或首页"惊喜单集"卡片上的"更多"入口，进入"单集总列表页"
- 2. 系统从数据库中聚合读取所有订阅节目下的单集，按当前保存的排序方式与筛选条件，读取第 1 页数据（每页 150 条）
- 3. 同时读取满足当前筛选条件的单集总数
- 4. 在列表中展示首屏单集；当筛选条件为空时展示"暂无单集"空状态文案，当筛选条件非空且无匹配项时展示"筛选后无单集"空状态文案
- 5. 首屏加载完成后恢复上次离开时的滚动位置

## 场景二

### 场景概述

用户在"单集总列表页"向下滚动到底部时，系统自动加载下一页单集并追加到列表末尾。

### 场景逻辑步骤

- 1. 用户在"单集总列表页"持续向下滚动
- 2. 当滚动接近列表底部，且仍存在未加载的更多数据时，系统自动发起下一页加载（每页 150 条）
- 3. 在加载过程中列表末尾展示一个加载占位项
- 4. 新一页数据加载完成后追加到列表末尾并移除占位项；若返回数据不足一页（少于 150 条），则标记为已无更多数据，不再继续分页加载
- 5. 加载失败时移除占位项并记录错误，列表保持已有内容

## 场景三

### 场景概述

用户通过"排序"入口选择单集的排序维度（日期或时长）及升降序，选择后持久化并刷新列表。

### 场景逻辑步骤

- 1. 用户在"单集总列表页"顶部工具栏点击"排序"
- 2. 弹出排序选择面板，展示可选维度：日期（旧→新 / 新→旧）、时长（短→长 / 长→短）
- 3. 用户点击某一排序选项（首次选择某维度时取该维度的默认升降序，再次点击同维度则切换升降序）
- 4. 系统将所选排序方式持久化保存，并触发明细列表刷新，按新排序重新读取单集数据

## 场景四

### 场景概述

用户通过"筛选"入口按状态筛选单集，可多组叠加；更改后重置到第 1 页并刷新，支持一键重置全部筛选。

### 场景逻辑步骤

- 1. 用户在"单集总列表页"顶部工具栏点击"筛选"，或点击列表上方的筛选信息条
- 2. 弹出筛选面板，展示以下可切换的状态分组，每组为二选一或清除：已播 / 未播、暂停 / 未暂停、收藏 / 非收藏、有媒体 / 无媒体、在队列 / 不在队列、已下载 / 未下载
- 3. 用户切换任意分组的状态（可同时启用多个分组，每个分组内可选其中一项或不选）
- 4. 系统立即按当前所选筛选条件重新从数据库读取第 1 页数据，并刷新筛选信息条以反映已生效的筛选；同时更新单集总数
- 5. 若用户在筛选面板点击"重置"，则清除全部筛选条件、恢复为展示全部单集，并刷新列表

## 场景五

### 场景概述

用户在"单集总列表页"长按单集进入多选模式，对所选单集批量执行操作（如标记已播、下载等）。

### 场景逻辑步骤

- 1. 用户在"单集总列表页"长按任一单集
- 2. 系统进入多选模式，浮出悬浮操作菜单，菜单包含可对所选单集批量执行的若干操作（删除、下载、标记未播、标记已播、移出队列、加入队列、分享、移出收件箱、加入收藏、取消收藏、重置播放进度等）
- 3. 用户继续点击其他单集以追加选择（或在懒加载模式下选择后续所有未加载页）
- 4. 用户点击悬浮菜单中的某个操作
- 5. 系统对该批所选单集执行对应操作并写入数据库，完成后退出多选模式，并通过提示展示操作结果（如已加入队列 N 集）

## 场景六

### 场景概述

用户在多选模式下对大批量单集执行"标记已播"或"标记未播"时，系统进行二次确认以防误操作。

### 场景逻辑步骤

- 1. 用户在多选模式下选择了较多单集（达到 25 集或以上），或在懒加载模式下勾选了后续所有未加载页
- 2. 用户点击悬浮菜单中的"标记已播"或"标记未播"
- 3. 系统弹出二次确认对话框，提示将对大量单集执行该操作
- 4. 用户点击"确认"后系统对该批单集执行操作并写入数据库；若用户取消则不执行任何操作

## 场景七

### 场景概述

用户在多选模式下未选择任何单集即点击悬浮操作菜单中的操作时，系统提示未选择项。

### 场景逻辑步骤

- 1. 用户进入多选模式但未选中任何单集
- 2. 用户点击悬浮操作菜单中的某个操作
- 3. 系统弹出提示"未选择任何项"，不执行该操作

## 场景八

### 场景概述

用户在"单集总列表页"通过下拉刷新或工具栏"刷新"入口触发节目更新。

### 场景逻辑步骤

- 1. 用户在"单集总列表页"顶部下拉，或点击工具栏"刷新"
- 2. 系统触发节目源更新（必要时征得用户同意），并在刷新过程中展示刷新指示
- 3. 节目更新完成后，单集列表自动重新加载以反映最新数据

## 场景九

### 场景概述

"单集总列表"功能的作用范围与不作用范围。

### 场景逻辑步骤

- 1. **仅作用于** 以下入口进入的"单集总列表页"：抽屉导航中的"单集"入口、首页"惊喜单集"卡片上的"更多"入口
- 2. **不作用于** 以下位置：首页"惊喜单集"卡片本身（仅展示随机单集子集，不展示全部单集，也不提供本页的排序/筛选/批量操作）；其他节目详情页、队列页、下载页等独立页面

> [偏差] 多选悬浮菜单中"移到顶部""移到底部"两项在本页默认不展示（仅在队列等特定页面可见），故未纳入本页场景。
