# SPEC: REQ-039 错误状态与重试

## 需求概述

应用在各类操作中（下载、Feed 解析、网络请求、同步）遇到错误时，应提供清晰的错误信息显示和重试机制。错误信息包含错误类型、用户可读的描述和详细技术信息。单集下载失败自动重试最多 3 次，不可恢复错误立即失败。所有错误记录到下载日志中，用户可查看历史和重试。

## 原子场景

### 场景 39.1：单集下载失败自动重试（Happy Path）
**触发**: 单集下载过程中遇到可恢复错误
**执行步骤**:
1. 应用尝试下载单集
2. 下载失败（如网络波动、HTTP 数据错误）
3. 应用显示"正在重试"提示
4. 自动重试下载（最多 3 次尝试）
5. 若重试中某次成功 → 下载完成
6. 若 3 次均失败 → 发送下载失败通知

### 场景 39.2：不可恢复错误立即失败（边界场景）
**触发**: 下载遇到不可恢复的错误
**执行步骤**:
1. 下载请求返回特定错误（禁止访问、资源不存在、未授权、连接被阻止）
2. 应用判断为不可恢复错误
3. 不执行重试，立即标记为失败
4. 发送下载失败通知
5. 记录到下载日志

### 场景 39.3：HTTP 416 范围请求错误（边界场景）
**触发**: 下载请求了无效的文件范围
**执行步骤**:
1. 服务器返回 HTTP 416（范围请求无效）
2. 应用删除已下载的部分文件
3. 从文件开头重新下载
4. 自动重试

### 场景 39.4：查看下载日志（Happy Path）
**触发**: 用户查看下载历史和错误
**执行步骤**:
1. 用户打开下载日志页面
2. 列表显示所有下载记录（成功和失败）
3. 每条记录显示：时间、单集标题、状态
4. 失败记录显示错误类型和用户可读描述
5. 用户可点击查看详细错误信息

### 场景 39.5：从下载日志重试（Happy Path）
**触发**: 用户从下载日志中重试失败的下载
**执行步骤**:
1. 用户在下载日志中找到失败记录
2. 点击重试按钮
3. 应用重新发起下载请求
4. 下载过程中正常执行重试机制

### 场景 39.6：RSS Feed 解析失败（异常场景）
**触发**: Feed 内容下载成功但无法解析
**执行步骤**:
1. Feed 内容下载完成
2. XML 解析器报解析异常
3. 记录解析错误（ERROR_PARSER_EXCEPTION）
4. 错误信息记录到下载日志
5. 用户可查看错误详情

### 场景 39.7：Feed 订阅时的网络错误（异常场景）
**触发**: 添加新订阅时网络不可达
**执行步骤**:
1. 用户输入 Feed URL 尝试订阅
2. 网络请求失败（连接超时、未知主机）
3. 显示错误对话框，提示网络错误
4. 提供重试入口
5. 用户可重试或编辑 URL

### 场景 39.8：下载空间不足（异常场景）
**触发**: 设备存储空间不足无法下载
**执行步骤**:
1. 下载时检测到存储空间不足
2. 记录 ERROR_NOT_ENOUGH_SPACE 错误
3. 显示"空间不足"错误提示
4. 建议用户清理空间或修改缓存设置
5. 记录到下载日志

### 场景 39.9：需认证的下载（异常场景）
**触发**: 下载需要认证但未提供凭证
**执行步骤**:
1. 下载请求返回 HTTP 401（未授权）
2. 记录 ERROR_UNAUTHORIZED 错误
3. 显示"需要认证"错误提示
4. 引导用户在播客设置中配置用户名密码

### 场景 39.10：证书错误（异常场景）
**触发**: 下载时 SSL 证书验证失败
**执行步骤**:
1. HTTPS 连接时证书验证失败
2. 记录 ERROR_CERTIFICATE 错误
3. 显示"证书错误"提示
4. 不自动重试，需用户检查网络环境

### 场景 39.11：同步失败（异常场景）
**触发**: 同步服务连接失败
**执行步骤**:
1. 同步任务执行失败
2. 记录同步错误状态
3. 同步设置页面显示"同步失败"
4. 显示上次同步时间
5. 用户可手动重试同步

### 场景 39.12：未知错误类型（边界场景）
**触发**: 下载遇到未映射的错误代码
**执行步骤**:
1. 错误代码不在已知映射表中
2. 显示"未知错误"提示
3. 记录到下载日志
4. 调试模式下可能抛出异常

## 验收要点
- 下载失败自动重试最多 3 次
- 不可恢复错误（禁止/不存在/未授权）立即失败不重试
- HTTP 416 错误自动删除部分文件重试
- 所有下载结果（成功/失败）记录到日志
- 每种错误有用户可读的本地化描述
- 下载日志可查看历史和详细错误信息
- 可从下载日志重试失败下载
- Feed 解析失败有清晰错误提示
- 空间不足、认证缺失、证书错误等特殊场景有专属错误提示
- 同步失败状态可见且可重试
