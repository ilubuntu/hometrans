# SPEC: REQ-038 分享

## 需求概述

用户可以通过系统分享面板分享播客订阅链接和单集信息。支持分享播客订阅深度链接、单集网页链接、单集媒体文件链接（带或不带播放位置）、以及已下载的媒体文件。分享文本自动组装播客名称、单集标题、网页链接和媒体链接。长文本支持缩写。

## 原子场景

### 场景 38.1：分享单集网页链接（Happy Path）
**触发**: 用户在单集详情页选择分享网页链接
**执行步骤**:
1. 用户在单集详情页点击分享按钮
2. 弹出分享选项对话框
3. 用户选择"分享单集链接"
4. 应用组装分享文本：播客名称 + 单集标题 + 网页链接
5. 打开系统分享面板
6. 用户选择分享目标应用
7. 链接发送到目标应用

### 场景 38.2：分享带播放位置的媒体链接（Happy Path）
**触发**: 用户选择分享带当前播放位置的媒体链接
**执行步骤**:
1. 用户在单集详情页点击分享
2. 选择"分享媒体链接（带位置）"
3. 应用组装分享文本：
   - 播客名称 + 单集标题
   - 起始位置：HH:MM:SS
   - 媒体文件 URL + 播放位置锚点 (#t=秒数)
4. 打开系统分享面板
5. 接收方可通过链接跳转到指定播放位置

### 场景 38.3：分享播客订阅链接（Happy Path）
**触发**: 用户在播客详情页分享播客
**执行步骤**:
1. 用户在播客详情页点击分享按钮
2. 应用生成订阅深度链接
3. 链接格式包含播客 Feed URL 和标题
4. 打开系统分享面板
5. 接收方可通过链接快速订阅该播客

### 场景 38.4：分享已下载的媒体文件（Happy Path）
**触发**: 用户分享已下载的音频/视频文件
**执行步骤**:
1. 单集已下载且用户选择"分享文件"
2. 应用获取文件的临时读取 URI
3. 打开系统分享面板，附带文件流
4. 分享类型为媒体 MIME 类型
5. 接收方可直接接收文件

### 场景 38.5：单集没有网页链接（边界场景）
**触发**: 单集没有可分享的网页链接
**执行步骤**:
1. 应用检查单集是否有网页链接
2. 若无网页链接，分享文本中不包含链接部分
3. 仅分享播客名称和标题（及媒体链接如有）

### 场景 38.6：长标题缩写（边界场景）
**触发**: 单集标题或链接过长
**执行步骤**:
1. 分享文本组装时检测长度
2. 超过 50 字符的标题或链接截断
3. 添加省略号（…）
4. 保持分享文本简洁

### 场景 38.7：没有可下载媒体的单集（边界场景）
**触发**: 单集没有媒体文件下载地址
**执行步骤**:
1. 应用检查单集是否有媒体文件
2. 若无媒体文件，分享文本不包含媒体链接
3. 仅分享标题和网页链接

### 场景 38.8：系统分享面板未弹出（异常场景）
**触发**: 系统中没有可接收分享的应用
**执行步骤**:
1. 应用尝试打开系统分享面板
2. 若无应用可处理分享 Intent
3. 系统显示"没有应用可执行此操作"提示
4. 不崩溃

## 验收要点
- 可分享单集网页链接，文本包含播客名称和标题
- 可分享带播放位置的媒体链接（时间锚点）
- 可分享播客订阅深度链接
- 可分享已下载的媒体文件
- 长文本自动缩写（50 字符截断加省略号）
- 无网页链接时不包含链接部分
- 无媒体文件时不包含媒体链接
- 通过系统分享面板选择目标应用
