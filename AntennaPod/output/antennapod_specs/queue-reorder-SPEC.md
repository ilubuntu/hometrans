# REQ-017 队列排序与移动 - 需求规格

## 概述
用户可以在队列页面通过拖拽或上下文菜单重新排列队列中剧集的顺序。支持拖拽到任意位置，以及快速移动单个或多个剧集到队列顶部或底部。队列锁定或保持排序模式下禁用手动排序。

## 原子场景

### 场景 1：拖拽移动剧集到任意位置（正常流程）
**触发**：用户长按某剧集的拖拽手柄并上下拖动

**执行步骤**：
1. 检测到拖拽手柄的按下事件
2. 启动拖拽模式，该剧集项变为可拖动状态
3. 用户上下拖动时：
   a. 实时在内存列表中移动该剧集的位置
   b. 列表实时反映新的排列顺序
4. 松手时：
   a. 检测拖拽起始位置和目标位置是否不同
   b. 将移动操作持久化到数据库
   c. 发送移动事件通知其他组件

### 场景 2：通过封面区域拖拽
**触发**：用户长按某剧集封面区域的左半部分

**执行步骤**：
1. 检测到封面左半区域的按下事件
2. 启动拖拽模式（同场景 1）
3. 封面右半区域的按下不触发拖拽（避免误触）

### 场景 3：通过上下文菜单移动到顶部
**触发**：用户长按某剧集，在弹出菜单中选择"移到顶部"

**执行步骤**：
1. 弹出上下文菜单
2. 若该剧集已在顶部：隐藏"移到顶部"选项
3. 若队列开启保持排序：隐藏移动选项
4. 选择"移到顶部"：
   a. 从当前位置移除该剧集
   b. 插入到队列索引 0 的位置
   c. 持久化到数据库
   d. 列表更新显示
5. 信息栏更新集数和时长汇总

### 场景 4：通过上下文菜单移动到底部
**触发**：用户长按某剧集，在弹出菜单中选择"移到底部"

**执行步骤**：
1. 弹出上下文菜单
2. 若该剧集已在底部：隐藏"移到底部"选项
3. 若队列开启保持排序：隐藏移动选项
4. 选择"移到底部"：
   a. 从当前位置移除该剧集
   b. 追加到队列末尾
   c. 持久化到数据库
   d. 列表更新显示

### 场景 5：多选批量移动到顶部
**触发**：用户在多选模式下选中多个剧集，点击"移到顶部"

**执行步骤**：
1. 系统检查选中项是否可以移到顶部（canMove 检查）
   a. 若选中项已连续位于顶部：隐藏"移到顶部"按钮
   b. 若选中项数量等于队列总数：隐藏移动按钮
2. 执行批量移动：
   a. 反转选中项顺序（保持原始相对顺序）
   b. 从队列中移除所有选中项
   c. 依次插入到队列顶部
3. 持久化到数据库
4. 列表刷新显示

### 场景 6：多选批量移动到底部
**触发**：用户在多选模式下选中多个剧集，点击"移到底部"

**执行步骤**：
1. 系统检查选中项是否可以移到底部
   a. 若选中项已连续位于底部：隐藏"移到底部"按钮
2. 执行批量移动：
   a. 从队列中移除所有选中项
   b. 依次追加到队列末尾
3. 持久化到数据库
4. 列表刷新显示

### 场景 7：队列锁定时禁用排序
**触发**：队列处于锁定状态时用户尝试拖拽

**执行步骤**：
1. 拖拽手柄不可见
2. 封面区域的拖拽触摸监听被移除
3. 上下文菜单中的移动选项隐藏
4. 用户无法手动排序

### 场景 8：队列保持排序时禁用排序
**触发**：队列处于"保持排序"模式时

**执行步骤**：
1. 拖拽手柄不可见
2. 上下文菜单中的移动选项隐藏
3. 队列顺序由排序规则自动管理
