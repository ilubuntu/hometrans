# SPEC: REQ-008 单集详情页

## 需求概述

点击单集后进入单集详情页，展示单集标题、播客名称、发布时间、封面、描述（支持 HTML 渲染和时间码跳转）。页面提供播放/流式播放、下载/取消下载/删除、加入队列、收藏、分享等操作按钮。操作按钮根据单集状态动态切换。详情页支持左右滑动切换相邻单集。点击播客名称或封面可跳转到播客详情页。

## 原子场景

### 场景 8.1：查看单集详情页（Happy Path）
**触发**: 用户从单集列表点击某个单集
**执行步骤**:
1. 导航到单集详情页
2. 异步加载单集数据（含关联的播客和媒体信息）
3. 展示单集标题
4. 展示所属播客名称
5. 展示发布时间
6. 展示封面图片
7. 展示描述（HTML 渲染）
8. 展示时长（若有媒体且有时长数据）

### 场景 8.2：描述区域时间码跳转（Happy Path）
**触发**: 用户点击描述中的时间码
**执行步骤**:
1. 用户在描述区域看到可点击的时间码
2. 点击时间码
3. 若当前正在播放同一单集 → 播放进度跳转到该时间码位置
4. 若未在播放同一单集 → 提示"需先播放此单集才能使用进度跳转"

### 场景 8.3：播放操作 - 已下载单集（Happy Path）
**触发**: 用户点击已下载单集的播放按钮
**执行步骤**:
1. 操作按钮 1 显示为"播放"
2. 用户点击播放按钮
3. 启动本地媒体播放
4. 按钮切换为"暂停"状态
5. 播放状态变化时按钮实时更新

### 场景 8.4：播放操作 - 未下载单集（流式播放）（Happy Path）
**触发**: 用户点击未下载单集的播放按钮
**执行步骤**:
1. 操作按钮 1 显示为"流式播放"
2. 用户点击按钮
3. 启动在线流式播放
4. 播放状态变化时按钮实时更新

### 场景 8.5：暂停操作（Happy Path）
**触发**: 正在播放的单集，用户点击暂停
**执行步骤**:
1. 操作按钮 1 显示为"暂停"（当前正在播放）
2. 用户点击暂停按钮
3. 播放暂停
4. 按钮切换回"播放"状态

### 场景 8.6：下载操作（Happy Path）
**触发**: 用户点击未下载单集的下载按钮
**执行步骤**:
1. 操作按钮 2 显示为"下载"
2. 用户点击下载按钮
3. 开始下载，显示下载进度环形指示器
4. 下载进行中按钮切换为"取消下载"
5. 下载完成后按钮切换为"删除下载"
6. 下载进度实时更新

### 场景 8.7：取消下载操作（Happy Path）
**触发**: 正在下载的单集，用户点击取消
**执行步骤**:
1. 操作按钮 2 显示为"取消下载"
2. 用户点击取消按钮
3. 下载中止
4. 按钮切换回"下载"

### 场景 8.8：删除下载操作（Happy Path）
**触发**: 已下载的单集，用户点击删除下载
**执行步骤**:
1. 操作按钮 2 显示为"删除下载"
2. 用户点击删除按钮
3. 删除本地媒体文件（保留单集元数据）
4. 按钮切换回"下载"

### 场景 8.9：本地 Feed 单集播放（边界场景）
**触发**: 本地文件夹播客的单集播放
**执行步骤**:
1. 操作按钮 1 显示为"本地播放"
2. 用户点击按钮
3. 直接播放本地文件

### 场景 8.10：无媒体单集详情页（边界场景）
**触发**: 某单集没有关联媒体文件
**执行步骤**:
1. 操作按钮 1 显示为"标记已播放"
2. 操作按钮 2 显示为"访问网站"
3. 隐藏时长显示
4. 显示"无媒体"提示
5. 页面不崩溃

### 场景 8.11：智能行为配置提示（Happy Path）
**触发**: 用户的操作习惯与默认行为设置不一致
**执行步骤**:
1. 基于使用统计分析用户偏好（如偏好流式播放但点击了下载）
2. 检测到偏好与当前操作不一致
3. 弹出气球提示，询问是否切换默认行为
4. 用户可选择切换或忽略

### 场景 8.12：点击播客名称跳转（Happy Path）
**触发**: 用户点击详情页的播客名称
**执行步骤**:
1. 用户点击播客名称文本
2. 若播客已订阅 → 导航到播客详情页
3. 若播客未订阅 → 进入在线 Feed 预览页

### 场景 8.13：点击封面跳转（Happy Path）
**触发**: 用户点击详情页的封面图片
**执行步骤**:
1. 用户点击封面
2. 导航到该单集所属的播客详情页
3. 返回后回到单集详情页

### 场景 8.14：复制标题和播客名（Happy Path）
**触发**: 用户长按标题或播客名称
**执行步骤**:
1. 长按标题文本 → 复制标题到剪贴板
2. 长按播客名称 → 复制播客名称到剪贴板
3. 显示复制成功提示

### 场景 8.15：上下文菜单 / 更多操作（Happy Path）
**触发**: 用户打开详情页的更多操作菜单
**执行步骤**:
1. 弹出操作菜单
2. 菜单选项包括：
   - 加入队列 / 移出队列
   - 收藏 / 取消收藏
   - 分享单集
   - 标记已播放 / 标记未播放
   - 删除下载
   - 访问网站
3. 选择操作后执行并关闭菜单

### 场景 8.16：左右滑动切换相邻单集（Happy Path）
**触发**: 用户在单集详情页左右滑动
**执行步骤**:
1. 向左/右滑动切换到相邻的单集详情页
2. 新页面加载对应单集数据
3. 标题、播客名、描述、操作按钮全部更新
4. 滑动到列表边界时不可继续滑动

### 场景 8.17：状态实时更新（Happy Path）
**触发**: 单集状态在其他位置发生变化
**执行步骤**:
1. 应用收到状态变化通知（下载事件、播放状态变化、单集事件等）
2. 详情页操作按钮实时更新
3. 下载进度、播放状态同步刷新
4. 无需手动刷新页面

### 场景 8.18：返回原列表（Happy Path）
**触发**: 用户从单集详情页返回
**执行步骤**:
1. 用户点击返回
2. 回到进入详情前的单集列表页面
3. 列表状态保持（滚动位置、选中状态等）

## 验收要点
- 详情页可打开，展示标题、播客名称、发布时间、封面、描述
- 主要操作按钮存在（播放/下载/暂停等）且根据状态动态切换
- 描述支持 HTML 渲染和时间码跳转
- 点击播客名称或封面可跳转到播客详情页
- 支持左右滑动切换相邻单集
- 支持加入队列、收藏、分享等操作
- 长按可复制标题和播客名称
- 返回后回到原列表，状态保持
- 状态变化时按钮实时更新
- 无媒体单集正确处理
