# SPEC: REQ-033 主题与外观设置

## 需求概述

用户可以自定义应用的外观和导航方式，包括：主题模式（跟随系统/浅色/深色）、纯黑主题、动态取色（基于壁纸）、剩余时间/已播放时间显示切换、导航抽屉条目显隐、底部导航/侧边抽屉切换、通知栏快捷按钮自定义、剧集列表全局排序方式、滑动手势配置、优先流式播放/下载按钮选择。外观修改后立即生效。

## 原子场景

### 场景 33.1：切换主题模式（Happy Path）
**触发**: 用户更改应用主题
**执行步骤**:
1. 用户进入设置 → 用户界面 → 主题
2. 显示主题选择器，可选：跟随系统、浅色、深色
3. 用户选择一个主题
4. 应用立即重建页面，应用新主题
5. 所有页面颜色立即更新

### 场景 33.2：开启纯黑主题（Happy Path）
**触发**: 用户在深色模式下开启纯黑背景
**执行步骤**:
1. 用户切换"纯黑主题"开关
2. 应用立即重建页面
3. 深色模式下背景变为纯黑
4. 节省 OLED 屏幕电量

### 场景 33.3：开启动态取色（Happy Path）
**触发**: 用户（Android 12+）开启基于壁纸的动态取色
**执行步骤**:
1. 用户切换"动态取色"开关
2. 应用立即重建页面
3. 主题颜色根据系统壁纸动态调整
4. 注意：仅特定系统版本以上显示此选项

### 场景 33.4：切换时间显示方式（Happy Path）
**触发**: 用户将时间显示从"已播放时间"切换为"剩余时间"
**执行步骤**:
1. 用户切换"显示剩余时间"开关
2. 应用广播状态更新事件
3. 所有剧集列表项的时间标签立即刷新
4. 播放器界面时间显示同步更新
5. 显示剩余时长而非已播放时长

### 场景 33.5：自定义导航抽屉条目（Happy Path）
**触发**: 用户隐藏不需要的导航抽屉条目
**执行步骤**:
1. 用户点击"导航抽屉条目"设置
2. 弹出对话框，列出所有可用导航条目
3. 每个条目有显示/隐藏开关
4. 用户勾选要隐藏的条目
5. 确认后导航抽屉仅显示选中的条目

### 场景 33.6：自定义通知栏快捷按钮（Happy Path）
**触发**: 用户设置播放通知栏的快捷按钮
**执行步骤**:
1. 用户点击"通知栏按钮"设置
2. 弹出多选对话框，列出可选按钮：跳过、下一章节、播放速度、睡眠定时器
3. 用户必须选择恰好 2 个按钮
4. 若选择数量不等于 2，显示错误提示
5. 选择 2 个后确认保存
6. 播放通知栏显示自定义的 2 个快捷按钮

### 场景 33.7：设置全局默认排序（Happy Path）
**触发**: 用户设置剧集列表的默认排序方式
**执行步骤**:
1. 用户点击"默认排序"设置
2. 弹出排序选项对话框
3. 用户选择排序方式（如按日期、标题、时长等）
4. 确认后所有剧集列表使用新的默认排序

### 场景 33.8：切换底部导航和侧边抽屉（Happy Path）
**触发**: 用户切换导航模式
**执行步骤**:
1. 用户切换"底部导航"开关
2. 若关闭，显示弃用警告对话框
3. 确认后切换导航模式
4. 开启时显示底部导航栏，"返回键打开抽屉"设置变为不可用
5. 关闭时使用侧边抽屉导航

### 场景 33.9：切换优先流式播放（Happy Path）
**触发**: 用户更改默认操作按钮偏好
**执行步骤**:
1. 用户切换"优先流式播放"开关
2. 应用刷新所有剧集列表
3. 开启时列表项默认按钮变为"流式播放"
4. 关闭时默认按钮变为"下载"
5. 不再显示流式播放建议提示

### 场景 33.10：配置滑动手势（Happy Path）
**触发**: 用户自定义列表项滑动手势动作
**执行步骤**:
1. 用户点击"滑动手势"设置
2. 打开滑动手势配置子页面
3. 分别配置左滑和右滑的动作
4. 可选动作：加入队列、加入收藏、标记已播等
5. 确认后列表项滑动手势使用新配置

### 场景 33.11：动态取色在低版本不可用（边界场景）
**触发**: 低版本系统用户查看动态取色选项
**执行步骤**:
1. 检测系统版本低于阈值
2. "动态取色"选项不显示
3. 用户无法启用该功能

## 验收要点
- 支持跟随系统、浅色、深色三种主题模式，切换后立即生效
- 支持纯黑主题和动态取色（版本受限）
- 可切换时间显示方式（剩余/已播放），全局立即刷新
- 可自定义导航抽屉显示条目
- 可自定义通知栏 2 个快捷按钮（必须恰好选 2 个）
- 可设置全局默认排序方式和滑动手势
- 可切换底部导航和侧边抽屉
- 可切换默认操作按钮（流式播放/下载）
- 外观修改后立即生效，无需重启应用
