# SPEC: REQ-030 标签与分组

## 需求概述

用户可以为订阅的播客设置一个或多个自定义标签，实现分组管理。标签在导航抽屉中作为分组显示，用户可按标签过滤和浏览订阅列表。支持添加新标签、从已有标签中选择、删除标签，以及批量为多个播客设置标签。标签数据持久化存储。

## 原子场景

### 场景 30.1：为单个播客添加标签（Happy Path）
**触发**: 用户在播客设置中为播客添加标签
**执行步骤**:
1. 用户在播客设置页点击"标签"设置项
2. 弹出标签编辑对话框
3. 显示该播客已有的标签（芯片形式排列）
4. 用户在输入框中输入新标签名称
5. 输入时从已有标签列表自动补全提示
6. 确认添加后标签以芯片形式显示在列表中
7. 用户可继续添加多个标签
8. 点击确认后保存所有标签到本地

### 场景 30.2：删除已有标签（Happy Path）
**触发**: 用户移除播客的某个标签
**执行步骤**:
1. 标签编辑对话框中显示已有标签芯片
2. 每个芯片有删除按钮
3. 用户点击删除按钮移除标签
4. 芯片从列表中消失
5. 确认后保存更新后的标签集合

### 场景 30.3：为多个播客批量设置标签（Happy Path）
**触发**: 用户在订阅列表中多选播客后设置标签
**执行步骤**:
1. 用户在订阅列表中进入多选模式
2. 选择多个播客
3. 点击"设置标签"操作
4. 弹出标签编辑对话框
5. 仅显示所有选中播客的公共标签（交集）
6. 如有多个播客，显示提示信息说明仅显示公共标签
7. 用户添加或删除标签
8. 确认后所有选中播客更新标签（移除原公共标签，添加新标签）

### 场景 30.4：按标签过滤浏览（Happy Path）
**触发**: 用户在导航抽屉中点击某个标签
**执行步骤**:
1. 导航抽屉中按标签分组显示订阅
2. 每个标签显示标题和关联播客数量
3. 用户点击某标签
4. 展开显示该标签下的所有播客
5. 用户可选择某个播客进入详情页

### 场景 30.5：设置根标签（Happy Path）
**触发**: 用户将播客标记为根级显示
**执行步骤**:
1. 标签编辑对话框中有"根文件夹"复选框
2. 仅在非底部导航模式下显示该选项
3. 用户勾选后该播客显示在导航根目录
4. 确认后保存

### 场景 30.6：标签名称重复校验（边界场景）
**触发**: 用户尝试添加已存在的标签
**执行步骤**:
1. 用户输入的标签名已存在于当前列表中
2. 不添加重复标签
3. 输入框保持焦点等待用户修正

### 场景 30.7：未分类标签显示（边界场景）
**触发**: 播客没有设置任何标签
**执行步骤**:
1. 系统自动将无标签播客归入"未分类"分组
2. 导航抽屉中显示"未分类"条目
3. 用户可展开查看所有未设置标签的播客

### 场景 30.8：空标签名称校验（边界场景）
**触发**: 用户尝试添加空名称的标签
**执行步骤**:
1. 用户输入为空或纯空格
2. 不添加标签
3. 输入框保持焦点

## 验收要点
- 可为单个播客添加、删除多个自定义标签
- 支持多选播客批量设置标签（公共标签交集处理）
- 导航抽屉按标签分组显示，每组显示播客数量
- 点击标签可展开查看该标签下的播客
- 支持根标签（在非底部导航模式下）
- 标签输入支持自动补全已有标签
- 无标签播客自动归入"未分类"
- 标签数据持久化存储
