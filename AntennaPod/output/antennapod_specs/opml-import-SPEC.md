# SPEC: REQ-031 OPML导入

## 需求概述

用户可以从 OPML 文件中批量导入播客订阅。应用解析 OPML 文件后，展示文件中发现的所有播客列表，用户可勾选需要导入的播客，确认后批量添加到订阅列表并自动刷新。支持从系统文件选择器选择文件，处理多种编码格式，并在解析失败时提供清晰的错误信息。

## 原子场景

### 场景 31.1：从文件选择器导入 OPML 文件（Happy Path）
**触发**: 用户在存储设置中点击"OPML 导入"并选择文件
**执行步骤**:
1. 用户进入设置 → 存储 → OPML 导入
2. 应用打开系统文件选择器
3. 用户选择一个 .opml 文件
4. 应用打开 OPML 导入页面
5. 显示加载进度条
6. 解析 OPML 文件，提取所有播客条目
7. 展示多选列表，每个条目显示播客名称
8. 默认所有条目已选中
9. 用户确认导入

### 场景 31.2：选择部分播客导入（Happy Path）
**触发**: 用户在导入列表中取消勾选部分播客
**执行步骤**:
1. 导入列表展示所有从 OPML 文件解析的播客
2. 用户取消勾选不需要导入的播客
3. 菜单显示"全选"和"取消全选"按钮
4. 用户点击"取消全选"后可手动勾选需要的播客
5. 点击确认后仅导入勾选的播客

### 场景 31.3：批量添加订阅（Happy Path）
**触发**: 用户确认导入选中的播客
**执行步骤**:
1. 用户点击确认按钮
2. 显示导入进度条
3. 应用遍历选中的播客条目
4. 为每个条目创建订阅记录并保存到本地数据库
5. 导入完成后自动刷新所有新订阅的 Feed
6. 导航到主页面，新订阅出现在列表中

### 场景 31.4：OPML 文件格式无效（异常场景）
**触发**: 选择的文件不是有效的 OPML 格式
**执行步骤**:
1. 应用尝试解析文件
2. XML 解析器报错
3. 隐藏进度条
4. 显示错误对话框，标题为"错误"
5. 错误信息包含用户可读的描述和详细技术信息（灰色显示）
6. 用户点击确定后关闭导入页面

### 场景 31.5：未选择文件（边界场景）
**触发**: 文件 URI 为空
**执行步骤**:
1. 应用检测到没有有效的文件 URI
2. 显示提示对话框："未选择文件"
3. 用户点击确定后返回

### 场景 31.6：OPML 条目缺少 xmlUrl（边界场景）
**触发**: OPML 文件中某些 outline 元素没有 xmlUrl 属性
**执行步骤**:
1. 解析器跳过没有 xmlUrl 的元素
2. 仅展示有有效 Feed URL 的播客条目
3. 用户正常选择和导入

### 场景 31.7：条目缺少标题的回退处理（边界场景）
**触发**: OPML 条目没有 title 或 text 属性
**执行步骤**:
1. 解析器首先尝试使用 title 属性
2. 若 title 为空，尝试使用 text 属性
3. 若 text 也为空，使用 xmlUrl 作为显示名称
4. 列表正常显示

### 场景 31.8：家长控制验证（边界场景）
**触发**: 启用了家长控制密码且订阅需验证
**执行步骤**:
1. 用户点击确认导入
2. 检测到家长控制已启用
3. 弹出密码验证对话框
4. 用户输入正确密码后继续导入流程
5. 密码错误则不执行导入

### 场景 31.9：编码处理（边界场景）
**触发**: OPML 文件使用 BOM 编码（如 UTF-16）
**执行步骤**:
1. 应用检测文件的 BOM 标记
2. 根据 BOM 确定正确的字符编码
3. 无 BOM 时默认使用 UTF-8
4. 使用正确的编码读取文件内容

### 场景 31.10：文件读取权限不足（异常场景）
**触发**: 应用没有读取所选文件的权限
**执行步骤**:
1. 文件读取失败，错误信息包含"permission"
2. 应用请求存储读取权限
3. 权限授予后重新尝试导入
4. 权限拒绝后显示提示，可选择再次请求或退出

## 验收要点
- 可通过系统文件选择器选择 OPML 文件
- 解析 OPML 文件并展示所有发现的播客列表
- 支持多选和全选/取消全选
- 批量导入选中的播客并自动刷新
- 支持 BOM 编码检测和多种字符集
- 解析失败时显示清晰的错误信息
- 条目缺少标题/xmlUrl 时有合理的回退处理
- 支持家长控制密码验证
- 导入完成后导航到主页面
