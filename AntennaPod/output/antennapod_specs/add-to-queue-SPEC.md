# REQ-015 加入播放队列 - 需求规格

## 概述
用户可以将剧集加入播放队列。加入队列的操作可以从剧集列表的上下文菜单、多选批量操作、或滑动操作中触发。加入后，剧集会根据用户配置的插入位置添加到队列中，队列页面实时显示更新。

## 原子场景

### 场景 1：从剧集上下文菜单加入队列（正常流程）
**触发**：用户在剧集列表中长按某剧集，选择"加入队列"

**执行步骤**：
1. 检查该剧集是否已有媒体且不在队列中
2. 若已在队列中：菜单项不可见
3. 若无媒体：菜单项不可见
4. 执行加入队列操作：
   a. 读取当前播放队列
   b. 根据用户配置的插入位置（队列顶部/底部/当前播放项之后）计算插入索引
   c. 将剧集插入到计算的位置
   d. 为剧集标记"队列"标签
   e. 若剧集是新状态，将其标记为未播放
   f. 若开启了队列自动排序，重新排序队列
5. 保存队列到数据库
6. 发送队列变更事件，队列页面实时更新列表
7. 剧集列表中的队列标记状态更新

### 场景 2：通过滑动操作加入队列
**触发**：用户在剧集列表中向左/右滑动某剧集

**执行步骤**：
1. 检测到滑动操作配置为"加入队列"
2. 若该剧集已在队列中：执行从队列移除操作
3. 若该剧集无媒体：显示"无媒体"提示消息
4. 否则：执行加入队列操作（同场景 1 步骤 4-7）

### 场景 3：多选批量加入队列
**触发**：用户在多选模式下选中多个剧集，点击"加入队列"操作

**执行步骤**：
1. 从选中的剧集中过滤出有媒体且不在队列中的
2. 批量执行加入队列操作
3. 计算每个剧集的插入位置并依次插入
4. 保存队列到数据库
5. 显示提示消息："已添加 N 集到队列"
6. 队列页面实时更新

### 场景 4：队列开启自动排序时加入
**触发**：用户开启了队列"保持排序"，然后加入新剧集

**执行步骤**：
1. 将剧集添加到队列
2. 根据排序规则（如日期、标题等）自动重新排序整个队列
3. 发送排序事件，队列页面整体刷新
4. 新添加的剧集显示在排序后的正确位置

### 场景 5：重复加入已在队列中的剧集
**触发**：用户尝试将已在队列中的剧集再次加入

**执行步骤**：
1. 检测到该剧集已标记"队列"标签
2. 跳过该剧集，不做任何修改
3. 剧集在队列中的位置不变
