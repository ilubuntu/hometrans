# SPEC: REQ-036 同步设置

## 需求概述

用户可以配置播客同步服务，将订阅列表和播放进度在多设备间同步。支持 gpodder.net 和 Nextcloud Gpodder 两种同步服务。未登录时显示"选择同步服务"入口；登录后可查看服务信息、修改密码、手动触发同步、强制全量同步、登出。同步状态实时显示。

## 原子场景

### 场景 36.1：选择并登录同步服务（Happy Path）
**触发**: 未登录用户首次配置同步服务
**执行步骤**:
1. 用户进入设置 → 同步
2. 页面显示"选择同步服务"标题和说明
3. 用户点击该入口
4. 弹出同步服务选择器，列出可选服务（带图标和名称）
5. 用户选择 gpodder.net 或 Nextcloud Gpodder
6. 打开对应的登录认证对话框
7. 用户输入服务器地址、用户名、密码
8. 验证成功后保存凭证
9. 页面更新为已登录状态

### 场景 36.2：已登录查看同步状态（Happy Path）
**触发**: 已登录用户查看同步设置页面
**执行步骤**:
1. 页面显示当前同步服务的图标和名称
2. 显示登录用户名和服务器地址
3. 标题栏显示上次同步时间和结果（成功/失败）
4. "立即同步"、"强制全量同步"、"登出"按钮可用
5. "修改密码"按钮可用

### 场景 36.3：手动触发同步（Happy Path）
**触发**: 用户点击"立即同步"
**执行步骤**:
1. 用户点击"立即同步"按钮
2. 应用将同步任务加入队列
3. 同步服务执行增量同步（仅同步变化部分）
4. 同步状态栏实时显示进度（同步中）
5. 同步完成后显示成功或失败状态
6. 更新上次同步时间

### 场景 36.4：强制全量同步（Happy Path）
**触发**: 用户点击"强制全量同步"
**执行步骤**:
1. 用户点击"强制全量同步"按钮
2. 应用将全量同步任务加入队列
3. 同步服务重新同步所有数据（非增量）
4. 同步状态栏显示进度
5. 完成后更新状态

### 场景 36.5：修改登录密码（Happy Path）
**触发**: 用户修改同步服务密码
**执行步骤**:
1. 用户点击"设置登录信息"
2. 弹出对话框，显示当前用户名（只读）和密码输入框
3. 用户输入新密码
4. 确认后保存新密码到本地
5. 后续同步使用新密码

### 场景 36.6：登出同步服务（Happy Path）
**触发**: 用户登出当前同步服务
**执行步骤**:
1. 用户点击"登出"按钮
2. 应用清除本地存储的同步凭证
3. 清除同步任务队列
4. 显示"已登出"提示
5. 清除当前选中的同步服务标识
6. 页面恢复到未登录状态（显示"选择同步服务"入口）

### 场景 36.7：同步失败的错误显示（异常场景）
**触发**: 同步服务连接失败或认证失败
**执行步骤**:
1. 同步任务执行失败
2. 同步状态栏显示"同步失败"
3. 显示上次同步时间和失败状态
4. 用户可重试同步

### 场景 36.8：同步进行中的状态显示（边界场景）
**触发**: 同步任务正在执行
**执行步骤**:
1. 同步状态事件触发页面更新
2. 标题栏显示"同步中"状态
3. 同步完成或失败后更新为最终状态

### 场景 36.9：登录认证失败（异常场景）
**触发**: 用户输入错误的用户名或密码
**执行步骤**:
1. 认证对话框提交登录请求
2. 服务器返回认证失败
3. 显示错误提示（如"用户名或密码错误"）
4. 用户可修正后重试

### 场景 36.10：Nextcloud 登录（Happy Path）
**触发**: 用户选择 Nextcloud Gpodder 服务
**执行步骤**:
1. 用户在服务选择器选择 Nextcloud Gpodder
2. 打开 Nextcloud 认证对话框
3. 输入 Nextcloud 服务器地址
4. 输入用户名和 App 密码
5. 验证成功后保存凭证

## 验收要点
- 未登录时显示"选择同步服务"入口
- 支持 gpodder.net 和 Nextcloud Gpodder 两种同步服务
- 登录后可手动触发同步和强制全量同步
- 同步状态实时显示（进度、结果、时间）
- 可修改登录密码
- 可登出并清除凭证
- 登录失败有清晰的错误提示
- 登出后恢复未登录状态
