# SPEC: REQ-004 通过稳定 RSS feed 订阅播客

## 需求概述

用户输入稳定的播客 RSS Feed URL 后，应用应下载并解析 Feed 信息，展示播客预览（名称、封面、描述和单集列表），并允许用户确认订阅。订阅成功后进入该播客详情页或订阅列表。对于无效 URL、网络失败、解析失败等情况应提供错误信息和重试入口。

## 原子场景

### 场景 4.1：输入有效 RSS URL 完成订阅（Happy Path）
**触发**: 用户输入一个有效的 RSS Feed URL 并确认
**执行步骤**:
1. 用户通过添加播客页面输入 RSS URL
2. 应用检查该 URL 对应的播客是否已订阅
3. 若未订阅，开始下载 Feed 内容
4. 下载完成后解析 Feed（提取标题、封面、描述、单集列表）
5. 显示 Feed 预览页面：播客名称、封面、描述、单集列表
6. 用户查看预览信息后点击"订阅"按钮
7. 将 Feed 数据保存到本地数据库
8. 导航到该播客的详情页
9. 新订阅出现在订阅列表中

### 场景 4.2：Feed URL 为网页而非直接 Feed（Happy Path）
**触发**: 用户输入的 URL 是普通网页而非 RSS Feed 地址
**执行步骤**:
1. 应用下载 URL 内容
2. 检测到内容不是有效的 RSS/Atom Feed
3. 尝试在页面内容中搜索 Feed 链接
4. 若找到 Feed 链接 → 使用找到的 Feed URL 重新下载和解析
5. 若找到多个 Feed 链接 → 弹出 Feed 选择对话框，用户选择后继续
6. 选择后继续解析和预览流程

### 场景 4.3：通过搜索反查 Feed URL（Happy Path）
**触发**: 输入的 URL 无法直接找到 Feed，但可通过搜索匹配
**执行步骤**:
1. 直接 URL 查找失败
2. 应用尝试通过搜索平台按网页标题/曲名搜索对应播客
3. 找到匹配的播客及其 Feed URL
4. 使用找到的 Feed URL 继续下载和解析流程

### 场景 4.4：多 Feed 发现与选择（Happy Path）
**触发**: 下载的页面包含多个 Feed 链接
**执行步骤**:
1. 应用解析页面发现多个 Feed 链接
2. 弹出 Feed 发现对话框，列出所有找到的 Feed
3. 每个 Feed 显示标题和 URL
4. 用户选择其中一个
5. 使用选中的 Feed URL 继续解析和预览流程

### 场景 4.5：订阅前预览确认（Happy Path）
**触发**: Feed 解析成功后用户查看预览
**执行步骤**:
1. 预览页面显示播客标题
2. 预览页面显示播客封面
3. 预览页面显示播客描述
4. 预览页面显示单集列表（标题、日期等）
5. 预览页面提供"订阅"按钮
6. 用户确认后才正式订阅（不自动订阅）

### 场景 4.6：已订阅的 Feed 重复订阅（边界场景）
**触发**: 用户输入一个已订阅播客的 Feed URL
**执行步骤**:
1. 应用检查到该 Feed 已在订阅列表中
2. 提示用户该播客已订阅
3. 不执行重复订阅操作
4. 可选择直接跳转到该播客详情页

### 场景 4.7：无效 URL 格式（异常场景）
**触发**: 用户输入格式无效的 URL
**执行步骤**:
1. URL 格式校验失败
2. 显示"地址无效"错误提示
3. 不发起网络请求
4. 用户可修正 URL 后重试

### 场景 4.8：网络请求失败（异常场景）
**触发**: RSS Feed URL 有效但网络不可达
**执行步骤**:
1. 应用尝试下载 Feed 内容
2. 网络请求超时或连接失败
3. 显示错误对话框，提示网络错误
4. 提供重试入口和返回入口
5. 应用不崩溃

### 场景 4.9：Feed 解析失败（异常场景）
**触发**: Feed 内容下载成功但无法解析
**执行步骤**:
1. 下载的内容不是有效的 RSS/Atom 格式
2. 解析器报错
3. 显示错误对话框，提示解析失败
4. 提供"编辑 URL"入口，允许用户修正 URL 后重试
5. 提供返回入口

### 场景 4.10：未找到播客（异常场景）
**触发**: URL 处理后未找到任何可用的播客 Feed
**执行步骤**:
1. 所有查找方式（直接下载、搜索反查、Feed 发现）均失败
2. 显示"未找到播客"错误提示
3. 提供"编辑 URL"入口
4. 提供返回入口

### 场景 4.11：需要认证的 Feed（边界场景）
**触发**: 用户输入的 Feed 需要用户名/密码认证
**执行步骤**:
1. 应用下载时收到认证要求（如 HTTP 401）
2. 提示用户输入用户名和密码
3. 用户输入认证信息后重新发起请求
4. 认证成功 → 继续解析和预览流程
5. 认证失败 → 显示认证错误提示

### 场景 4.12：URL 编辑与重试（Happy Path）
**触发**: 用户在错误对话框中点击"编辑 URL"
**执行步骤**:
1. 显示 URL 编辑界面，预填之前的 URL
2. 用户修改 URL
3. 确认后使用新 URL 重新执行下载和解析流程

## 验收要点
- 能从有效 URL 解析播客信息（标题、封面、描述、单集列表）
- 订阅前可预览 Feed 信息，确认后才订阅
- 订阅成功后进入播客详情页，新订阅出现在列表中
- 网页 URL 能自动查找 Feed 链接
- 多 Feed 页面可选择具体 Feed
- 无效 URL、网络失败、解析失败有错误提示和重试入口
- 已订阅播客不重复订阅
- 支持需要认证的 Feed
