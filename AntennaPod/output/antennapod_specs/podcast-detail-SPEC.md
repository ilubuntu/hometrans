# SPEC: REQ-006 播客详情页

## 需求概述

播客详情页应展示播客标题、封面、描述、订阅状态、设置入口、筛选/排序入口和单集列表。页面采用折叠式头部设计，展开时显示播客详情信息，滚动折叠后在标题栏显示播客名称。单集列表支持分页加载和左右滑动操作。从详情页返回后仍在订阅列表中。

## 原子场景

### 场景 6.1：查看播客详情页（Happy Path）
**触发**: 用户从订阅列表或导航点击进入某播客详情页
**执行步骤**:
1. 页面加载，显示进度指示器
2. 异步加载播客信息和单集列表
3. 头部区域展示：播客封面、标题、描述、订阅状态
4. 单集列表加载第一页数据（每页 N 集）
5. 隐藏进度指示器，显示内容

### 场景 6.2：折叠式头部交互（Happy Path）
**触发**: 用户在详情页向上/向下滚动
**执行步骤**:
1. 向上滚动时，头部区域逐渐折叠
2. 头部完全折叠后，标题栏显示播客标题
3. 向下滚动时，头部区域逐渐展开
4. 头部展开后，标题栏隐藏标题，显示完整的封面/描述区域

### 场景 6.3：单集列表浏览与分页加载（Happy Path）
**触发**: 用户浏览单集列表
**执行步骤**:
1. 单集列表展示已加载的单集项
2. 每项显示标题、日期、时长、状态等（详见单集列表项需求）
3. 用户滚动到列表底部
4. 自动触发加载下一页单集
5. 新单集追加到列表底部
6. 无更多单集时不触发加载

### 场景 6.4：加载更多单集（Happy Path）
**触发**: 用户点击底部"加载更多"或已加载所有本地单集后需要拉取更多
**执行步骤**:
1. 列表底部显示"加载更多"入口
2. 点击后触发 Feed 更新，从源拉取更多历史单集
3. 拉取完成后新单集追加到列表
4. 无更多单集时隐藏加载入口

### 场景 6.5：单集列表项点击进入详情（Happy Path）
**触发**: 用户点击某个单集
**执行步骤**:
1. 用户点击单集项
2. 导航到单集详情页
3. 返回后仍在当前播客详情页的单集列表

### 场景 6.6：排序功能（Happy Path）
**触发**: 用户希望调整单集列表排序
**执行步骤**:
1. 用户点击工具栏排序入口
2. 弹出排序选项（日期、标题、时长、持续时间等）
3. 选择后列表按新排序方式重新排列
4. 排序偏好保存

### 场景 6.7：筛选功能（Happy Path）
**触发**: 用户希望筛选单集列表
**执行步骤**:
1. 用户点击工具栏筛选入口
2. 弹出筛选选项（已播放、未播放、已下载、未下载、在队列、收藏等）
3. 可组合多个筛选条件
4. 确认后列表只显示符合条件的单集

### 场景 6.8：搜索当前播客单集（Happy Path）
**触发**: 用户希望在当前播客内搜索单集
**执行步骤**:
1. 用户点击工具栏搜索入口
2. 进入搜索界面（限定在当前播客范围）
3. 输入关键词
4. 搜索结果只包含当前播客的单集
5. 点击结果可进入单集详情

### 场景 6.9：左右滑动操作（Happy Path）
**触发**: 用户在单集项上左右滑动
**执行步骤**:
1. 向左/右滑动单集项
2. 触发预设的滑动操作（如加入队列、标记已播放等）
3. 操作可配置（用户可在设置中自定义滑动绑定的操作）
4. 操作执行后单集状态更新

### 场景 6.10：进入播客设置页（Happy Path）
**触发**: 用户从详情页进入播客设置
**执行步骤**:
1. 用户点击工具栏或头部的设置入口
2. 进入播客独立设置页面
3. 可配置新单集处理方式、自动下载、播放速度等
4. 修改后保存，返回详情页

### 场景 6.11：查看播客详细信息（Happy Path）
**触发**: 用户从详情头部进入播客信息页
**执行步骤**:
1. 用户点击头部信息入口
2. 进入播客信息页
3. 展示：标题、作者、语言、完整描述、源地址 URL
4. 提供分享播客和访问网站入口
5. 返回后回到详情页

### 场景 6.12：标记全部已播放（Happy Path）
**触发**: 用户希望将当前播客所有单集标记为已播放
**执行步骤**:
1. 用户点击工具栏"标记全部已播放"选项
2. 确认操作
3. 所有单集状态变为已播放
4. 列表视觉更新（已播放样式）

### 场景 6.13：多选批量操作（Happy Path）
**触发**: 用户长按某个单集进入多选模式
**执行步骤**:
1. 进入多选模式
2. 底部显示批量操作菜单（下载、加入队列、标记已播放等）
3. 可选择多个单集
4. 执行批量操作后退出多选模式

### 场景 6.14：下拉刷新（Happy Path）
**触发**: 用户在详情页下拉
**执行步骤**:
1. 触发下拉刷新
2. 后台检查该播客的更新
3. 有新单集则更新列表
4. 刷新完成后动画停止

### 场景 6.15：实时状态更新（Happy Path）
**触发**: 单集的下载/播放/队列状态发生变化
**执行步骤**:
1. 应用收到状态变化通知
2. 详情页单集列表对应项实时更新
3. 下载进度、播放状态、队列标记等同步刷新

### 场景 6.16：无单集的播客详情页（边界场景）
**触发**: 用户进入一个暂无单集的播客详情页
**执行步骤**:
1. 详情头部正常显示播客信息
2. 单集列表区域显示空状态
3. 提示"暂无单集"或显示加载入口

## 验收要点
- 详情信息完整（标题、封面、描述、订阅状态）
- 折叠式头部交互正常（展开显示详情，折叠显示标题）
- 单集列表可滚动，支持分页加载
- 支持排序、筛选、搜索功能
- 支持左右滑动操作
- 设置入口可达
- 返回后仍在订阅列表
- 单集状态变化实时更新
