# REQ-016 队列页面 - 需求规格

## 概述
队列页面展示用户加入播放队列的所有剧集，以列表形式呈现。页面提供剩余集数和总时长的汇总信息，支持排序、清空和锁定操作，以及实时反映队列变更事件。

## 原子场景

### 场景 1：查看队列列表（正常流程）
**触发**：用户导航到队列页面

**执行步骤**：
1. 异步从数据库加载队列数据
2. 显示加载进度指示器
3. 加载完成后以列表形式展示队列中的所有剧集
4. 每个剧集项显示：封面、标题、播客名、播放状态、下载状态等
5. 信息栏显示队列汇总："N 集 • 剩余 Xh Ym"
6. 剩余时长按各剧集未播放部分累加（可选择按播放速度计算）
7. 恢复上次的滚动位置

### 场景 2：队列为空时的显示
**触发**：队列中没有剧集

**执行步骤**：
1. 显示空状态视图（图标 + 标题 + 提示消息）
2. 若收件箱中有新剧集：显示"前往收件箱"按钮
3. 点击按钮跳转到收件箱页面

### 场景 3：排序队列
**触发**：用户点击工具栏的"排序"按钮

**执行步骤**：
1. 弹出排序选择对话框
2. 显示可选排序方式：日期（新→旧/旧→新）、标题（A→Z/Z→A）、时长（长→短/短→长）、随机、智能随机
3. 显示"保持排序"复选框
4. 用户选择排序方式：
   a. 若选择"随机"，自动取消"保持排序"
   b. 其他选择时恢复"保持排序"状态
5. 立即对队列执行排序并保存
6. 列表刷新显示排序后的顺序

### 场景 4：清空队列
**触发**：用户点击工具栏的"清空队列"按钮

**执行步骤**：
1. 弹出确认对话框："确定要清空队列吗？"
2. 用户点击确认：
   a. 清空数据库中的队列
   b. 发送队列清空事件
   c. 列表清空，显示空状态
3. 用户点击取消：不做任何操作

### 场景 5：锁定/解锁队列
**触发**：用户点击工具栏的"锁定"按钮

**执行步骤**：
1. 若队列当前未锁定且首次使用：
   a. 弹出锁定警告对话框（解释锁定后无法手动排序）
   b. 提供"不再显示"复选框
   c. 用户确认后锁定队列
2. 若队列当前已锁定：直接解锁
3. 锁定后：禁用拖拽排序和滑动操作
4. 解锁后：恢复拖拽排序和滑动操作
5. 若开启了"保持排序"：隐藏锁定按钮

### 场景 6：实时更新队列列表
**触发**：其他页面修改了队列（加入/移除/排序/清空）

**执行步骤**：
1. 队列页面通过事件总线接收队列变更事件
2. 根据事件类型更新内存中的队列列表
3. 通知适配器刷新对应的列表项（增/删/移/整体刷新）
4. 更新信息栏的集数和时长汇总
5. 更新工具栏状态（锁定按钮等）

### 场景 7：更新当前播放项进度
**触发**：播放进度更新事件

**执行步骤**：
1. 在队列列表中找到当前正在播放的剧集项
2. 更新该项的进度显示（进度条/位置文字）
3. 不影响其他剧集项的显示

### 场景 8：下拉刷新
**触发**：用户在队列页面下拉

**执行步骤**：
1. 触发播客源更新（检查是否有新剧集）
2. 显示刷新动画
3. 更新完成后隐藏刷新动画
