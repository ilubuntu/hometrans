# SPEC: REQ-003 添加播客入口

## 需求概述

用户可以从首页或主导航进入"添加播客"页面。该页面应提供完整的播客添加入口，包括：播客搜索（支持多个搜索源）、RSS Feed URL 手动输入、OPML 文件导入、以及本地文件夹添加。各入口布局清晰、可见可操作。

## 原子场景

### 场景 3.1：进入添加播客页面（Happy Path）
**触发**: 用户通过主导航或首页入口进入添加播客页面
**执行步骤**:
1. 用户点击主导航中的"添加播客"入口，或空库状态的引导入口，或订阅列表的浮动添加按钮
2. 加载添加播客页面
3. 页面显示以下入口区域：
   - 搜索输入框（统一搜索）
   - 多个搜索平台快捷按钮
   - RSS URL 输入按钮
   - OPML 导入按钮
   - 本地文件夹添加按钮

### 场景 3.2：统一搜索 - 关键词搜索（Happy Path）
**触发**: 用户在搜索输入框输入关键词并触发搜索
**执行步骤**:
1. 用户在搜索输入框输入播客名称或关键词
2. 点击搜索按钮或按下搜索键
3. 判断输入是否为 URL 格式：
   - 若为 URL（以 http:// 或 https:// 开头）→ 直接进入 RSS 订阅流程
   - 若非 URL → 进入联合搜索结果页面
4. 联合搜索聚合多个搜索源的结果
5. 隐藏键盘，清除搜索框焦点

### 场景 3.3：搜索平台快捷入口（Happy Path）
**触发**: 用户点击某个搜索平台按钮
**执行步骤**:
1. 页面提供多个搜索平台入口（如 iTunes、Fyyd、Podcast Index）
2. 用户点击某个平台按钮
3. 进入该平台的搜索页面
4. 用户可输入关键词在该平台搜索播客

### 场景 3.4：RSS URL 输入 - 正常流程（Happy Path）
**触发**: 用户点击"按 URL 添加"按钮
**执行步骤**:
1. 弹出 URL 输入对话框
2. 对话框提示"RSS 地址"
3. 检查系统剪贴板：
   - 若剪贴板内容以 "http" 开头 → 自动预填到输入框
4. 用户输入或确认 URL
5. 点击确认按钮
6. 校验 URL 格式：
   - 格式合法 → 关闭对话框，进入 RSS Feed 预览订阅流程
   - 格式非法 → 输入框显示错误提示，不关闭对话框

### 场景 3.5：RSS URL 输入 - 格式校验失败（异常场景）
**触发**: 用户输入无效 URL 并确认
**执行步骤**:
1. 用户输入非 URL 格式的文本（如纯文字）
2. 点击确认
3. 输入框下方显示"地址无效"错误提示
4. 对话框保持打开，用户可修正后重新提交
5. 点击取消 → 关闭对话框，不执行任何操作

### 场景 3.6：OPML 文件导入入口（Happy Path）
**触发**: 用户点击 OPML 导入按钮
**执行步骤**:
1. 点击 OPML 导入按钮
2. 调用系统文件选择器
3. 用户选择 OPML 文件
4. 进入 OPML 导入流程页面

### 场景 3.7：OPML 导入 - 文件选择器不可用（异常场景）
**触发**: 系统没有可用的文件选择器
**执行步骤**:
1. 点击 OPML 导入按钮
2. 系统无法启动文件选择器
3. 显示错误提示消息（"无法启动系统文件管理器"）
4. 页面不崩溃

### 场景 3.8：本地文件夹添加入口（Happy Path）
**触发**: 用户点击添加本地文件夹按钮
**执行步骤**:
1. 点击本地文件夹按钮
2. 调用系统文件夹选择器
3. 用户选择一个文件夹
4. 系统获取该文件夹的持久访问权限
5. 将文件夹创建为"本地播客"并保存到数据库
6. 触发该本地播客的媒体文件扫描
7. 成功后导航到该播客的详情页

### 场景 3.9：本地文件夹添加失败（异常场景）
**触发**: 本地文件夹添加过程中出错
**执行步骤**:
1. 文件夹选择或访问失败
2. 显示错误提示消息
3. 页面不崩溃，用户可重试

### 场景 3.10：从订阅列表浮动按钮进入（Happy Path）
**触发**: 用户在订阅列表页面点击浮动添加按钮
**执行步骤**:
1. 用户在订阅列表页面看到浮动添加按钮
2. 点击按钮
3. 进入添加播客页面
4. 页面布局与从导航进入时一致

## 验收要点
- 添加播客页面布局完整，所有入口可见
- 搜索框可输入并触发搜索（关键词或 URL 自动识别）
- 多个搜索平台快捷按钮可点击
- RSS URL 输入支持剪贴板预填和格式校验
- OPML 导入入口可触发系统文件选择器
- 本地文件夹添加功能正常
- 文件选择器不可用时显示友好错误提示，不崩溃
