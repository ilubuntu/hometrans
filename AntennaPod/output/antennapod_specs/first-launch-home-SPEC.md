# SPEC: REQ-001 首次启动与空库首页

## 需求概述

应用首次启动（或数据库中无任何单集数据）时，应正常进入主界面并展示空库状态。空库状态下页面需清楚展示：应用名称/标识、主导航入口、添加播客入口，以及当前没有订阅/没有内容的空状态提示。用户能通过空状态页面找到添加播客或导入订阅的入口。

## 原子场景

### 场景 1.1：首次安装后启动应用（Happy Path）
**触发**: 用户首次安装并打开应用
**执行步骤**:
1. 系统初始化应用，加载主界面框架
2. 应用查询本地数据库中的单集总数
3. 查询结果为 0（无任何单集数据）
4. 主界面显示空状态欢迎视图（包含应用标识、空状态提示文案）
5. 空状态视图显示添加播客入口指引（箭头指向主导航中的添加入口）
6. 首页内容分区列表被隐藏
7. 用户可看到主导航结构（底部导航栏或侧边导航面板，取决于用户设置）

### 场景 1.2：空库状态下添加播客入口可见性验证
**触发**: 首次启动后用户查看空状态页面
**执行步骤**:
1. 空状态页面展示应用名称或标识
2. 空状态页面包含"添加播客"的引导提示文本
3. 引导提示包含方向指引，指向主导航中的添加入口：
   - 若使用底部导航模式，箭头指向底部导航栏的添加按钮
   - 若使用侧边导航模式，箭头指向侧边导航面板的添加入口
4. 用户点击该入口后可进入添加播客页面

### 场景 1.3：非首次启动但无单集数据（边界场景）
**触发**: 用户曾安装应用、清除了数据或从未成功订阅过播客，重新启动应用
**执行步骤**:
1. 应用启动，加载主界面
2. 查询单集总数，结果为 0
3. 显示空状态欢迎视图（与首次启动行为一致）
4. 首页内容分区被隐藏
5. 主导航正常可用

### 场景 1.4：订阅播客后首页从空状态切换（状态转换）
**触发**: 用户从空状态完成首次订阅播客操作
**执行步骤**:
1. 用户通过添加入口订阅了一个播客
2. 播客订阅数据写入本地数据库
3. 单集列表同步到本地后，单集总数 > 0
4. 首页收到数据更新通知
5. 空状态欢迎视图隐藏
6. 首页内容分区列表显示（队列、收件箱、订阅、下载等分区）
7. 下拉刷新功能恢复可用

### 场景 1.5：空状态页面滚动行为（边界场景）
**触发**: 用户在空状态页面尝试滚动
**执行步骤**:
1. 空状态页面的滚动位置固定在顶部（不允许向下滚动）
2. 页面不崩溃，不产生异常

### 场景 1.6：空状态下的导航模式适配（边界场景）
**触发**: 用户在设置中切换导航模式（底部导航 ↔ 侧边导航）后回到首页
**执行步骤**:
1. 应用检测到导航模式变更
2. 空状态页面的入口指引箭头方向更新：
   - 底部导航模式 → 箭头指向底部
   - 侧边导航模式 → 箭头指向侧边
3. 主导航结构按新模式重新渲染

### 场景 1.7：应用启动不崩溃（健壮性验证）
**触发**: 应用启动过程中数据库初始化等耗时操作
**执行步骤**:
1. 应用启动，初始化本地数据库
2. 查询单集总数的操作在后台异步执行
3. 查询期间页面显示默认状态（不崩溃、不卡死）
4. 查询完成后正确切换到空状态或内容状态
5. 若查询出错，记录错误日志，不影响应用基本功能

## 验收要点
- 应用可正常启动，空库状态下不崩溃
- 空库状态页面清晰展示应用标识和空状态提示
- 添加播客入口可见且可达
- 入口指引方向与当前导航模式一致
- 订阅播客后首页自动从空状态切换到内容列表
