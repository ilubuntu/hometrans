# REQ-009 播放单集 - 需求规格

## 概述
用户可以从剧集列表或详情页播放单个播客剧集。播放启动后，音频开始播放，界面显示迷你播放器和完整播放页，包含标题、播放/暂停按钮、进度信息和跳过功能。

## 原子场景

### 场景 1：从剧集列表启动播放（正常流程）
**触发**：用户在剧集列表中点击某个剧集的播放按钮

**执行步骤**：
1. 获取被点击剧集的媒体信息（标题、时长、播放位置等）
2. 通过播放启动器发起播放请求，传入媒体对象
3. 即使播放服务已在运行也强制启动播放该媒体
4. 播放服务加载媒体、准备播放器、开始播放
5. 界面显示迷你播放器（包含标题、封面、播放/暂停按钮、进度条）
6. 播放按钮图标切换为暂停状态
7. 进度条开始随播放进度更新

### 场景 2：从迷你播放器展开播放详情页
**触发**：用户点击迷你播放器区域

**执行步骤**：
1. 检测当前播放的媒体类型
2. 若为音频类型：将底部面板从折叠状态展开为完整播放页
3. 播放详情页显示：封面页（包含封面图、标题、播客名）和描述页
4. 底部控制栏显示：后退按钮（带秒数）、播放/暂停按钮、快进按钮（带秒数）、跳过按钮
5. 进度条显示当前位置、总时长（或剩余时间）
6. 若已在播放详情页，点击导航按钮可折叠回迷你播放器

### 场景 3：恢复上次播放（无指定媒体时）
**触发**：用户点击播放按钮且无特定媒体上下文

**执行步骤**：
1. 检测播放服务是否运行
2. 若服务未运行或无可播放媒体，从偏好设置中读取上次播放的媒体 ID
3. 加载该媒体信息
4. 启动播放服务并从上次停止位置继续播放
5. 更新 UI 显示当前播放状态和进度

### 场景 4：点击跳过按钮播放下一集
**触发**：用户在播放页点击跳过按钮

**执行步骤**：
1. 调用播放服务跳到播放队列中的下一个媒体项
2. 若队列中无下一项，播放结束，播放服务关闭
3. 更新播放页显示新剧集信息

### 场景 5：视频类型媒体的播放
**触发**：用户播放的视频类型媒体

**执行步骤**：
1. 迷你播放器检测到媒体类型为视频
2. 隐藏迷你播放器中的播放按钮（视频在独立页面播放）
3. 底部面板锁定为折叠状态
4. 点击迷你播放器区域跳转到视频播放页面
