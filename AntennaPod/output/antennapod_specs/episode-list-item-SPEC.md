# SPEC: REQ-007 单集列表项

## 需求概述

单集列表项应展示丰富的单集信息，包括：标题、发布日期/时长、播放状态（播放进度/正在播放/已播放）、下载状态（未下载/下载中/已下载）、是否在队列、是否为新、是否收藏等状态标记。每个列表项还提供上下文感知的快捷操作按钮和更多操作菜单。该列表项在多个页面（播客详情、全部单集、收件箱、收藏、队列等）中统一复用。

## 原子场景

### 场景 7.1：列表项基本信息展示（Happy Path）
**触发**: 单集列表加载并渲染某个单集项
**执行步骤**:
1. 列表项显示单集标题
2. 列表项显示发布日期（缩写格式）
3. 列表项显示封面图片（回退到播客封面）
4. 若单集有媒体且有时长 → 显示时长
5. 若无媒体 → 隐藏时长，显示其他状态

### 场景 7.2：新单集状态标记（Happy Path）
**触发**: 某单集为新的/未读状态
**执行步骤**:
1. 列表项显示"新单集"标记图标
2. 该标记可见
3. 用户查看或操作后（如标记已播放）该标记消失

### 场景 7.3：已播放状态视觉区分（Happy Path）
**触发**: 某单集已被标记为已播放
**执行步骤**:
1. 列表项整体透明度降低（如 50%）
2. 不显示"新单集"标记
3. 与未播放单集在视觉上有明显区分

### 场景 7.4：播放进度展示（Happy Path）
**触发**: 某单集正在播放或播放中途中断
**执行步骤**:
1. 若正在播放 → 列表项高亮（激活态）
2. 显示播放进度条（显示已播放百分比）
3. 显示当前播放位置文本
4. 显示总时长

### 场景 7.5：下载状态 - 未下载（Happy Path）
**触发**: 某单集未下载且不在下载中
**执行步骤**:
1. 操作按钮显示为"下载"图标
2. 下载进度环形指示器显示 0%
3. 用户点击按钮可开始下载

### 场景 7.6：下载状态 - 下载中（Happy Path）
**触发**: 某单集正在下载
**执行步骤**:
1. 操作按钮显示为"取消下载"图标
2. 下载进度环形指示器显示当前进度百分比
3. 下载排队中时显示不确定进度动画
4. 进度实时更新
5. 用户点击按钮可取消下载

### 场景 7.7：下载状态 - 已下载（Happy Path）
**触发**: 某单集已下载完成
**执行步骤**:
1. 下载进度环形指示器显示 100%（满圆）
2. 操作按钮根据播放状态切换为"播放"
3. 用户可离线播放

### 场景 7.8：队列状态标记（Happy Path）
**触发**: 某单集已在播放队列中
**执行步骤**:
1. 列表项显示"在队列中"标记图标
2. 该图标可见
3. 从队列移除后该标记消失

### 场景 7.9：收藏状态标记（Happy Path）
**触发**: 某单集已被收藏
**执行步骤**:
1. 列表项显示"已收藏"标记图标
2. 该图标可见
3. 取消收藏后该标记消失

### 场景 7.10：视频类型标记（Happy Path）
**触发**: 某单集为视频媒体
**执行步骤**:
1. 列表项显示"视频"类型标记图标
2. 与音频单集视觉区分

### 场景 7.11：上下文感知操作按钮（Happy Path）
**触发**: 不同状态的单集展示操作按钮
**执行步骤**:
1. 正在播放的单集 → 操作按钮为"暂停"
2. 已下载的单集 → 操作按钮为"播放"
3. 未下载的单集 → 操作按钮为"下载"
4. 正在下载的单集 → 操作按钮为"取消下载"
5. 本地 Feed 的单集 → 操作按钮为"本地播放"
6. 按钮图标随状态实时切换

### 场景 7.12：封面点击导航（Happy Path）
**触发**: 用户点击列表项封面图片
**执行步骤**:
1. 点击封面
2. 导航到该单集所属的播客详情页
3. 返回后回到原列表

### 场景 7.13：更多操作菜单（Happy Path）
**触发**: 用户点击列表项的更多操作按钮或长按
**执行步骤**:
1. 弹出操作菜单/底部弹窗
2. 菜单选项包括：
   - 播放/流式播放
   - 下载/删除下载
   - 加入队列/移出队列
   - 收藏/取消收藏
   - 分享
   - 标记已播放/标记未播放
   - 访问网站
3. 选择操作后执行并关闭菜单
4. 列表项状态相应更新

### 场景 7.14：无媒体单集的列表项（边界场景）
**触发**: 某单集没有关联的媒体文件
**执行步骤**:
1. 隐藏时长、进度条、播放状态相关 UI
2. 操作按钮切换为非媒体操作（如"标记已播放"、"访问网站"）
3. 隐藏视频类型标记
4. 列表项不崩溃

### 场景 7.15：状态实时同步（Happy Path）
**触发**: 单集状态在其他页面或后台发生变化
**执行步骤**:
1. 应用收到状态变化通知（下载事件、播放事件、单集事件等）
2. 列表项对应项实时更新
3. 操作按钮、状态标记、进度条同步刷新
4. 不需要手动刷新页面

### 场景 7.16：列表项无障碍支持（边界场景）
**触发**: 辅助功能（如屏幕阅读器）读取列表项
**执行步骤**:
1. 列表项有内容描述
2. 已播放单集的内容描述包含"已播放"信息
3. 发布日期有完整格式的无障碍描述
4. 时长有本地化的无障碍描述

## 验收要点
- 列表项信息清晰（标题、日期、时长、封面）
- 播放状态、下载状态、队列状态、收藏状态、新单集状态有明确视觉标记
- 已播放单集透明度降低，与新单集视觉区分
- 操作按钮上下文感知（播放/下载/暂停/取消随状态切换）
- 下载进度实时显示
- 更多操作菜单可打开且功能完整
- 封面可点击进入播客详情
- 状态变化实时同步，无需手动刷新
- 无媒体单集正确处理
