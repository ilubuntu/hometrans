# SPEC: REQ-037 统计页面

## 需求概述

应用提供收听统计页面，以三个标签页展示数据：按订阅统计（每个播客的播放时长排行）、按年份统计（每年的播放时长趋势）、下载空间统计（每个播客占用的存储空间）。支持时间范围过滤、包含/排除手动标记已播的数据、重置统计数据。特定时期可显示年度回顾入口。

## 原子场景

### 场景 37.1：查看按订阅统计（Happy Path）
**触发**: 用户打开统计页面
**执行步骤**:
1. 用户从导航入口打开统计页面
2. 默认显示"订阅"标签页
3. 顶部显示总播放时长摘要
4. 列表按播放时长降序排列每个播客
5. 每条目显示：播客封面、标题、播放时长
6. 播放时长以条形图可视化

### 场景 37.2：查看按年份统计（Happy Path）
**触发**: 用户切换到年份统计标签页
**执行步骤**:
1. 用户点击"年份"标签页
2. 显示每年的总播放时长
3. 柱状图展示年度播放趋势
4. 可直观比较不同年份的收听量

### 场景 37.3：查看下载空间统计（Happy Path）
**触发**: 用户切换到下载空间标签页
**执行步骤**:
1. 用户点击"下载"标签页
2. 显示每个播客已下载文件占用的存储空间
3. 按空间占用降序排列
4. 显示总下载空间

### 场景 37.4：设置统计过滤条件（Happy Path）
**触发**: 用户调整统计的时间范围或数据包含方式
**执行步骤**:
1. 用户打开统计页面的设置菜单
2. 可选择是否"包含手动标记已播"的数据
3. 可设置统计的起始时间
4. 可设置统计的结束时间
5. 设置变更后统计数据自动刷新

### 场景 37.5：重置统计数据（边界场景）
**触发**: 用户重置所有统计数据
**执行步骤**:
1. 用户点击菜单中的"重置统计"
2. 弹出确认对话框，提示将清除所有统计
3. 用户确认后清除统计偏好设置
4. 清除数据库中的播放统计记录
5. 页面刷新，统计数据归零

### 场景 37.6：取消重置统计（边界场景）
**触发**: 用户在确认对话框中取消
**执行步骤**:
1. 弹出重置确认对话框
2. 用户点击"取消"
3. 不执行任何操作
4. 统计数据保持不变

### 场景 37.7：无播放数据时查看统计（边界场景）
**触发**: 新用户或重置后查看统计
**执行步骤**:
1. 统计页面正常打开
2. 各标签页显示空状态
3. 总播放时长为 0
4. 列表为空或显示提示

### 场景 37.8：年度回顾入口（边界场景）
**触发**: 特定时期或调试模式下显示年度回顾
**执行步骤**:
1. 满足显示条件时，统计页面菜单出现"年度回顾"选项
2. 用户点击后进入年度回顾页面
3. 展示年度精彩收听回顾

### 场景 37.9：切换标签页保持状态（Happy Path）
**触发**: 用户在三个标签页间切换
**执行步骤**:
1. 用户从订阅标签页切换到年份标签页
2. 再切换到下载标签页
3. 各标签页数据保持已加载状态
4. 工具栏根据当前标签页更新

## 验收要点
- 提供三个统计标签页：按订阅、按年份、按下载空间
- 按订阅统计显示各播客播放时长排行（降序）
- 按年份统计显示年度播放趋势（柱状图）
- 按下载空间统计显示各播客存储占用
- 支持时间范围过滤和手动标记已播的包含/排除
- 支持重置统计数据（带确认）
- 无数据时显示空状态
- 统计数据可刷新
