# SPEC: REQ-029 播客设置

## 需求概述

用户可以对每个已订阅的播客进行独立设置，包括：新单集到达时的处理方式（加入收件箱/加入队列/不做任何操作）、是否自动下载新单集、自动删除策略、播放速度、跳过片头片尾秒数、静音跳过、音量适配、剧集过滤规则、标签分类、是否保持自动更新、新单集通知、用户名密码认证、播客重命名、修改来源 URL、重新连接本地文件夹。设置修改后实时生效并持久化存储。

## 原子场景

### 场景 29.1：查看播客设置页面（Happy Path）
**触发**: 用户从播客详情页进入播客设置
**执行步骤**:
1. 用户在播客详情页打开设置入口
2. 应用异步加载该播客的偏好数据
3. 加载期间设置页面不显示部分数据（避免闪烁）
4. 数据加载完成后展示所有设置项及其当前值
5. 每项设置显示当前状态摘要（如"跟随全局"、"已启用"等）

### 场景 29.2：修改新单集处理方式（Happy Path）
**触发**: 用户更改某播客的新单集处理策略
**执行步骤**:
1. 用户点击"新单集处理"设置项
2. 弹出选项列表：跟随全局、加入收件箱、加入队列、不做任何操作
3. 用户选择一个选项
4. 应用保存设置到本地
5. 更新设置项摘要显示当前策略
6. 后续该播客新单集按新策略处理

### 场景 29.3：已开启自动下载时的单集处理限制（边界场景）
**触发**: 用户尝试修改已开启自动下载的播客的单集处理方式
**执行步骤**:
1. 检测到该播客已开启自动下载
2. 显示提示信息，告知用户当前不可修改
3. 单集处理选项被锁定（或禁用）
4. 用户需先关闭自动下载才能修改

### 场景 29.4：设置自动下载开关（Happy Path）
**触发**: 用户开启或关闭某播客的自动下载
**执行步骤**:
1. 用户点击"自动下载"设置项
2. 弹出选项：跟随全局、启用、禁用
3. 用户选择后保存
4. 若启用，显示"剧集过滤"设置项
5. 更新摘要显示实际启用状态（结合全局设置计算）

### 场景 29.5：设置单播客播放速度（Happy Path）
**触发**: 用户为某播客单独设置播放速度
**执行步骤**:
1. 用户点击"播放速度"设置项
2. 弹出速度选择对话框，包含速度滑块（0.25x~4x）
3. 显示"使用全局速度"复选框
4. 若勾选全局，滑块和静音跳过选项变灰不可用
5. 用户调整速度后确认
6. 保存设置并广播速度变更事件，播放器实时应用

### 场景 29.6：设置跳过片头片尾（Happy Path）
**触发**: 用户设置某播客播放时跳过片头和/或片尾的秒数
**执行步骤**:
1. 用户点击"跳过片头片尾"设置项
2. 弹出对话框，分别输入跳过片头秒数和片尾秒数
3. 用户输入数值后确认
4. 保存设置并广播变更事件
5. 该播客后续播放时自动跳过指定秒数

### 场景 29.7：设置标签（Happy Path）
**触发**: 用户为播客设置标签
**执行步骤**:
1. 用户点击"标签"设置项
2. 弹出标签编辑对话框
3. 显示当前已设置的标签（芯片形式）
4. 用户可输入新标签名，从已有标签自动补全
5. 可删除现有标签
6. 确认后保存到播客偏好

### 场景 29.8：设置认证信息（Happy Path）
**触发**: 用户为需要认证的播客设置用户名密码
**执行步骤**:
1. 用户点击"认证"设置项
2. 弹出对话框，输入用户名和密码
3. 用户填写后确认
4. 保存认证信息到本地
5. 自动触发该播客的 Feed 刷新，使用新的认证信息

### 场景 29.9：修改来源 URL（Happy Path）
**触发**: 用户修改播客的 RSS Feed 来源地址
**执行步骤**:
1. 用户点击"编辑来源 URL"设置项
2. 弹出对话框，预填当前 URL
3. 用户修改 URL 后确认
4. 保存新的下载地址
5. 后续刷新使用新 URL

### 场景 29.10：重命名播客（Happy Path）
**触发**: 用户修改播客的显示名称
**执行步骤**:
1. 用户点击"重命名"设置项
2. 弹出对话框，预填当前名称
3. 用户修改后确认
4. 保存新名称，UI 立即更新

### 场景 29.11：本地播客的特殊设置视图（边界场景）
**触发**: 用户查看本地文件夹类型播客的设置
**执行步骤**:
1. 检测到播客为本地类型
2. 隐藏认证、自动下载、剧集过滤、编辑 URL 等不适用项
3. 显示"重新连接本地文件夹"选项
4. 用户点击后弹出确认警告
5. 确认后选择新的本地文件夹路径
6. 更新播客的本地文件夹路径

### 场景 29.12：设置音量适配（Happy Path）
**触发**: 用户为某播客设置音量增益或减弱
**执行步骤**:
1. 用户点击"音量适配"设置项
2. 弹出选项列表（如：无、轻减、轻增）
3. 用户选择后保存
4. 广播音量变更事件，播放器实时应用

## 验收要点
- 每个播客可独立设置新单集处理方式、自动下载、自动删除
- 支持单播客播放速度、跳过片头片尾、静音跳过、音量适配
- 支持标签分类、认证、剧集过滤、通知、重命名、来源 URL 修改
- 设置修改后立即生效并持久化
- 自动下载开启时单集处理策略受限
- 本地播客隐藏不适用项，提供本地文件夹重连功能
- 各设置项显示当前状态摘要
