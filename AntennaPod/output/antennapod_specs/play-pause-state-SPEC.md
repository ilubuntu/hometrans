# REQ-011 播放暂停状态 - 需求规格

## 概述
播放/暂停切换是播放控制的核心交互。当用户点击播放按钮时，根据当前播放状态执行暂停或恢复播放操作，同时按钮图标通过动画平滑切换，确保界面始终与实际播放状态一致。

## 原子场景

### 场景 1：从暂停状态恢复播放（正常流程）
**触发**：当前处于暂停状态，用户点击播放按钮

**执行步骤**：
1. 检测当前播放状态为暂停
2. 向播放服务发送恢复播放指令
3. 播放按钮图标通过动画从"播放"切换为"暂停"
4. 更新按钮的无障碍描述为"暂停"
5. 所有播放相关界面（迷你播放器、播放详情页）同步更新按钮状态

### 场景 2：从播放状态暂停
**触发**：当前正在播放，用户点击播放按钮

**执行步骤**：
1. 检测当前播放状态为播放中
2. 向播放服务发送暂停指令
3. 播放按钮图标通过动画从"暂停"切换为"播放"
4. 更新按钮的无障碍描述为"播放"
5. 所有播放相关界面同步更新按钮状态

### 场景 3：点击封面暂停/播放
**触发**：用户在播放详情页点击封面图片

**执行步骤**：
1. 检测当前是否正在播放
2. 正在播放 → 发送暂停指令
3. 未播放 → 启动播放服务恢复播放
4. 封面页的其他播放状态指示同步更新

### 场景 4：媒体正在准备时的播放按钮行为
**触发**：媒体处于准备中状态时用户点击播放按钮

**执行步骤**：
1. 检测到状态为准备中（PREPARING）
2. 切换"准备完成后是否自动播放"的标志
3. 按钮图标反映新的预期状态（准备后自动播放→显示暂停图标；准备后不播放→显示播放图标）

### 场景 5：服务未运行时的播放按钮
**触发**：播放服务未运行，用户点击播放按钮

**执行步骤**：
1. 检测到播放服务未运行
2. 通过播放启动器启动服务并加载当前媒体
3. 按钮图标切换为暂停图标
4. 服务启动后通过事件通知 UI 更新

### 场景 6：播放状态全局同步
**触发**：播放状态发生任何变化

**执行步骤**：
1. 播放服务发出状态变更事件
2. 迷你播放器接收事件并重新加载媒体信息，更新按钮
3. 播放详情页接收事件并重新加载媒体信息，更新按钮
4. 确保所有可见的播放控制按钮状态一致
