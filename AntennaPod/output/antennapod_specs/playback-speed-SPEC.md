# REQ-013 播放速度 - 需求规格

## 概述
用户可以调整音频播放速度。通过弹出速度选择面板，用户可以使用滑动条选择精确速度，或从预设速度列表中快速选择。速度设置会立即应用到播放并持久化保存。同时还提供跳过静音功能。

## 原子场景

### 场景 1：通过滑动条调整播放速度（正常流程）
**触发**：用户点击播放页速度按钮，在弹出的面板中拖动速度滑动条

**执行步骤**：
1. 弹出底部速度选择面板
2. 面板显示当前播放速度，滑动条定位到当前速度值
3. 用户拖动滑动条，实时回调当前速度值
4. 每次速度变化立即保存到用户偏好
5. 每次速度变化立即应用到播放服务（实时改变播放速率）
6. 播放页的速度文字同步更新为新速度值

### 场景 2：从预设速度中选择
**触发**：用户在速度面板中点击某个预设速度

**执行步骤**：
1. 面板展示用户保存的预设速度网格（3 列布局）
2. 用户点击某个预设速度
3. 将该速度保存为当前播放速度
4. 应用到播放服务
5. 面板自动关闭
6. 播放页速度文字更新

### 场景 3：添加当前速度为预设
**触发**：用户在速度面板中点击"添加当前速度"按钮

**执行步骤**：
1. 获取滑动条当前的速度值
2. 检查该速度是否已存在于预设列表中
3. 若已存在：显示提示"预设已存在"
4. 若不存在：添加到预设列表、排序、保存到偏好
5. 预设网格刷新显示

### 场景 4：删除预设速度
**触发**：用户长按某个预设速度

**执行步骤**：
1. 从预设列表中移除该速度
2. 保存更新后的列表到偏好
3. 预设网格刷新显示

### 场景 5：开启/关闭跳过静音
**触发**：用户在速度面板中切换"跳过静音"开关

**执行步骤**：
1. 记录开关状态
2. 保存到用户偏好
3. 应用到播放服务（开启后播放时自动跳过静音部分）

### 场景 6：速度对时间显示的影响
**触发**：播放速度非 1.0 时查看进度

**执行步骤**：
1. 进度条位置和时长文字按当前播放速度转换显示
2. 例如 2.0x 速度下，30 分钟的媒体显示为 15 分钟
3. 队列页的剩余时间也考虑播放速度（若用户启用了此选项）
