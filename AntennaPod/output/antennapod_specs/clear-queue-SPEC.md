# REQ-018 清空队列 - 需求规格

## 概述
用户可以一键清空整个播放队列。清空操作需要二次确认以防误操作。清空后队列页面显示空状态，所有剧集的队列标记被移除，信息栏更新为零。

## 原子场景

### 场景 1：清空队列（正常流程）
**触发**：用户在队列页面点击"清空队列"按钮并确认

**执行步骤**：
1. 弹出确认对话框，显示确认消息
2. 用户点击确认按钮：
   a. 清空数据库中的所有队列记录
   b. 发送队列清空事件
3. 队列页面接收事件：
   a. 清空内存中的队列列表
   b. 列表适配器更新为空列表
   c. 刷新工具栏状态
   d. 刷新信息栏
4. 显示空状态视图（图标 + 标题 + 提示消息）
5. 信息栏显示："0 集 • 剩余 0m"
6. 拖拽排序状态更新

### 场景 2：取消清空操作
**触发**：用户在确认对话框中点击取消

**执行步骤**：
1. 弹出确认对话框
2. 用户点击取消按钮或点击对话框外部
3. 对话框关闭
4. 队列保持不变

### 场景 3：清空队列后的剧集列表状态
**触发**：队列被清空后查看剧集列表

**执行步骤**：
1. 剧集列表在下次加载/刷新时重新查询队列状态
2. 之前在队列中的剧集不再显示"队列"标记
3. 剧集的"加入队列"操作按钮重新变为可用
4. 滑动操作恢复为"加入队列"（而非"从队列移除"）

### 场景 4：清空队列后当前播放不受影响
**触发**：清空队列时有剧集正在播放

**执行步骤**：
1. 清空队列操作不影响当前正在播放的媒体
2. 当前播放继续进行
3. 迷你播放器和播放详情页正常显示
4. 当前剧集播放完毕后，不会自动播放下一集（队列已空）
5. 播放服务根据空队列状态决定后续行为

### 场景 5：清空空队列
**触发**：队列为空时用户点击"清空队列"

**执行步骤**：
1. 弹出确认对话框
2. 用户确认后执行清空操作
3. 由于队列已为空，操作为空操作（无实际影响）
4. 队列页面保持空状态显示

### 场景 6：清空队列后从收件箱恢复
**触发**：清空队列后，收件箱中有新剧集

**执行步骤**：
1. 空状态视图检测到收件箱有新剧集
2. 显示提示消息："收件箱中有新剧集"
3. 显示"前往收件箱"按钮
4. 用户点击按钮跳转到收件箱页面
5. 用户可从收件箱重新加入队列
