# SPEC: REQ-035 下载设置

## 需求概述

用户可以配置全局下载行为，包括：Feed 自动更新间隔、是否允许移动数据下更新、全局自动下载开关、自动下载间隔和数量限制、自动删除策略（播放后删除、收藏项保留）、缓存清理规则、HTTP 代理设置、下载文件存储目录选择。修改后立即生效。

## 原子场景

### 场景 35.1：设置 Feed 更新间隔（Happy Path）
**触发**: 用户修改 Feed 自动更新的时间间隔
**执行步骤**:
1. 用户进入设置 → 下载
2. 修改"更新间隔"设置
3. 选择间隔时间（如每小时、每 2 小时、手动等）
4. 应用监听到设置变更
5. 重启 Feed 更新定时任务，使用新间隔
6. 后续按新间隔自动检查新剧集

### 场景 35.2：开启/关闭移动数据更新（Happy Path）
**触发**: 用户更改是否允许移动网络下更新 Feed
**执行步骤**:
1. 用户切换"移动数据更新"开关
2. 应用监听到设置变更
3. 重启更新定时任务
4. 开启时：移动网络下也自动更新 Feed
5. 关闭时：仅在 WiFi 下自动更新

### 场景 35.3：开启全局自动下载（Happy Path）
**触发**: 用户开启自动下载新剧集
**执行步骤**:
1. 用户进入自动下载设置子页
2. 开启"自动下载"开关
3. 后续检测到新剧集时自动下载（受各播客单独设置影响）
4. 单播客可覆盖全局设置（启用/禁用/跟随全局）

### 场景 35.4：设置自动删除策略（Happy Path）
**触发**: 用户开启播放后自动删除
**执行步骤**:
1. 用户进入自动删除设置子页
2. 开启"自动删除"开关
3. 可选：播放完成后自动删除已下载文件
4. 可选：收藏剧集不被自动删除
5. 保存后清理算法按新规则执行

### 场景 35.5：设置 HTTP 代理（Happy Path）
**触发**: 用户配置下载代理服务器
**执行步骤**:
1. 用户点击"代理"设置项
2. 弹出代理配置对话框
3. 输入代理主机和端口
4. 确认后保存
5. 后续下载请求通过代理服务器

### 场景 35.6：选择下载存储目录（Happy Path）
**触发**: 用户更改下载文件存储位置
**执行步骤**:
1. 用户点击"数据目录"设置项
2. 弹出目录选择对话框
3. 显示可用存储位置（内部存储、SD 卡）
4. 用户选择目标目录
5. 保存路径并更新设置摘要
6. 后续下载文件存储到新目录

### 场景 35.7：缓存空间不足时自动清理（边界场景）
**触发**: 下载新剧集时存储空间不足
**执行步骤**:
1. 自动下载检测到空间不足
2. 触发缓存清理算法
3. 根据播放状态和队列位置清理旧剧集
4. 收藏项不被清理（若设置了保留收藏）
5. 清理后重试下载

### 场景 35.8：移动数据下载确认（边界场景）
**触发**: 用户在移动网络下手动触发下载
**执行步骤**:
1. 检测当前网络为移动数据
2. 检查用户是否允许移动数据下载
3. 若不允许，提示用户选择是否临时允许
4. 确认后继续下载，否则取消

### 场景 35.9：无存储空间时设置数据目录（异常场景）
**触发**: 用户选择的目录不可写入
**执行步骤**:
1. 应用验证目录可写性
2. 若不可写，显示错误提示
3. 用户重新选择有效目录

## 验收要点
- 可设置 Feed 自动更新间隔，修改后定时任务立即重启
- 可控制移动数据下是否自动更新
- 可开启/关闭全局自动下载
- 可设置自动删除策略（播放后删除、收藏保留）
- 可配置 HTTP 代理
- 可选择下载文件存储目录（支持 SD 卡）
- 缓存空间不足时按规则自动清理
- 移动数据下载有确认机制
